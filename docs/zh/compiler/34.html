运行时环境

转换需要将程序的静态源文本与在运行时必须发生的动态操作相关联以实现程序。 该程序由过程名称，标识符等组成，需要在运行时与实际内存位置进行映射。

运行时环境是目标机器的状态，其可以包括软件库，环境变量等，以向系统中运行的进程提供服务。

来源语言问题

激活树

程序由过程组成，过程定义是一种声明，它以最简单的形式将标识符（过程名称）与语句（过程的主体）相关联。 每次执行程序都称为程序的激活。 激活的生命周期是执行过程中存在的步骤序列。 如果'a'和'b'是两个过程，那么它们的激活将是非重叠的（当一个被调用之后）或嵌套（嵌套过程）。 如果在相同过程的早期激活结束之前开始新的激活，则过程是递归的。 激活树显示控件进入和离开激活的方式。
激活树的属性是： -



  • 每个节点表示过程的激活。
  • 根显示主函数的激活。
  • 当且仅当控制从过程x流向过程y时，过程'x'的节点是过程'y'的节点的父节点。

示例 -考虑以下Quicksort程序 

 main() {

      Int n;
      readarray();
      quicksort(1,n);
}

quicksort(int m, int n) {

     Int i= partition(m,n);
     quicksort(m,i-1);
     quicksort(i+1,n);
}

第一个主函数作为root然后main调用readarray和quicksort。 Quicksort反过来再次调用partition和quicksort。 程序中的控制流程对应于从根开始的激活树的深度优先遍历。

控制堆栈和激活记录

控制堆栈或运行时堆栈用于跟踪实时过程激活，即执行尚未完成的过程。 程序名称在被调用（激活开始）时被推送到堆栈，并在它返回时弹出（激活结束）。 使用激活记录或帧来管理单个执行过程所需的信息。 调用过程时，激活记录被推入堆栈，一旦控制返回到调用者函数，就会弹出激活记录。

一般激活记录包括以下内容：

  • 局部变量：保存过程执行的本地数据。
  • 临时值 ：存储表达式求值中出现的值。
  • 机器状态：在函数调用之前保存有关机器状态的信息。
  • 访问链接（可选）：指其他激活记录中保存的非本地数据。
  • 控制链接（可选）：指向呼叫者的激活记录。
  • 返回值：被调用过程用于将值返回给调用过程
  • 实际参数

运行记忆的细分

运行时存储可以细分为：

  • 目标代码 - 程序代码，它是静态的，因为它的大小可以在编译时确定
  • 静态数据对象
  • 动态数据对象堆
  • 自动数据对象 - 堆栈

存储分配技术

I.静态存储分配

    • 对于任何程序，如果我们在编译时创建内存，将在静态区域中创建内存。
    • 对于任何程序，如果我们仅在编译时创建内存，则只创建一次内存。
    • 它不支持动态数据结构，即在编译时创建内存并在程序完成后解除分配。
    • 静态存储分配的缺点是不支持递归。
    • 另一个缺点是在编译时应该知道数据的大小

例如，FORTRAN旨在允许静态存储分配。

II。 堆栈存储分配

  • 存储按堆栈组织，激活记录在激活开始和结束时分别被推送和弹出。 本地包含在激活记录中，因此它们在每次激活时都绑定到新存储。
  • 堆栈分配支持递归

III。 堆存储分配

  • 内存分配和释放可以在任何时间和任何地点进行，具体取决于用户的要求。
  • 堆分配用于动态地为变量分配内存，并在不再需要变量时将其声明回来。
  • 支持递归。

参数传递

程序之间的通信介质称为参数传递。 调用过程中的变量值通过某种机制传递给被调用过程。

基本术语：

  • R值：表达式的值称为r值。 如果单个变量中包含的值出现在赋值运算符的右侧，则该值也将变为r值。 R值始终可以分配给其他变量。
  • L值：存储表达式的内存（地址）的位置称为该表达式的l值。 如果赋值运算符，它总是出现在左侧。

    i.Formal参数：获取调用者过程传递的信息的变量称为形式参数。 这些变量在被调用函数的定义中声明。

    ii.Actual Parameter：将值和函数传递给被调用函数的变量称为实际参数。 这些变量在函数调用中指定为参数。

将参数传递给过程的不同方式

  • 按价值呼叫
    在按值调用时，调用过程传递实际参数的r值，编译器将其放入调用过程的激活记录中。 形式参数保存调用过程传递的值，因此形式参数中的任何更改都不会影响实际参数。
  • 通过引用调用在通过引用调用时，形式和实际参数指的是相同的存储器位置。 实际参数的l值被复制到被调用函数的激活记录中。 因此，被调用的函数具有实际参数的地址。 如果实际参数没有l值（例如-i + 3），则在新的临时位置进行评估，并传递位置的地址。 形式参数中所做的任何更改都会反映在实际参数中（因为更改是在地址处进行的）。

  • 通过复制还原调用
    在调用时，复制还原编译器在调用过程时复制形式参数中的值，并在控制返回到被调用函数时将它们复制回实际参数中。 传递r值并在返回时将形式的r值复制到实际的l值中。
  • 按名称呼叫
    在按名称调用时，实际参数将替换在过程中出现形式的所有地方的形式。 它也被称为惰性评估，因为仅在需要时才对参数进行评估。
