为何选择FIRST和FOLLOW？

为何选择FIRST？
我们在上一篇关于语法分析简介的文章中看到了回溯的必要性， 这篇文章实际上是一个复杂的过程。 可以更简单地解决此问题：

如果编译器事先知道，那么“应用生产规则时产生的字符串的第一个字符”是什么，并将它与它看到的输入字符串中的当前字符或标记进行比较，它可以明智地采取决定适用哪种生产规则。

我们从前一篇文章中采用相同的语法： 

 S -> cAd
A -> bc|a 
And the input string is “cad”.

因此，在上面的例子中，如果它知道在读取输入字符串中的字符'c'并应用S-> cAd后，输入字符串中的下一个字符是'a'，那么它将忽略生成规则A-> bc（因为'b'是此生产规则生成的字符串的第一个字符，而不是'a'），并直接使用生成规则A-> a（ 因为'a'是由此生成的字符串的第一个字符生成规则，与输入字符串的当前字符相同，也就是“a” 。
因此，验证了如果编译器/解析器知道可以通过应用生成规则获得的字符串的第一个字符，那么它可以明智地应用正确的生成规则来获得给定输入字符串的正确语法树。

为什么要关注？
解析器面临另一个问题。 让我们考虑下面的语法来理解这个问题。 

 A -> aBb
 B -> c | ε
 And suppose the input string is “ab” to parse.

由于输入中的第一个字符是a，解析器应用规则A-> aBb。 

 A
        / | \
      a B b

现在，解析器检查输入字符串的第二个字符是b，并且要派生的非终端是B，但是解析器不能从B派生任何包含b作为第一个字符的字符串。
但是语法确实包含一个生成规则B - >ε，如果应用那么B将消失，解析器得到输入“ab”，如下所示。 但是解析器只有在知道跟随B的字符与输入中的当前字符相同时才能应用它。

在A - > aBb的RHS中，b遵循非终端B，即FOLLOW（B）= {b}，并且当前输入字符读取也是b。 因此解析器应用此规则。 并且它能够从给定的语法中获得字符串“ab”。 



 AA
        / | \ / \                                                
      a B b => ab |
           ε

因此，如果需要从解析树生成字符串，FOLLOW可以使非终端消失。

结论是，我们需要为给定的语法找到FIRST和FOLLOW集合，以便解析器可以在正确的位置正确应用所需的规则。
