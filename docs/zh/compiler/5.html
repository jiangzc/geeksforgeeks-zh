编译器中的错误检测和恢复

在此编译阶段，检测用户所做的所有可能错误，并以错误消息的形式向用户报告。 定位错误并将其报告给用户的过程称为错误处理过程 。
错误处理程序的功能

  • 发现
  • 报告
  • 复苏

错误分类

编译时错误有三种类型： -

词汇相位错误

在词法分析阶段检测到这些错误。 典型的词汇错误是



  • 超过标识符或数字常量的长度。
  • 出现非法人物
  • 无与伦比的字符串 
 Example 1 : printf("Geeksforgeeks");$
This is a lexical error since an illegal character $ appears at the end of statement.

Example 2 : This is a comment */
This is an lexical error since end of comment is present but beginning is not present.

错误恢复：
恐慌模式恢复

  • 在该方法中，一次一个地移除来自输入的连续字符，直到找到指定的一组同步令牌。 同步令牌是分隔符，例如; 要么 }
  • 优点是它易于实现并且保证不会进入无限循环
  • 缺点是跳过大量输入而不检查其他错误

句法相位错误

在语法分析阶段检测到这些错误。 典型的语法错误是

  • 结构错误
  • 缺少操作员
  • 拼错的关键字
  • 括号不平衡 
 Example : swicth(ch)
              {
                 .......
                 .......
              }

关键字开关被错误地写为swicth。 因此，出现“未识别的关键字/标识符”错误。

错误恢复：

  1. 恐慌模式恢复
    • 在该方法中，一次一个地删除来自输入的连续字符，直到找到指定的一组同步令牌。 同步令牌是熟食仪表，如; 要么 }
    • 优点是它易于实现并保证不会进入无限循环
    • 缺点是跳过大量输入而不检查其他错误
  2. 语句模式恢复
    • 在此方法中，当解析器遇到错误时，它会对剩余输入执行必要的更正，以便输入语句的其余部分允许解析器提前解析。
    • 更正可以删除额外的分号，用分号替换逗号或插入缺少的分号。
    • 在进行校正时，应注意不要进入无限循环。
    • 缺点是难以处理在检测点之前发生实际错误的情况。
  3. 生成错误
    • 如果用户知道可能遇到的常见错误，则可以通过使用生成错误构造的错误产生来扩充语法来合并这些错误。
    • 如果使用它，那么在解析期间可以生成适当的错误消息并且可以继续解析。
    • 缺点是难以维护。
  4. 全球更正

    • 解析器检查整个程序并尝试找出最接近的匹配，这是没有错误的。
    • 最接近的匹配程序具有较少的插入，删除和令牌更改以从错误输入中恢复。
    • 由于高时间和空间复杂性，该方法实际上没有实现。

语义错误

在语义分析阶段检测到这些错误。 典型的语义错误是

  • 不兼容的操作数类型
  • 未声明的变量
  • 不匹配实际参数与正式参数 
 Example : int a[10], b;
                 .......
                 .......
                 a = b;

由于a和b的类型不兼容，它会生成语义错误。

错误恢复

  • 如果遇到错误“未声明的标识符” ，则从中恢复对应标识符的符号表条目。
  • 如果两个操作数的数据类型不兼容，则自动类型转换由编译器完成。
