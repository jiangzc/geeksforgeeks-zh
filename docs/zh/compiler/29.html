编译器中的三个地址代码

先决条件 - 中间代码生成

三个地址代码是一种易于生成的中间代码，可以很容易地转换为机器代码。它最多使用三个地址和一个运算符来表示一个表达式，并且每个指令计算的值存储在临时变量中由编译器生成。 编译器决定三个地址代码给出的操作顺序。

一般代表 - 

 x = y op z

其中a，b或c表示操作数，如名称，常量或编译器生成的临时值，op表示运算符

示例-1：将表达式a * - （b + c）转换为三个地址代码。



示例2：为以下代码写入三个地址代码 

 for(i = 1; i<=10; i++)
 {
  a[i] = x * 5;                                       
 }

实施三个地址代码 -
三个地址码有三种表示形式

  1. 四人间
  2. 三同
  3. 间接三元组

1.四倍 -
它是由4个字段组成的结构，即op，arg1，arg2和result。 op表示运算符，arg1和arg2表示两个操作数，结果用于存储表达式的结果。

优势 -

  • 易于重新排列代码以进行全局优化。
  • 可以使用符号表快速访问临时变量的值。

坏处 -

  • 包含很多临时工。
  • 临时变量创建增加了时间和空间的复杂性。

示例 -考虑表达式a = b * - c + b * - c。
三个地址代码是： 

 t1 = uminus c
t2 = t1 * b
t3 = uminus c
t4 = t3 * b 
t5 = t2 + t4
a = t5

2.三元组 -
当需要引用另一个三元组值时，此表示不使用额外的临时变量来表示单个操作，而是使用指向该三元组的指针。 因此，它只包含三个字段，即op，arg1和arg2。

坏处 -

  • Temporaries隐含且难以重新排列代码。
  • 优化很困难，因为优化涉及移动中间代码。 移动三元组时，还必须更新引用它的任何其他三元组。 在指针的帮助下，可以直接访问符号表条目。

示例 -考虑表达式a = b * - c + b * - c

3.间接三元组 -
该表示使用指向所有对计算的引用的列表的指针，该列表分别进行并存储。 与四倍表示相比，它的实用性相似但需要的空间比它少。 Temporaries是隐式的，更容易重新排列代码。

示例 -考虑表达式a = b * - c + b * - c

问题 -为下面的表达式写出四倍，三倍和间接三元组：（x + y）*（y + z）+（x + y + z）

说明 -三个地址代码是： 

 t1 = x + y
t2 = y + z
t3 = t1 * t2
t4 = t1 + z
t5 = t3 + t4
