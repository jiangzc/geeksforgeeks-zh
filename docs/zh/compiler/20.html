自上而下解析器的分类

解析分为两类，即自上而下解析和自下而上解析。 自上而下的解析基于左大多数推导，而自下而上的解析依赖于逆向最大推导。

构造从根开始并向下到叶子的解析树的过程是Top-Down Parsing。

  • 自上而下的解析器构造来自语法，没有歧义和左递归。
  • Top Down Parsers使用最左边的派生来构造一个解析树。
  • 它允许一个没有Left Factoring的语法。

自上而下的解析分类 -

  1. 使用回溯：蛮力技术或递归下降解析
  2. 没有回溯：预测性解析或非递归解析或LL（1）解析或表驱动程序解析

蛮力技术或递归下降解析 -

  1. 每当非终端首次使用时，请使用第一个替代方案并与给定的I / P字符串进行比较
  2. 如果没有匹配，则使用第二个替代方法并与给定的I / P字符串进行比较。
  3. 如果再次找不到匹配，那么请选择替代方案，依此类推。
  4. 此外，如果对于至少一个替代方案发生匹配，则成功解析I / P字符串。

LL（1）或表驱动程序或预测解析器 -

  1. 在LL1中，1代表Look Ahead Symbol，1st L代表Left Most Derivation，而Second L代表字符串的从左到右扫描。
  2. LL（1）解析是从语法构造的，该语法没有左递归，公共前缀和歧义。
  3. LL（1）解析器依赖于1个向前看符号来预测生成以扩展解析树。
  4. 此解析器是非递归的。
