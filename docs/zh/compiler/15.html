FOLLOW在语法分析中设置

我们已经讨论了关于语法分析的以下主题。

语法分析简介
为何选择FIRST和FOLLOW？
首先在语法分析中设置

在这篇文章中，讨论了FOLLOW Set。

跟随（X）成为可以以某种句子形式立即出现在非终端X右侧的一组终端。
例： 

 S ->Aa | Ac
A ->b  

      SS / \ / \
    A a AC | |
    bb
Here, FOLLOW (A) = {a, c}


计算FOLLOW集的规则： 

 1) FOLLOW(S) = { $ } // where S is the starting Non-Terminal

2) If A -> pBq is a production, where p, B and q are any grammar symbols,
   then everything in FIRST(q) except Є is in FOLLOW(B.

3) If A->pB is a production, then everything in FOLLOW(A) is in FOLLOW(B).

4) If A->pBq is a production and FIRST(q) contains Є, 
   then FOLLOW(B) contains { FIRST(q) – Є } U FOLLOW(A)

例1： 

 Production Rules:
E -> TE'
E' -> +T E'|Є
T -> F T'
T' -> *F T' | Є
F -> (E) | id

FIRST set
FIRST(E) = FIRST(T) = { ( , id }
FIRST(E') = { +, Є }
FIRST(T) = FIRST(F) = { ( , id }
FIRST(T') = { *, Є }
FIRST(F) = { ( , id }

FOLLOW Set
FOLLOW(E) = { $ , ) } // Note ')' is there because of 5th rule
FOLLOW(E') = FOLLOW(E) = { $, ) } // See 1st production rule
FOLLOW(T) = { FIRST(E') – Є } U FOLLOW(E') = { + , $ , ) }
FOLLOW(T') = FOLLOW(T) = { + , $ , ) }
FOLLOW(F) = { FIRST(T') – Є } U FOLLOW(T') = { *, +, $) }


例2： 

 Production Rules:
S -> ACB|Cbb|Ba
A -> da|BC
B-> g|Є
C-> h| Є

FIRST set
FIRST(S) = FIRST(A) U FIRST(B) U FIRST(C) = { d, g, h, Є, b, a}
FIRST(A) = { d } U FIRST(B) = { d, g, h, Є }
FIRST(B) = { g, Є }
FIRST(C) = { h, Є }

FOLLOW Set
FOLLOW(S) = { $ }
FOLLOW(A) = { h, g, $ }
FOLLOW(B) = { a, $, h, g }
FOLLOW(C) = { b, g, $, h }


注意 ：

  1. Є作为FOLLOW并不意味着什么（Є是一个空字符串）。
  2. $被称为结束标记，表示输入字符串的结尾，因此在解析时用于指示输入字符串已完全处理。
  3. 上面使用的语法是Context-Free Grammar（CFG）。 可以使用CFG指定编程语言的语法。
  4. CFG的形式为A - > B，其中A是单个非终端，B可以是一组语法符号（即终端以及非终端）
