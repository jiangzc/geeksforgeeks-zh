中间代码生成

在编译器的分析综合模型中，编译器的前端将源程序转换为独立的中间代码，然后编译器的后端使用此中间代码生成目标代码（可由机器理解） ）。

使用与机器无关的中间代码的好处是：

  • 由于机器独立的中间代码，可移植性将得到增强。例如，假设编译器将源语言转换为其目标机器语言而没有生成中间代码的选项，那么对于每个新机器，完整的本机编译器是需要。 因为很明显，根据机器规格，编译器本身有一些修改。
  • 促进重新定位
  • 通过优化中间代码，更容易应用源代码修改来提高源代码的性能。

如果我们直接从源代码生成机器代码，那么对于n个目标机器，我们将有n个优化器和n个代码生成器，但如果我们将有一个独立于机器的中间代码，
我们只有一个优化器。 中间代码可以是特定于语言的（例如，用于Java的字节码）或语言。 独立的（三地址代码）。

以下是常用的中间代码表示：



  1. 后缀表示法 -

    写a和b之和的普通（中缀）方式是中间的运算符：a + b
    同一表达式的后缀表示法将操作符置于右端作为ab +。 通常，如果e1和e2是任何后缀表达式，并且+是任何二元运算符，则将+应用于由e1和e2表示的值的结果是e1e2 +的后缀表示法。 后缀表示法中不需要括号，因为运算符的位置和arity（参数个数）仅允许一种方式来解码后缀表达式。 在后缀表示法中，运算符遵循操作数。

    示例 -表达式（a-b）*（c + d）+（a-b）的后缀表示形式为：ab - cd + ab - + *。
    阅读更多： Infix到Postfix

  2. 三地址代码 -
    涉及不超过三个引用的语句（两个用于操作数，一个用于结果）称为三个地址语句。 三个地址语句的序列称为三个地址代码。 三个地址语句的形式为x = y op z，这里x，y，z将具有地址（存储位置）。 有时语句可能包含少于三个引用，但仍然称为三个地址语句。

    示例 -表达式a + b * c + d的三个地址代码：

    T 1 = b * c
    T 2 = a + T 1
    T 3 = T 2 + d

    T 1，T 2，T 3是临时变量。

  3. 语法树 -
    语法树只不过是解析树的压缩形式。 解析树的运算符和关键字节点被移动到它们的父节点，并且单个节目链被语法树中的单个链接替换，内部节点是运算符，子节点是操作数。 为了形成语法树，将括号放在表达式中，这样就很容易识别哪个操作数应该首先出现。

    示例 -
    x =（a + b * c）/（a - b * c）
