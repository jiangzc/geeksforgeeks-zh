解析器的作用：

在语法分析阶段，编译器验证词法分析器生成的标记是否根据语言的语法规则进行分组。 这是由解析器完成的。 解析器从词法分析器获取一串令牌，并验证该字符串可以是源语言的语法。 它检测并报告任何语法错误，并生成一个解析树，从中可以生成中间代码。

在讨论解析器类型之前，我们将讨论关于理解解析所需的一些重要事项的一些想法。

上下文免费语法 ：
编程语言的语法由上下文无关语法（CFG）描述。 CFG由一组终端，一组非终端，一个起始符号和一组产品组成。
符号 - α→β其中α是单个变量[V]
β∈（V + T）*

删除左递归：
如果语法具有非终端（变量）S使得它们是派生，则该语法是递归的
S - >Sα| β
其中α∈（V + T）*和β∈（V + T）*（不以S开头的终端和非终端序列）
由于存在左递归，一些自上而下的解析器进入无限循环，因此我们必须消除左递归。
让制作的形式为A - >Aα1| Aα2| Aα3| ...... .. | Aαm| β1| β2| ...。 | βn
没有βi以A开头。 然后我们用A替换A-productions
A - >β1A'| β2A'| ...... .. | βn''
A' - >α1A'| α2A'| α3A'| ...... .. | αmA'| ε
非终结符A生成与以前相同的字符串，但不再是递归的。

删除左保理：
据说语法在形式时被考虑在内 -
A - >αβ1| αβ2| αβ3| ...... | αβn| γ即，产品以相同的终端（或终端组）开始。 在看到输入α时，我们无法立即告诉哪个生产选择扩展A.
左因子分解是一种语法转换，可用于产生适用于预测或自上而下解析的语法。 当两个替代A制作之间的选择不明确时，我们可能能够重写制作以推迟决定，直到看到足够的输入做出正确的选择。
对于语法A - >αβ1| αβ2| αβ3| ...... | αβn| γ
等效的左因式语法将是 -
A - >αA'| γ
A' - >β1| β2| β3| ...... | βn


从给定语法中导出字符串的过程称为派生（解析）。
根据推导的完成方式，我们有两种解析器： -

  1. 自上而下的解析器
  2. 自下而上的解析器

我们将从GATE的角度研究解析器。

自上而下的解析器
自上而下解析尝试构建从根到离开的解析树。 自上而下的解析器将从开始符号开始并继续到字符串。 它遵循最左边的推导。 在最左边的推导中，总是选择每个句子中最左边的非终端。

          else if ( Xi equals the current input, increment input)

      }

重要笔记 

 1. If a grammar contain left factoring then it can not be LL(1)
        Eg - S -> aS | a ---- both productions go in a
      2. If a grammar contain left recursion it can not be LL(1)
        Eg - S -> Sa | b 
                S -> Sa goes to FIRST(S) = b
                S -> b goes to b, thus b has 2 entries hence not LL(1)
      3. If a grammar is ambiguous then it can not be LL(1)
      4. Every regular grammar need not be LL(1) because 
         regular grammar may contain left factoring, left recursion or ambiguity.
