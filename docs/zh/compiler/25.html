语法定向翻译

背景： Parser使用CFG（Context-free-Grammer）来验证输入字符串并为编译器的下一阶段生成输出。 输出可以是解析树或抽象语法树。 现在，为了将语义分析与编译器的语法分析阶段交错，我们使用语法定向转换。

定义
语法定向翻译是语法的增强规则，便于语义分析。 SDT涉及以附加到节点的属性的形式自下而上和/或自上而下地传递信息。 语法定向转换规则使用1）节点的词汇值，2）常量和3）与定义中的非终端相关联的属性。

例 

 E -> E+T | T
T -> T*F | F
F -> INTLIT

这是一种语法，用于语法验证其中包含加法和乘法的表达式。 现在，为了进行语义分析，我们将SDT规则扩展到这个语法，以便在解析树上传递一些信息并检查语义错误（如果有的话）。 在这个例子中，我们将重点关注给定表达式的评估，因为我们没有任何语义断言来检查这个非常基本的例子。 

 E -> E+T { E.val = E.val + T.val } PR#1
E -> T { E.val = T.val } PR#2
T -> T*F { T.val = T.val * F.val } PR#3
T -> F { T.val = F.val } PR#4
F -> INTLIT { F.val = INTLIT.lexval } PR#5

为了进一步理解翻译规则，我们将第一个SDT增加到[E - > E + T]生成规则。 考虑的转换规则具有val作为非终端的属性 - E和T.转换规则的右侧对应于生产规则的右侧节点的属性值，反之亦然。 概括，SDT是CFG的扩充规则，它将1）属性集与语法的每个节点相关联，2）使用属性，常量和词汇值将转换规则集合到每个生产规则。

附加信息
Synthesized Attributes是仅依赖于子节点的属性值的属性。
因此，[E - > E + T {E.val = E.val + T.val}]具有对应于节点E的合成属性val。如果合成了增强语法中的所有语义属性，则进行一次深度优先搜索遍历。任何顺序都足以进行语义分析阶段。

继承属性是依赖于父和/或兄弟属性的属性。
因此[Ep - > E + T {Ep.val = E.val + T.val，T.val = Ep.val}]，其中E＆Ep是相同的生产符号注释以区分父母和孩子，具有继承权属性val对应于节点T.
