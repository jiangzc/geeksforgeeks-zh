编译器中的符号表

先决条件 - 编译器的阶段

符号表是由编译器创建和维护的重要数据结构，用于跟踪变量的语义，即它存储有关范围和关于名称的绑定信息的信息，有关各种实体的实例的信息，例如变量和函数名称，类，对象等

  • 它建立在词法和语法分析阶段。
  • 信息由编译器的分析阶段收集，并由编译器的综合阶段使用以生成代码。
  • 编译器使用它来实现编译时效率。
  • 它由编译器的各个阶段使用如下： -
    1. 词法分析：在表中创建新表条目，例如关于令牌的条目。
    2. 语法分析：在表中添加有关属性类型，范围，维度，引用线，使用等的信息。
    3. 语义分析：使用表中的可用信息来检查语义，即验证表达式和赋值在语义上是否正确（类型检查）并相应地更新它。
    4. 中间代码生成：引用符号表以了解分配了多少和什么类型的运行时，并且表有助于添加临时变量信息。
    5. 代码优化：使用符号表中的信息进行机器相关优化。
    6. 目标代码生成：使用表中存在的标识符的地址信息生成代码。

符号表条目 - 符号表中的每个条目都与支持不同阶段编译器的属性相关联。
存储在符号表中的项目：

  • 变量名称和常量
  • 过程和函数名称
  • 文字常量和字符串
  • 编译器生成临时文件
  • 源语言的标签

符号表中编译器使用的信息：

  • 数据类型和名称
  • 申报程序
  • 存储偏移
  • 如果是结构或记录那么，指向结构表的指针。
  • 对于参数，参数是通过值传递还是通过引用传递
  • 传递给函数的参数的数量和类型
  • 基地址

符号表的实现 -
以下是实现符号表的常用数据结构： -

  1. 清单 -
    • 在此方法中，数组用于存储名称和相关信息。
    • 在所有存储的记录的末尾保持指针“可用” ，并且在它们到达时按顺序添加新名称
    • 要搜索名称，我们从列表的开头直到可用的指针开始，如果没有找到，我们会收到错误“使用未声明的名称”
    • 在插入新名称时，我们必须确保它不存在，否则会出现错误，即“多个已定义的名称”
    • 插入是快速O（1），但对于大型表，查找速度很慢 - 平均为O（n）
    • 优点是它占用的空间最小。

  2. 关联名单 -
    • 此实现使用链接列表。 每个记录都会添加一个链接字段。
    • 搜索名称按链接字段链接指示的顺序进行。
    • 保持指针“First”以指向符号表的第一记录。
    • 插入是快速O（1），但对于大型表，查找速度很慢 - 平均为O（n）

  3. 哈希表 -
    • 在散列方案中，维护了两个表 - 散列表和符号表，并且是实现符号表的最常用方法。
    • 哈希表是一个索引范围为0到表大小的数组 - 1.这些条目是指向符号表名称的指针。
    • 要搜索名称，我们使用哈希函数，这将导致0到tablesize - 1之间的任何整数。
    • 插入和查找可以非常快速地进行 - O（1）。
    • 优点是wuick搜索是可能的，缺点是哈希实现起来很复杂。

  4. 二叉搜索树 -
    • 实现符号表的另一种方法是使用二叉搜索树，即我们添加两个链接字段，即左和右子。
    • 所有名称都创建为根节点的子节点，始终遵循二叉搜索树的属性。
    • 插入和查找平均为O（log 2 n）。
