语法分析简介

当输入字符串（源代码或某种语言的程序）被提供给编译器时，编译器会在几个阶段处理它，从词法分析 （扫描输入并将其划分为标记）开始到目标代码生成。

语法分析或解析是第二阶段，即在词法分析之后。 它检查给定输入的语法结构，即给定输入是否使用正确的语法（输入已被写入的语言）。 它通过构建称为Parse树或语法树的数据结构来实现。 通过使用语言的预定义语法和输入字符串来构造解析树。 如果在语法树的帮助下（在派生过程中）可以生成给定的输入字符串，则会发现输入字符串的语法正确。

语言语法由生产规则组成。

假设语言语法的生成规则是： 

 S -> cAd
  A -> bc|a
  And the input string is “cad”.

在上面的步骤iii中，生产规则A-> bc不适合应用（因为生成的字符串是“cbcd”而不是“cad”），这里解析器需要回溯，并应用下一个可用的生成规则在步骤iv中示出了A，并且产生了字符串“cad”。
