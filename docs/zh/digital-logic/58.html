块编码

将数字数据转换为数字信号涉及三种技术：

  1. 线路编码
  2. 块编码
  3. 扰

在总是需要线编码的情况下，可能需要也可能不需要块编码和加扰。

块编码有助于错误检测和信号的重传。 它通常被称为mB / nB编码，因为它用n位数据组（其中n> m）替换每个m位数据组。 因此，它增加了额外的比特（冗余比特），有助于在接收者和发送者端同步，并提供某种错误检测能力。

它通常包括三个步骤：划分，替换和组合。 在除法步骤中，比特序列被分成m比特组。 在替换步骤中，我们将m位组替换为n位组。 最后，将n比特组组合在一起以形成比原始比特具有更多比特的流。

mB / nB编码的示例：



4B / 5B（四个二进制/五个二进制） -
该编码方案与NRZ-1组合使用。 NRZ-I的问题在于它对于长序列的零具有同步问题。 因此，为了克服它，我们在用NRZ-I编码之前将比特流替换为4比特到5比特的数据组。 所以它没有很长的零流。 块编码流的连续零个数不超过三个（参见编码表）。

在接收器处，首先将NRZ-1编码的数字信号解码为比特流，然后再次解码以移除冗余比特。

缺点 -虽然4B / 5B编码解决了同步问题，但它增加了NRZ-L的信号速率。而且，它没有解决NRZ-L的直流分量问题。

8B / 10B（八进制/十二进制） -
该编码类似于4B / 5B编码，除了一组8位数据现在被10位代码替换并且它提供比4B / 5B更大的错误检测能力。
它实际上是5B / 6B和3B / 4B编码的组合.10比特块的大多数五个有效位被馈送到5B / 6B编码器; 至少3个有效位被馈入3B / 4B编码器。 完成拆分以简化映射表。

一组8比特可以具有2 ^ 8个不同的组合，而一组10比特可以具有2 ^ 10个不同的组合。 这意味着有2 ^ 10-2 ^ 8 = 768个冗余组不用于8B / 10B编码，可用于错误检测和差异检查。

因此，这种技术优于4B / 5B，因为它具有更好的错误检查能力和更好的同步性。
