1.要将浮点转换为十进制，我们在32位浮点表示中有3个元素：
i）签名
ii）指数
iii）尾数

  • 符号位是二进制表示的第一位。 “1”表示负数，“0”表示正数。
    示例： 11000001110100000000000000000000这是负数。

  • 指数由接下来的8位二进制表示决定。 127是32位浮点表示的唯一编号。 它被称为偏见。 它由2 k-1 -1确定，其中'k'是指数字段中的位数。

    在8位表示中有2个指数位，在32位表示中有8个指数位。
    从而
    偏置= 3表示8位转换（2 2-1 -1 = 4-1 = 3）
    对于32位转换，bias = 127。 （2 8-1 -1 = 128-1 = 127）

    示例： 01000001110100000000000000000000
    10000011 =（131） 2
    131-127 = 4

    因此，2的指数将是4，即2 4 = 16。

  • 尾数是从二进制表示的剩余24位计算的。 它由'1'和小数部分组成，由下式确定：

    例：



    01000001110100000000000000000000

    尾数的小数部分由下式给出：

    1 *（1/2）+ 0 *（1/4）+ 1 *（1/8）+ 0 *（1/16）+ ......... = 0.625

    因此，尾数将是1 + 0.625 = 1.625

    十进制数因此给出为：符号*指数*尾数=（ - 1）*（16）*（1.625）= -26

2.要将十进制转换为浮点，我们在32位浮点表示中有3个元素：
i）签名（MSB）
ii）指数（MSB后8位）
iii）尾数（剩余23位）

  • 符号位是二进制表示的第一位。 “1”表示负数，“0”表示正数。
    示例：将-17转换为32位浮点表示符号位= 1
  • 指数由最接近的小于或等于2 n的数字决定。 对于17,16是最接近的2 n 。 因此，2的指数将是4，因为2 4 = 16. 127是32位浮点表示的唯一编号。 它被称为偏见。 它由2 k-1 -1确定，其中'k'是指数字段中的位数。

    因此，对于32位，偏差= 127。 （2 8-1 -1 = 128-1 = 127）

    现在，127 + 4 = 131，即二进制表示中的10000011。

  • 尾数：二进制17 = 10001。

    移动二进制点，使左侧只有一位。 调整指数2，使值不会改变。 这使数字正常化。 1.0001 x 2 4 。 现在，考虑小数部分并通过添加零来表示为23位。

    00010000000000000000000

    • 因此，-17的浮点表示为1 10000011 0001000000000000000000000
