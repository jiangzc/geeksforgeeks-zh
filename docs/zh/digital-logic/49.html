计数器增量的摊销分析

摊销分析是指确定序列（非个人）操作的时间平均运行时间。 它与普通案例分析不同，因为在这里，我们不假设数据以平均（非常非常糟）的方式排列，就像我们为平均案例分析进行快速排序一样 。 也就是说，摊销分析是最坏的案例分析，但是对于一系列操作而不是个别操作。 它适用于包含操作顺序的方法，其中绝大多数操作都很便宜但是一些操作很昂贵。 这可以借助于下面实现的二进制计数器来可视化。

让我们通过在C中实现增量计数器来看到这一点。首先，让我们看看计数器增量是如何工作的。
让变量i包含值0，我们执行i ++很多次。 因为，在硬件上，每个操作都以二进制形式执行。 让二进制数存储在8位。 所以，价值是00000000.让我们增加很多时间。 因此，我们发现的模式如下：

00000000,00000001,00000010,00000011,00000100,00000101,00000110,00000111,00001000等......

脚步 ：
1.从最右边迭代并使所有一个为零直到找到第一个零。
2.迭代后，如果index大于或等于零，则将该位置的零位置设为1。

建议：在继续使用解决方案之前，请先在{IDE}上尝试您的方法。 



#include <bits / stdc++.h>
using namespace std;

int main()
{
    char str[] = "10010111";
    int length = strlen(str);
    int i = length - 1;
    while (str[i] == '1') {
        str[i] = '0';
        i--;
    }
    if (i >= 0)
        str[i] = '1';
    printf("% s", str);
}

输出： 

 10011000

简单地看一下程序或算法，它的运行成本看起来与位数成正比，但实际上它与位数不成比例。 我们来看看怎么样！

假设增量运算是在k时间执行的。 我们看到，在每个增量中，最右边的位都会被翻转。 因此，LSB的翻转次数为k。 因为，第二个最右边在间隙之后翻转，即以2个增量1次。 最右边 - 4次增量中的1次。 最右边第4个 - 增加8个1次。 因此，第二最右位的翻转次数为k / 2，最右边第3位为k / 4，第四最右位为k / 8，依此类推......

总费用将是翻转总数，即
C（k）= k + k / 2 + k / 4 + k / 8 + k / 16 + ......这是几何级数系列而且，
C（k）<k + k / 2 + k / 4 + k / 8 + k / 16 + k / 32 + ......达到无穷大
所以，C（k）<k /（1-1 / 2）
所以，C（k）<2k
所以，C（k）/ k <2
因此，我们发现计数器一次递增的平均成本是恒定的，并且它不依赖于位数。 我们得出结论，计数器的增量是恒定成本操作。
