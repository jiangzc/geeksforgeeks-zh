三层交换机的VLAN间路由

先决条件 - 访问和中继端口
通常，路由器用于划分广播域和交换机（在第2层）在单个广播域中操作，但交换机也可以使用VLAN（虚拟LAN）的概念划分广播域。

Vlan是相同或不同广播域中设备的逻辑分组。 默认情况下，所有交换机端口都在Vlan 1中。由于单个广播域被划分为多个广播域，因此路由器或第3层交换机用于不同Vlan之间的相互通信。不同Vlan的相互通信过程称为Inter Vlan路由（IVR）。

假设我们已经制作了两个名为sales and finance的逻辑组设备（vlan）。如果销售部门的设备想要与财务部门的设备进行通信，则必须执行Vlan间路由。 这些可以由路由器或第3层交换机执行。

交换机虚拟接口（SVI） -
SVI是多层交换机上的逻辑接口，它为与该VLAN相关的所有交换机端口提供数据包的第3层处理。可以为Vlan创建单个SVI。 第3层交换机的SVI提供管理和路由服务，而第2层交换机上的SVI仅提供管理服务，如创建vlan或telnet / SSH服务。

三层交换机进行Inter Vlan路由的过程 -
为各个Vlan创建的SVI就像该路由器的子接口一样（在路由器的过程中），作为该Vlan的默认网关。 如果要将数据包传递到不同的vlan，即要在第3层交换机上执行VLAN间路由，则首先将数据包传送到第3层交换机，然后传送到目的地，就像在路由器上的路由器一样。



配置 -

这是一种拓扑结构，其中我们有一个连接到主机设备的第3层交换机，即PC1，PC2，PC3，PC4。 主机PC1，PC2将在Vlan 10和PC3中，PC4将在Vlan 20中。为所有主机提供IP地址。 PC1-192.168.1.10 / 24，PC2-192.168.1.20 / 24，PC3-192.168.2.10 / 24，PC4-192.168.2.20 / 24。

现在在第3层交换机上创建vlan，即交换机端口fa0 / 1,2上的vlan 2和vlan 3上的fa0 / 3,4。 

 Switch# vlan 2
Switch# vlan 3
Switch# int range fa0/1-2
Switch# switchport access vlan 2
Switch# int range fa0/3-4
Switch# switchport access vlan 3

现在为vlan 2创建SVI，为其提供IP地址192.168.1.1/24和vlan 3的SVI，给出IP地址192.168.2.1/24 

 Switch# ip routing
Switch# int vlan 2
Switch# ip address 192.168.1.1 255.255.255.0
Switch# int vlan 3
Switch# ip address 192.168.2.1 255.255.255.0

现在，如果我们尝试将PC1 ping到PC4。

数据包首先被传送到交换机然后到达目的地。 由于目的地存在于其他网络中，因此数据包将首先传送到具有两个vlan的SVI（充当网关）的交换机。

优点 -
在Stick on stick方法中，需要交换机和路由器，但在使用第3层交换机时，单个交换机将执行inter-vlan路由以及第2层功能（Vlan），因此这种方法具有成本效益且配置也较少需要。
