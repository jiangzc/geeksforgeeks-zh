网络拓扑| 计算机网络

通过发送器和接收器组成的包括节点和连接线的网络的布置称为网络拓扑。 各种网络拓扑结构如下：

a）网格拓扑：

在网状拓扑中，每个设备都通过特定通道连接到另一个设备。

图1 ：每个设备都通过专用通道与另一个设备连接。 这些渠道称为链接。

  • 如果假设，N个设备在网状拓扑中相互连接，则每个设备所需的端口总数为N-1。 在图1中，有5个设备相互连接，因此所需的端口总数为4。
  • 如果假设，N个设备在网状拓扑中相互连接，则连接它们所需的专用链路总数为N C 2，即N（N-1）/ 2。 在图1中，有5个设备相互连接，因此所需的链路总数为5 * 4/2 = 10。

此拓扑的优点：



  • 它很健壮。
  • 故障很容易诊断出来。 数据是可靠的，因为数据通过专用信道或链路在设备之间传输。
  • 提供安全性和隐私。

此拓扑的问题：

  • 安装和配置很困难。
  • 由于需要大量布线，因此电缆成本高，因此适用于较少数量的设备。
  • 维护成本很高。

b）星形拓扑：

在星型拓扑中，所有设备都通过电缆连接到单个集线器。 该集线器是中心节点，所有其他节点都连接到中心节点。 集线器本质上可以是无源的，即不是诸如广播设备之类的智能集线器，同时集线器可以是智能的，称为有源集线器。 有源集线器中有中继器。

图2 ：具有连接到单个连接点即集线器的四个系统的星形拓扑。

此拓扑的优点：

  • 如果N个设备在星型拓扑中相互连接，则连接它们所需的电缆数量为N.因此，它很容易设置。
  • 每个设备只需要1个端口即连接到集线器。

此拓扑的问题：

  • 如果整个拓扑所依赖的集中器（集线器）发生故障，整个系统将崩溃。
  • 安装成本很高。
  • 性能基于单一集中器即集线器。

c）总线拓扑：

总线拓扑是一种网络类型，其中每台计算机和网络设备都连接到单根电缆。 它将数据从一端传输到另一端。 总线拓扑中没有双向功能。

图3 ：具有共享主干电缆的总线拓扑。 节点通过下拉线连接到通道。

此拓扑的优点：

  • 如果N个设备在总线拓扑中相互连接，那么连接它们所需的电缆数量为1，称为主干电缆，需要N个下降线。
  • 与其他拓扑相比，电缆的成本较低，但它用于构建小型网络。

此拓扑的问题：

  • 如果普通电缆出现故障，整个系统将崩溃。
  • 如果网络流量很大，则会增加网络中的冲突。 为了避免这种情况，MAC层中使用了各种协议，称为Pure Aloha，Slotted Aloha，CSMA / CD等。

d）环形拓扑：

在此拓扑中，它形成一个环，将设备与其恰好两个相邻设备连接起来。

图4 ：环形拓扑由4个站连接，每个站形成一个环...

环形拓扑中发生的以下操作是：

  1. 一个站称为监控站，负责执行操作。
  2. 为了传输数据，站必须保持令牌。 传输完成后，将释放令牌供其他站点使用。
  3. 当没有电台正在发送数据时，令牌将在环中循环。
  4. 有两种类型的令牌释放技术： 早期令牌释放在发送数据之后释放令牌， 延迟令牌释放在从接收器收到确认之后释放令牌。

此拓扑的优点：

  • 在这种类型的拓扑中，碰撞的可能性最小。
  • 便宜的安装和扩展。

此拓扑的问题：

  • 在此拓扑中难以进行故障排除。
  • 在站之间添加站或移除站可能会干扰整个拓扑。

e）混合拓扑：

该拓扑结构是上面描述的两个或更多拓扑的集合。 这是一种可扩展的拓扑，可以轻松扩展。 它是可靠的，但同时它是一种昂贵的拓扑结构。

图5 ：混合拓扑结构，它是环形和星形拓扑的组合。
