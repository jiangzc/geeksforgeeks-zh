载波侦听多路访问（CSMA）

先决条件 - 多种访问协议
开发此方法是为了减少两个或多个站开始通过数据链路层发送信号时发生冲突的可能性。 载波侦听多路访问要求每个站在发送之前首先检查介质的状态 。

脆弱的时间 - 

 Vulnerable time = Propagation time (Tp)

可以应用持久性方法来帮助站在信道忙/空闲时采取措施。

2.具有冲突避免的载波侦听多路访问（CSMA / CA） -

CSMA / CA背后的基本思想是站点应该能够在发送时接收以检测来自不同站点的冲突。 在有线网络中，如果发生了冲突，则接收信号的能量几乎加倍，并且站可以感知到碰撞的可能性。 在无线网络的情况下，大部分能量用于传输，并且如果发生冲突，接收信号的能量仅增加5-10％。 它不能被车站用来感知碰撞。 因此， CSMA / CA专为无​​线网络设计 。

这是三种策略：

  1. 帧间空间（IFS） -当站点发现信道繁忙时，它会发送一段称为IFS时间的时间。 IFS还可用于定义站或帧的优先级。 IFS越低越优先。
  2. 竞争窗口 -分为时隙的时间量。准备发送帧的站选择随机数的时隙作为等待时间 。
  3. 致谢 -肯定确认和超时计时器有助于保证帧的成功传输。
