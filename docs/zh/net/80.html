VRRP（虚拟路由器冗余协议）| 介绍和配置

VRRP是一种开放标准协议，用于在网络中提供冗余。 它是网络层协议（协议号-112）。

组中的路由器（组成员）数量充当虚拟逻辑路由器，它将成为所有本地主机的默认网关。 如果一台路由器出现故障，则可以使用其中一个组成员来转发流量。

与VRRP相关的一些重要术语：

  1. 虚拟IP地址： IP地址被指定为本地子网的虚拟IP地址，该子网被配置为所有本地主机的默认网关。
  2. 虚拟MAC地址：通过将最后8个字节作为十六进制的VRRP组号自动生成虚拟MAC地址。 在VRRP中，使用的Mac地址为0000.5e00.01xx。 这里，xx是十六进制的VRRP组号。
  3. Master路由器：其中一个VRRP备份组成员被选为Master路由器，承担转发本地流量的责任。 路由器是根据优先级选出的。 如果某个VRRP备份组成员的优先级高于其他成员，则会被选为Master路由器。 如果优先级相同（默认为100），则具有最高IP地址的路由器将成为主路由器。
  4. 备份路由器：只有一个VRRP备份组成员将成为主路由器，而其他成员将备份路由器。 如果主路由器出现故障，则其中一个备用路由器将成为主路由器。
  5. 主广告定时器：主路由器每1秒在224.0.0.18多播保活消息。
  6. 主死定时器：如果未收到主广告消息，备用路由器将承担主路由器职责的时间。 默认情况下，它是3.69秒。
  7. Prempt：这是一个备用路由器成为主路由器的状态（当主路由器出现故障时）。 此外，当主路由器再次启动时，它将成为主路由器，因为它的优先级仍然更高。
  8. 对象跟踪： VRRP对象跟踪提供了一种最佳路由器将成为主路由器的方式。 当前主路由器连续轮询跟踪对象状态（例如线路协议向上或向下）。 假设主路由器线路协议是关闭的，则备份路由器的优先级将动态增加，即它将成为主路由器。
  9. 认证：
    VRRP支持3种身份验证 -
    1.没有身份验证
    2.纯文本身份验证
    3. MD5身份验证

VRRP在以下任何一种情况下拒绝数据包：

1.认证方案在路由器和传入数据包上有所不同。
2. MD5摘要在路由器和传入数据包上有所不同。
3.路由器和传入数据包上的文本验证字符串不同。



配置：

在上面给出的拓扑中，有2个路由器名为R1和R2，其中R1 fa 0/0 ip地址为10.1.1.1/24，R2 fa0 / 0 ip地址为10.1.1.2/24。

为路由器R1分配IP地址： 

 r1# int fa0/0
r1# ip add 10.1.1.1 255.255.255.0

为路由器r2分配IP地址。 

 r2# int fa0/0
r2# ip address 10.1.1.2 255.255.255.0

现在，让我们提供虚拟IP地址（10.1.1.100），组名VRRP_TEST ，组号10和优先级110.此外，这里默认情况下已启用抢占，即如果主路由器发生故障，则备用路由器将自动成为主路由器。 

 r1# int fa0/0
r1# vrrp 10 ip 10.1.1.100
r1# vrrp 10 name VRRP_TEST
r1# vrrp 10 priority 110

现在，提供虚拟IP地址（10.1.1.100），组名VRRP_TEST和优先级100.此外，还分配了组号10。 

 r2# int fa0/0
r2# vrrp 10 ip 10.1.1.100
r2# vrrp 10 name VRRP_TEST
r2# vrrp 10 priority 100

注意：因为提供优先级110到r1，它将成为主路由器。
