LZW（Lempel-Ziv-Welch）压缩技术

为什么我们需要压缩算法？

有两类压缩技术，有损和无损。 虽然每个都使用不同的技术来压缩文件，但两者都有相同的目的：在图形中查找重复数据（LZW的GIF）并使用更紧凑的数据表示。 无损压缩通过识别和消除统计冗余来减少比特。 在无损压缩中没有信息丢失。另一方面，有损压缩通过删除不必要或不太重要的信息来减少比特。 所以我们需要数据压缩主要是因为：

  • 未压缩的数据可能占用大量空间，这对于有限的硬盘空间和互联网下载速度并不好。
  • 虽然硬件变得更好，更便宜，但减少数据大小的算法也有助于技术的发展。
  • 示例：一分钟未压缩的高清视频可以超过1 GB。如何在25 GB的蓝光光盘上安装两小时的电影？

有损压缩方法包括DCT（Discreet Cosine Transform），矢量量化和Huffman编码，而无损压缩方法包括RLE（运行长度编码），字符串表压缩，LZW（Lempel Ziff Welch）和zlib.There存在几种压缩算法，但我们正专注于LZW。

什么是Lempel-Ziv-Welch（LZW）算法？

LZW算法是一种非常常见的压缩技术。 此算法通常用于GIF，也可选择用于PDF和TIFF。 Unix的'compress'命令，以及其他用途。 它是无损的，意味着压缩时没有数据丢失。 该算法易于实现，并且在硬件实现中具有非常高的吞吐量的潜力。 它是广泛使用的Unix文件压缩实用程序压缩的算法，并用于GIF图像格式。



Idea依赖于重复出现的模式来节省数据空间。 LZW因其简单性和多功能性而成为通用数据压缩的最重要技术。 许多PC公用事业公司声称“硬盘容量翻倍”是其基础。

它是如何工作的？

LZW压缩的工作原理是读取一系列符号，将符号分组为字符串，然后将字符串转换为代码。 因为代码比它们替换的字符串占用更少的空间，所以我们得到压缩.LZW的特征包括，

  • LZW压缩使用代码表，4096作为表条目数的通用选择。 始终将代码表中的代码0-255分配为表示来自输入文件的单个字节。
  • 编码开始时，代码表仅包含前256个条目，表的其余部分为空白。 通过使用代码256到4095来表示字节序列来实现压缩。
  • 随着编码的继续，LZW识别数据中的重复序列，并将它们添加到代码表中。
  • 通过从压缩文件中获取每个代码并将其翻译通过代码表来查找它代表的字符或字符来实现解码。

示例：ASCII代码。通常，每个字符都存储有8个二进制位，最多允许256个唯一符号用于数据。 此算法尝试将库扩展为每个字符9到12位。新的唯一符号由先前在字符串中出现的符号组合组成。 它并不总能很好地压缩，尤其是短而多样的字符串。 但这对于压缩冗余数据很有用，并且不必使用数据保存新字典：此方法可以压缩和解压缩数据。

履行

压缩算法的思想如下：当正在处理输入数据时，字典保持最长遇到的单词和代码值列表之间的对应关系。 单词由相应的代码替换，因此输入文件被压缩。 因此，随着输入数据中长的重复单词的数量增加，算法的效率增加。

LZW编码 

 * PSEUDOCODE
  1 Initialize table with single character strings
  2 P = first input character
  3 WHILE not end of input stream
  4 C = next input character
  5 IF P + C is in the string table
  6 P = P + C
  7 ELSE
  8   output the code for P
  9 add P + C to the string table
  10 P = C
  11 END WHILE
  12 output code for P

LZW减压

LZW解压缩器在解压缩期间创建相同的字符串表。 它从初始化为单个字符的前256个表条目开始。对于输入流中的每个字符更新字符串表，除了第一个字符。通过读取代码并通过正在构建的代码表翻译代码来实现解码。

LZW减压算法 

 * PSEUDOCODE
1 Initialize table with single character strings
2 OLD = first input code
3 output translation of OLD
4 WHILE not end of input stream
5 NEW = next input code
6  IF NEW is not in the string table
7 S = translation of OLD
8   S = S + C
9 ELSE
10  S = translation of NEW
11 output S
12   C = first character of S
13   OLD + C to the string table
14 OLD = NEW
15 END WHILE

LZW摘要：该算法非常好地压缩重复的数据序列。 由于码字是12位，因此任何单个编码字符都将扩展数据大小而不是减小它。

LZW优于霍夫曼的优势：

  • LZW不需要有关输入数据流的先验信息。
  • LZW可以一次压缩输入流。
  • LZW的另一个优点是它的简单性，允许快速执行。
