应用层中的协议

应用层： -

应用程序层是最重要的，因为这是OSI模型顶部的唯一层，用户可以通过它进行交互。 它为用户提供服务。

应用层协议： -

1. TELNET：

Telnet代表电话网络。它有助于终端仿真。它允许Telnet客户端访问Telnet服务器的资源。它用于管理Internet上的文件。它用于设备的初始设置，如交换机.Telnet命令是使用Telnet协议与远程设备或系统通信的命令。
命令 

 telnet [\\RemoteServer]
\\RemoteServer : Specifies the name of the server to which you want to connect

2. FTP：

FTP代表文件传输协议。 它实际上是让我们传输文件的协议。它可以在使用它的任何两台机器之间实现这一点。 但FTP不仅仅是一个协议，它也是一个程序.FTP通过远程计算机促进文件共享，提供可靠，高效的数据传输

命令 

 ftp machinename

3. TFTP：

简单文件传输协议（TFTP）是FTP的精简版库存版本，但如果您确切知道自己想要什么以及在何处找到它，它就是首选协议。 它是一种在网络设备之间传输文件的技术，是FTP的简化版本



命令 

 tftp [ options... ] [host [port]] [-c command]

4. NFS：

它代表网络文件系统。它允许远程主机通过网络挂载文件系统，并与这些文件系统进行交互，就好像它们是本地安装的一样。 这使系统管理员能够将资源整合到网络上的集中式服务器上。

命令 

 service nfs start

5. SMTP：

它代表简单邮件传输协议。它是TCP / IP协议的一部分。使用称为“存储转发”的过程，SMTP会在网络上和网络之间移动您的电子邮件。 它与称为邮件传输代理（MTA）的东西密切合作，将您的通信发送到正确的计算机和电子邮件收件箱。

命令 

 MAIL FROM:<mail@abc.com?

6. LPD：

它代表行式打印机守护进程。它专为打印机共享而设计。它是接收和处理请求的部分。 “守护程序”是服务器或代理程序。

命令 

 lpd [ -d ] [ -l ] [ -D DebugOutputFile]

7. X窗口：

它定义了用于编写基于图形用户界面的客户端/服务器应用程序的协议。 这个想法是允许一个称为客户端的程序在一台计算机上运行。 它主要用于互连大型机的网络。

命令 

 Run xdm in runlevel 5

8. SNMP：

它代表简单网络管理协议。它通过轮询设备来收集数据
来自管理站的网络以固定或随机的间隔，要求
他们披露某些信息。这是服务器可以共享有关其当前状态的信息的方式，也是管理员可以通过其修改预定义值的渠道。
命令 

 snmpget -mALL -v1 -cpublic snmp_agent_Ip_address sysName.0

9. DNS：

它代表域名服务。因此，每次使用域名时，DNS服务都必须将名称转换为相应的IP地址。 例如，域名www.abc.com可能会转换为198.105.232.4。

命令 

 ipconfig /flushdns

10. DHCP：

它代表动态主机配置协议（DHCP）。它为主机提供IP地址。当主机向DHCP服务器注册IP地址时，DHCP服务器可以为主机提供大量信息。

命令 

 clear ip dhcp binding {address | * }
