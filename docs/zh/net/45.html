令牌环效率

令牌环协议是局域网（LAN）中使用的通信协议。 在令牌环协议中，网络的拓扑用于定义站发送的顺序。 这些站在一个环中相互连接。 它使用一个特殊的三字节帧，称为“令牌” ，围绕一个环行进。 它利用Token Passing控制访问机制。 帧也在令牌的方向上传输。 这样，它们将在环上循环并到达作为目的地的站。

戒指延迟 -
单个位围绕环行进所花费的时间称为环延迟。

哪里，
d =环的长度
v =环中的数据速度
N =不。 环形站
b =每个站在发送之前保持该位所用的时间（比特延迟）



将N * b转换为秒 - 

 RL = d/v + (N*b)/B (B – bandwidth)

将d / v转换为位 - 

 RL = (d/v)*B + N*b (B – bandwidth)

周期 -
令牌完成一圈旋转所花费的时间称为循环时间。 

 Cycle time = T p + (THT*N)
Where, THT - Token Holding Time
T p - Propagation delay(d/v)

令牌持有时间（THT） -
站点可以保持令牌帧的最大时间称为THT，默认情况下设置为10毫秒。 没有电台可以将令牌保持在THT之外。

计算THT：

1.延迟令牌重新插入（DTR） -

  • 在这种情况下，发送方发送数据包并等待整个数据包进行环的往返并返回它的时间。 当发件人收到整个数据包时，它会释放令牌
  • 一个实例的环中只有一个数据包
  • 比ETR更可靠

在这种情况下， 

 THT = T t + RL
    = T t + T p + N*b (In most cases, bit delay is 0) 
So, THT = T t + T p
 where T t = transmission delay
       T p = propagation delay

2.早期令牌重新插入（ETR） -

  • 发送者在释放令牌之前不等待数据包完成旋转。 一旦数据传输，令牌就会被释放
  • 环中存在多个数据包
  • 比DTR更不可靠

站1：接收令牌并发送数据D1然后释放令牌。
站2：接收D1（将其置于另一端）和令牌然后，发送数据D2并释放令牌。
站3：接收D1 - >发送D1
接收D2 - >发送D2
接收令牌 - >传输D3
释放令牌。

站4：接收D1 - >发送D1
接收D2 - >发送D2
接收D3 - >传输D3
接收令牌 - >传输D4
释放令牌。

站1：接收D1 - > D1完成其旅程后丢弃D1
接收D2 - >发送D2
接收D3 - >传输D3
接收D4 - >发送D4
接收令牌 - >传输D1（新）
释放令牌。
（循环继续......）

在这种情况下， 

 THT = T t
 where T t = transmission delay
       T p = propagation delay

效率 -
效率，e =有用时间/总时间

有用时间= N * T t
总时间=周期时间= T p +（THT * N）

所以，e =（N * T t ）/（T p +（THT * N））

1.延迟令牌重新插入 -
在这种情况下，THT = T t + T p
所以，周期时间= T p + N *（T t + T p ） 

 Efficiency, e = (N*T t )/(T p + N*(T t + T p )) 
                = 1/(1 + a*((N+1)/N)) 
 where a = T p /T t

2.早期令牌重新插入 -
在这种情况下，THT = T t
所以，周期时间= T p + N *（T t ） 

 Efficiency, e = (N*T t )/(T p + N*(T t )) 
                = 1/(1 + a*(1/N)) 
 where a = T p /T t
