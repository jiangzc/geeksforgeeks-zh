TCP / IP模型

先决条件 - OSI模型层

我们刚看到的OSI模型只是一个参考/逻辑模型。 它旨在通过将通信过程划分为更小和更简单的组件来描述通信系统的功能。 但是当我们谈论TCP / IP模型时，它是由国防部（DoD）在20世纪60年代设计和开发的，并且基于标准协议。 它代表传输控制协议/ Internet协议。 TCP / IP模型是OSI模型的简明版本。 它包含四层，与OSI模型中的七层不同。 这些图层是：

  1. 流程/应用层
  2. 主机到主机/传输层
  3. 互联网层
  4. 网络访问/链路层

第一层是代表发送方的Process层和代表接收方的Network Access层。 在本文中，我们将代表接收者进行讨论。



1.网络访问层 -

该层对应于OSI模型的数据链路层和物理层的组合。 它寻找硬件寻址，并且该层中存在的协议允许数据的物理传输。
我们刚才谈到ARP是Internet层的协议，但是将其声明为Internet层或网络访问层的协议存在冲突。 它被描述为驻留在第3层中，由第2层协议封装。

2.互联网层 -

该层与OSI网络层的功能相似。 它定义了负责整个网络上数据逻辑传输的协议。 驻留在该层的主要协议是：

  1. IP -代表Internet协议，它负责通过查看数据包标头中的IP地址将数据包从源主机传送到目标主机。 IP有2个版本：
    IPv4和IPv6。 IPv4是大多数网站目前使用的。 但随着与用户数量相比，IPv4地址的数量有限，IPv6正在增长。
  2. ICMP -代表Internet控制消息协议。 它封装在IP数据报中，负责向主机提供有关网络问题的信息。
  3. ARP -代表地址解析协议。 它的工作是从已知的IP地址中查找主机的硬件地址。 ARP有几种类型：反向ARP，代理ARP，Gratituous ARP和反向ARP。

3.主机到主机层 -

该层类似于OSI模型的传输层。 它负责端到端通信和无差错的数据传输。 它可以保护上层应用程序免受数据复杂性的影响。 该层中存在的两个主要协议是：

  1. 传输控制协议（TCP） -已知在终端系统之间提供可靠且无差错的通信。 它执行数据的排序和分段。 它还具有确认功能，并通过流量控制机制控制数据流。 这是一个非常有效的协议，但由于这些功能而有很多开销。 增加的开销导致成本增加。
  2. 用户数据报协议（UDP） -另一方面不提供任何此类功能。 如果您的应用程序不需要可靠的传输，那么它就是协议，因为它非常划算。 与TCP（面向连接的协议）不同，UDP是无连接的。

      3. 4.流程层 -

      该层执行OSI模型的前三层功能：应用程序，表示和会话层。 它负责节点到节点的通信并控制用户界面规范。 此层中存在的一些协议包括：HTTP，HTTPS，FTP，TFTP，Telnet，SSH，SMTP，SNMP，NTP，DNS，DHCP，NFS，X Window，LPD。 有关这些协议的一些信息，请查看应用层中的协议。 除链接文章中的协议以外的协议是：

      1. HTTP和HTTPS - HTTP代表超文本传输​​协议。 万维网使用它来管理Web浏览器和服务器之间的通信。 HTTPS代表HTTP-Secure。 它是HTTP与SSL（安全套接字层）的组合。 在浏览器需要填写表格，登录，验证和执行银行交易的情况下，它是高效的。
      2. SSH - SSH代表Secure Shell。 它是一个类似于Telnet的终端仿真软件。 SSH更受欢迎的原因是因为它能够保持加密连接。 它通过TCP / IP连接建立安全会话。
      3. NTP - NTP代表网络时间协议。 它用于将计算机上的时钟与一个标准时间源同步。 它在银行交易等情况下非常有用。 在没有NTP的情况下假设以下情况。 假设您执行了一项交易，您的计算机将在下午2:30的时间读取时间，而服务器会在下午2:28将其记录下来。 如果服务器不同步，服务器可能会崩溃。
