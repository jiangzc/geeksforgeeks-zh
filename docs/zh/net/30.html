受控访问协议

在受控访问中，站点彼此寻找信息以找出哪个站点有权发送。 它一次只允许一个节点发送，以避免共享介质上的消息冲突。
三种受控访问方法是：

  1. 保留
  2. 轮询
  3. 令牌传递

保留

  • 在预约方法中，电台需要在发送数据之前进行预约。
  • 时间线有两种时期：
    1. 保留间隔固定时间长度
    2. 可变帧的数据传输周期。
  • 如果有M个站，则预留间隔被分成M个时隙，每个站有一个时隙。
  • 假设如果站1具有要发送的帧，则它在时隙1期间发送1比特。在该时隙期间不允许其他站发送。
  • 通常， 第 i站可以通过在第 i 个时隙中插入1比特来宣告它具有要发送的帧。 在检查了所有N个时隙之后，每个站知道哪些站希望发送。
  • 保留其插槽的站按该顺序传输其帧。
  • 在数据传输周期之后，下一个预留间隔开始。
  • 由于每个人都同意谁接下来，所以永远不会有任何碰撞。

轮询

  • 轮询过程类似于课堂上的滚动呼叫。 就像老师一样，控制器依次向每个节点发送消息。
  • 在此，一个充当主站（控制器），其他充当二级站。 必须通过控制器进行所有数据交换。
  • 控制器发送的消息包含被选择用于授予访问权限的节点的地址。
  • 虽然所有节点都接收到该消息，但是被寻址的节点响应它并发送数据（如果有的话）。 如果没有数据，通常会发回“轮询拒绝”（NAK）消息。
  • 问题包括轮询消息的高开销和对控制器可靠性的高度依赖性。






效率
设T poll为轮询时间，T t为传输数据所需的时间。 然后， 

 Efficiency = T t /(T t + T poll )

令牌传递

  • 在令牌传递方案中，站以环的形式逻辑地彼此连接，并且站的访问由令牌控制。
  • 令牌是特殊的位模式或小消息，它以某个预定义的顺序从一个站循环到下一个站。
  • 在令牌环中，令牌从一个站传递到环中的另一个相邻站，而令牌总线，每个站
    使用总线以某种预定义的顺序将令牌发送到下一个站。
  • 在这两种情况下，token表示发送权限。 如果一个站在接收到令牌时有一个排队等待传输的帧，它可以在将该帧传递给下一个站之前发送该帧。 如果它没有排队的帧，它只是简单地传递令牌。
  • 在发送帧之后，每个站必须等待所有N个站（包括其自身）将令牌发送到它们的邻居和其他N-1个站以发送帧（如果它们有一个帧）。
  • 存在诸如令牌或令牌的重复丢失或新站的插入，移除站的问题，需要解决该方案的正确和可靠操作。

性能
令牌环的性能可以通过2个参数得出： -

  1. 延迟 ，它是数据包准备就绪和传送时间之间的时间尺度。因此，将令牌发送到下一个站所需的平均时间（延迟）= a / N.
  2. 吞吐量 ，衡量成功的流量。 
 Throughput, S = 1/(1 + a/N) for a<1

和 

 S = 1/{a(1 + 1/N)} for a>1.  
       where N = number of stations
             a = T p /T t 
(T p = propagation delay and T t = transmission delay)
