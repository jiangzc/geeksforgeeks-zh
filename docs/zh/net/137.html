我们来试试网络

我们大多数人都以非常抽象的方式研究过计算机网络。 换句话说，我们中很多人都不知道层和包的抽象概念如何在现实生活网络（如Internet）中进行转换。 因此，让我们做一个实验，看看这些层，包等是否也存在于任何真实的网络中。 所以，准备好并准备好钻研这个实用和实验网络的精彩世界！

我们的实验概要如下。 我们将捕获一些实时数据包，并了解这些数据包中的内容，我们将通过解析它们来分析这些数据包。 听起来外科手术？ 是的，是的。 Ĵ

首先，我们需要一台运行Windows XP并连接到Internet的PC。 如果您在网上阅读这篇文章，那么您可以准备好进行实验的可能性很高。 现在让我们回顾一下我们在网络书籍中读到的一些理论内容。 几乎每本书都告诉我们的第一件事 - 网络架构是分层的; 记住7层OSI协议栈！ 那么这些协议层在哪里？ 在我们的实验中，我们将使用5层Internet协议栈，以便我们可以解决这些层的神秘面纱。

我们通过安装Wireshark（以前称为Ethereal）开始我们的实验。 Wireshark是一种网络协议分析器，可以捕获和分析通过网络接口卡（NIC）发送/接收的数据包。 [你需要忍受我的这个缩写，因为网络充满了缩写词]我们从http://www.wireshark.org/download.html安装Wireshark（在撰写本文时，最新的Wireshark版本是1.0.3） 。 安装Wireshark时，请保留默认设置/选项。 现在我们的实验设置准备好了。 运行Wireshark并单击第一个图标（列出可用的捕获接口...）。 现在我们看到一个弹出窗口，显示捕获接口。 请参阅快照，如下所示。

根据PC的配置，“捕获接口”窗口中​​显示的接口数量和类型可能会有所不同。 对我而言，它显示了两个接口，我的Internet连接是通过Broadcom千兆接口。 因此，请选择您可以通过其连接Internet的界面。 现在让我们点击这个界面的O ptions按钮。 现在我们看到一个名为Capture Options的新窗口。 在此窗口中，在名为Capture Filter的文本字段中键入“端口80”。 请参阅以下快照以获取说明。



现在我们准备捕获通过NIC的数据包了。 通过将过滤器设置为“端口80”，我们已指示Wireshark仅捕获因http流量而导致的数据包（请记住，我们始终被告知默认的http端口为80！）。 现在单击“捕获选项”窗口上的“开始”按钮。 如果您的PC中的任何程序正在后台访问http流量，您可能会在Wireshark中看到一些数据包; 我们不关注那个。 现在打开浏览器并尝试访问http://google.com ，现在您应该会看到许多数据包在Wireshark中被捕获。 请参阅快照，如下所示。

让我们开始分析捕获的数据包。 首先，在Info字段中找到第一个具有GET / HTTP / 1.1的http数据包实例。 在上面的快照中，它以蓝色显示。 如果我们仔细观察一下，我们会发现这个数据包具有Internet协议栈所有5层的标头。

第1层 - 它是物理层。 这里框架显示在物理层。

第2层 - 它是数据链路层。 在这个数据包中，我们可以看到Ethernet II被用作数据链路层协议。 我们可以在此标头中找到源和目标的MAC地址。

第3层 - 它是网络层。 在此数据包中，我们看到IP用作网络层协议。 我们可以在此标头中看到源IP和目标IP。

第4层 - 它是传输层。 在该分组中，TCP用作传输层协议。 我们可以在此标头中找到源端口和目标端口。

第5层 - 它是应用层。 在此数据包中，HTTP用作应用层协议。

让我们探索其中一个层。 可以以类似的方式进一步探索其他层。 如果我们扩展第5层即HTTP标头，它看起来如下。

在这里，我们看到主机被称为google.com，这是我们尝试从浏览器访问的内容。 HTTP标头的User Agent字段显示浏览器详细信息。 在我的例子中，从这个标题中可以看出它是Mozilla Firefox。 目标IP 64.233.187.99应该是分配给托管Web服务器的Google服务器的IP地址之一。 它可以使用非常方便的实用程序命令“nslookup”进行验证。 可以在HTTP，TCP，IP，以太网II协议的标题中探索其他字段的详细信息。 一些有趣的领域是 - IP头中的区分服务字段（也称为QoS字段），TCP头中的窗口大小等。

所以我们已经看到所有那些层次等的修辞概念确实存在于真实网络中。 当您剖析通过接口卡的所有数据包时，这听起来很有趣。 通过这样做，您可以了解PC上的内容/来源！
