以太通道

EtherChannel是一种端口链路聚合技术，其中多个物理端口链路分组为一个逻辑链路。 它用于提供高速链路和冗余。 最多可以聚合8个链接以形成单个逻辑链接。

需要EtherChannel -

这是一种拓扑结构，其中两个交换机分别与一台PC连接。 交换机与PC之间的链路为1000mb / s，交换机之间的链路为100mb / s。

现在，假设您想要发送超过100mb / s的流量，那么我们就会出现拥塞，因为交换机之间的链路仅为100mb / s，数据包将开始丢弃。 现在，要解决这个问题，我们应该在交换机之间建立高速链路。 为了实现这一点，我们可以简单地用高速链路替换当前链路，或者我们可以捆绑多个速率为100mb / s的链路。 通过形成EtherChannel，您可以将多个链接捆绑到一个逻辑链接中。



但是，当您使用多个链路连接交换机时，STP（生成树协议）将阻止冗余链路最少。 由于我们已经建立了一个EtherChannel，所有链路（被分组为一个逻辑链路k）将被视为单个逻辑链路，因此不会阻止任何链路，并且它将为我们的网络提供高速链路和冗余。

标准 -要形成EtherChannel，所有端口都应具有：

  1. 相同的双工
  2. 速度相同
  3. 相同的VLAN配置（即本机VLAN和允许的VLAN应该相同）
  4. 交换机端口模式应相同（访问或中继模式）

EtherChannel协议 -为了形成EtherChannel，有2个协议，端口聚合协议（PAgP）和链路聚合控制协议（LACP）。

1.港口聚合协议（PAgP） -
端口聚合协议是Cisco专有协议，用于形成EtherChannel。 您可以使用不同的模式配置界面。 这些是：

  1. ON：在此模式下，接口将是EtherChannel的一部分，但不进行协商。
  2. 理想的：在此模式下，接口将不断尝试将其他侧接口转换为EtherChannel。
  3. 自动：在此模式下，当相反的接口请求接口时，接口将成为EtherChannel的一部分。
  4. 熄灭：接口上未配置EtherChannel。

配置 -

有一个小型拓扑，其中2个交换机S1和S2相互连接，我们必须将这两个链路捆绑成一个逻辑链路。 

 S1(config)# interface fa0/1
S1(config-if)# channel-group 1 mode desirable 
S1(config)# interface fa0/2
S1(config-if)# channel-group 1 mode desirable 

S1(config)# interface port-channel 1
S1(config-if)# switchport trunk encapsulation dot1q
S1(config-if)# switchport mode trunk

这里，用户使用了理想模式和switchport模式trunk。 两个开关上的模式应该相同，因此用户也可以在其他开关上进行配置。
现在，在交换机S2上配置： 

 S2(config)# interface fa0/1
S2(config-if)# channel-group 1 mode desirable 
S2(config)# interface fa0/1
S2(config-if)# channel-group 1 mode desirable 
S2(config)# interface port-channel 1
S2(config-if)# switchport trunk encapsulation dot1q
S2(config-if)# switchport mode trunk

2.链路聚合控制协议（LACP） -
链路聚合控制协议是一种IEEE协议，最初在802.3ad中定义，用于形成EtherChannel。 该协议与Cisco PAgP几乎相似。 您可以使用不同的模式配置界面。 这些是：

  1. ON：在此模式下，接口将是EtherChannel的一部分，但不进行协商
  2. 活动：在此模式下，接口将不断尝试将其他侧接口转换为EtherChannel。
  3. 被动：在此模式下，当相反的接口请求接口时，接口将成为EtherChannel的一部分。
  4. 熄灭：接口上未配置EtherChannel。

配置 -

采用相同的拓扑，您现在将在两个交换机上配置LACP。 首先，配置S1： 

 S1(config)# interface fa0/1
S1(config-if)# channel-group mode active 
S1(config)# interface fa0/2
S1(config-if)# channel-group mode active

S1(config)# interface port-channel 1
S1(config-if)# switchport trunk encapsulation dot1q
S1(config-if)# switchport mode trunk

现在，配置S2： 

 S2(config)# interface fa0/1
S2(config-if)# channel-group mode active
S2(config)# interface fa0/2
S2(config-if)# channel-group mode active 

S2(config)# interface port-channel 1
S2(config-if)# switchport trunk encapsulation dot1q
S2(config-if)# switchport mode trunk
