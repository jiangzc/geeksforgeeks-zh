路由信息协议（RIP）

路由信息协议 （RIP）是一种动态路由协议，它使用跳数作为路由度量来查找源和目标网络之间的最佳路径。 它是距离矢量路由协议，其AD值为120，适用于OSI模型的应用层。 RIP使用端口号520。

跳数：
跳数是源网络和目标网络之间发生的路由器数量。 具有最低跳数的路径被认为是到达网络的最佳路由，因此被放置在路由表中。 RIP通过限制源和目标路径中允许的希望数来防止路由循环。 RIP允许的最大跳数为15，跳数为16被认为是网络不可达。

RIP的特点：

1.定期交换网络更新。
2.始终广播更新（路由信息）。
3.更新中发送完整的路由表。
4.路由器始终信任从邻居路由器接收的路由信息​​。 这也被称为谣言路由 。

RIP版本：
路由信息协议有三种 - RIP版本1 ， RIP版本2和RIPng 。



RIP v1 RIP v2 RIPng协议
发送更新为广播 将更新作为多播发送 将更新作为多播发送
广播时间为255.255.255.255 组播在224.0.0.9 FF02 :: 9的组播（RIPng只能在IPv6网络上运行）
不支持更新消息的身份验证 支持RIPv2更新消息的认证 -
有类路由协议 无类协议，支持有类 发送无类更新

RIP v1称为有类路由协议，因为它不在其路由更新中发送掩码信息。
RIP v2称为无类路由协议，因为它不会在其路由更新中发送掩码信息。

>>使用debug命令获取详细信息： 

 # debug ip rip

>>使用此命令显示路由器中配置的所有路由，例如路由器R1： 

 R1# show ip route

>>使用此命令显示路由器中配置的所有协议，例如路由器R1：

配置：

考虑上面给出的具有3路由器R1，R2，R3的拓扑。 R1在s0 / 0/1上具有IP地址172.16.10.6/30，在fa0 / 0上具有192.168.20.1/24。 R2在s0 / 0/0上具有IP地址172.16.10.2/30，在fa0 / 0上具有192.168.10.1/24。 R3在s0 / 1上的IP地址为172.16.10.5/30，在s0 / 0上为172.16.10.1/30，在fa0 / 0上为10.10.10.1/24。

为R1配置RIP： 

 R1(config)# router rip
R1(config-router)# network 192.168.20.0
R1(config-router)# network 172.16.10.4
R1(config-router)# version 2
R1(config-router)# no auto-summary

注意： no auto-summary命令禁用自动no auto-summary 。 如果我们不选择无自动摘要，则子版掩码在版本1中将被视为有类别。

为R2配置RIP： 

 R2(config)# router rip
R2(config-router)# network 192.168.10.0
R2(config-router)# network 172.16.10.0
R2(config-router)# version 2
R2(config-router)# no auto-summary

同样，为R3配置RIP： 

 R3(config)# router rip
R3(config-router)# network 10.10.10.0
R3(config-router)# network 172.16.10.4
R3(config-router)# network 172.16.10.0
R3(config-router)# version 2
R3(config-router)# no auto-summary

RIP定时器：

  • 更新定时器：运行RIP的路由器交换的路由信息​​的默认时间是30秒。 使用Update定时器，路由器定期交换路由表。
  • 无效的计时器：如果在180秒之前没有更新，则目标路由器将其视为无效。 在这种情况下，目标路由器将该路由器的跳数标记为16。
  • 按住定时器：这是路由器等待邻居路由器响应的时间。 如果路由器无法在给定时间内响应，则声明它已死亡。 默认为180秒。
  • 冲洗时间：如果在冲洗时间内没有响应，则刷新路径的时间。 默认为60秒。 此路由器在声明路线无效后开始，60秒后即开始，即时间为180 + 60 = 240秒。

请注意，所有这些时间都是可调节的 使用此命令更改计时器： 

 R1(config-router)# timers basic
R1(config-router)# timers basic 20 80 80 90
