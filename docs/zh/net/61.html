路由协议集1（距离矢量路由）

距离矢量路由（DVR）协议要求路由器定期通知其邻居拓扑变化。 历史上称为旧的ARPANET路由算法（或称为Bellman-Ford算法）。

Bellman Ford Basics -每个路由器都维护一个距离向量表，其中包含自身与所有可能目标节点之间的距离。 基于所选度量的距离是使用来自邻居的距离向量的信息来计算的。

  •  Distance to itself = 0

距离矢量算法 -

  1. 路由器将其距离矢量发送到路由分组中的每个邻居。
  2. 每个路由器从其每个邻居接收并保存最近接收的距离矢量。
  3. 路由器在以下情况下重新计算其距离向量：
    • 它从包含与以前不同的信息的邻居接收距离矢量。
    • 它发现与邻居的链接已经关闭。

DV计算基于最小化每个目的地的成本 

 Dx(y) = Estimate of least cost from x to y 
C(x,v) = Node x knows cost to each neighbor v
Dx = [Dx(y): y ∈ N ] = Node x maintains distance vector
Node x also maintains its neighbors' distance vectors
– For each neighbor v, x maintains Dv = [Dv(y): y ∈ N ]

注意 -



  • 每个节点不时地向邻居发送其自己的距离矢量估计。
  • 当节点x从任何邻居v接收到新的DV估计时，它会保存v的距离向量，并使用BF方程更新自己的DV： 
     Dx(y) = min { C(x,v) + Dv(y)} for each node y ∈ N

距离矢量路由的优点 -

  • 配置和维护比链路状态路由更简单。

    • 距离矢量路由的缺点 -

    • 收敛比链路状态慢。
    • 从计数到无穷大的问题存在风险。
    • 它创建的流量多于链路状态，因为跳数更改必须传播到所有路由器并在每个路由器上处理。 即使网络拓扑没有变化，也会定期进行跳数更新，因此仍会出现带宽浪费的广播。
    • 对于较大的网络，距离矢量路由导致比链路状态更大的路由表，因为每个路由器必须知道所有其他路由器。 这也可能导致WAN链路拥塞。

    注 -距离矢量路由使用UDP（用户数据报协议）进行传输。
