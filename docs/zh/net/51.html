网络层| IPv4数据报碎片和延迟

为什么需要IPv4数据报碎片？
不同的网络可能具有不同的最大传输单元（MTU），例如由于LAN技术的差异。 当一个网络想要将数据报传输到具有较小MTU的网络时，路径上的路由器可以对数据报进行分段和重组。

碎片是如何完成的？
当在路由器处接收到分组时，检查目的地地址并确定MTU。 如果数据包的大小大于MTU，并且标头中的“Do not Fragment（DF）”位设置为0，则数据包将被分段为多个部分并逐个发送。 每个片段的最大大小是MTU减去IP头大小（最小20个字节和最大60个字节）。

每个片段都转换为数据包，并在数据报头中发生以下更改：

  1. 总长度字段更改为片段的大小。
  2. 除最后一个之外的所有分段分组都设置了更多分段比特（MF比特）。
  3. 基于正在设置的片段的数量和MTU来设置片段偏移字段。
  4. 重新计算标头校验和。

示例：对于4000字节的数据包和1500字节的MTU，我们有3980字节的实际数据要传输，1480字节是允许发送的最大数据大小。 那么，会有3个片段：
对于第一个片段，数据大小= 1480字节，偏移= 0，MF标志= 1
对于第二个片段，数据大小= 1480字节，偏移= 1480，MF标志= 1
对于第三个片段，数据大小= 1020字节，偏移= 2960，MF标志= 0

这里要注意的一点是，所有片段都具有相同的标识号，从而表明所有片段都属于同一个父数据包。





延迟 -
处理延迟：路由器处理数据包头所花费的时间。

排队延迟：路由队列中数据包所用的时间。

传输延迟：将数据包加载到传输通道所需的时间
D t = N / R，
N：要传输的位数
R：信道的速率或传输速度

传播延迟 - 数据包从源到达目的地所花费的时间

D p = D / S，
D：源和目的地之间的距离
S：是传播速度
