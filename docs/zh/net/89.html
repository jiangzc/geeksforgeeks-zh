不同网络中的数据包流

前提条件 - ARP如何工作 ， 数据包在同一网络中流动

要将数据包传送到目标主机，应知道源IP，目标IP，源MAC地址和目标MAC地址。 数据包流的一些基本规则：

  1. 如果目标主机位于同一网络中，则数据包将直接传送到目标主机。
  2. 如果目标主机存在于不同的网络中，则首先将数据包传送到默认网关，然后将该数据包传送到目标主机。
  3. 如果ARP未解决，则首先解析ARP。
  4. MAC地址永远不会跨越其广播域。

说明 -

这是一个拓扑，其中有主机A（IP地址 - 10.0.0.10和MAC地址 - 000D.BD22.7C22），主机C（IP地址 - 10.0.0.9），主机B（IP地址 - 20.0.0.10） ，主机C（IP地址-20.0.0.9和MAC地址 - 00E0.A3E2.03DC）和路由器（IP地址 - 10.0.0.20和MAC地址 - 000B.BE8E.5201 on fa0 / 0，IP地址 - 20.0.0.20和MAC地址 - 在fa0 / 1上的000B.BE8E.5202）。



现在我们将尝试从主机A（IP地址 - 10.0.0.10）ping到主机B（IP地址 - 20.0.0.10）。 首先，源主机在源IP地址，源子网掩码和目的IP地址，源子网掩码之间执行AND操作 ，以了解目的地是否存在于相同或不同的网络中。

如果结果相同则目标在同一网络中，否则在不同网络中。 这里，目的地存在于不同的网络中，因此结果将不同，并且数据包将被传送到默认网关。

我们看到2条消息生成ICMP（紫色）和ARP（绿色）。 ARP已生成，因为ARP尚未解决。

现在必须再次解析ARP，因为路由器必须将数据包传送到主机B，并且ARP表没有主机B的条目。因此，ARP请求在网络20.0.0.0/24中广播。 数据包由交换机接收，交换机又将请求广播给主机B和D.主机D将拒绝该请求，主机B将接受该请求，并为MAC地址000B.BE8E.5202生成ARP应答（路由器fa0 / 1 MAC地址）因为必须将ARP回复提供给已收到ARP请求的MAC地址。

现在，ICMP回应请求包将单播到主机B，如上面的3个图所示。

主机B将响应主机A的ICMP回应请求生成ICMP回应应答，主机A将首先传送到20.0.0.20（路由器的接口IP地址），然后单播到主机A.

MAC地址如何永远不会跨越其广播域？

这是主机A将ICMP回应请求转发到其默认网关时的IP和以太网报头。因此，源IP为10.0.0.10，目标IP为10.0.0.20，源MAC地址为000D.BD22.7C22（主机A MAC地址）目的MAC地址为000B.BE8E.5201（路由器的fa0 / 0接口MAC地址）。

但是现在当ICMP回送请求消息从路由器的fa0 / 1接口转发到主机B时，源MAC地址变为000B.BE8E.5202（路由器的fa0 / 1接口MAC地址），目的MAC地址为00E0.A3E2 .03DC（主机B MAC地址）。

这里路由器的fa0 / 0接口MAC地址不用作源MAC地址，而是将fa0 / 1 MAC地址用作MAC地址。 因此，fa0 / 0不用于其他广播域（20.0.0.0/24网络），因此MAC地址永远不会跨越其广播域。 以这种方式，PING在2个不同的网络中执行。
