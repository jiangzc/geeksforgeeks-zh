文件传输协议（FTP）

先决条件： 应用层中的协议

文件传输协议（FTP）是一种应用程序层协议，可在本地和远程文件系统之间移动文件。 它运行在TCP的顶部，如HTTP。 要传输文件，FTP并行使用2个TCP连接：控制连接和数据连接。

什么是控制连接？
为了发送控制信息，如用户标识，密码，更改远程目录的命令，检索和存储文件等命令，FTP利用控制连接。 控制连接在端口号21上启动。

什么是数据连接？
为了发送实际文件，FTP使用数据连接。 数据连接在端口号20上启动。
FTP在带外发送控制信息，因为它使用单独的控制连接。 某些协议在同一TCP连接中发送其请求和响应标题行和数据。 出于这个原因，据说他们在带内发送他们的控制信息。 HTTP和SMTP就是这样的例子。



FTP会话：
在客户端和服务器之间启动FTP会话时，客户端启动与服务器端的控制TCP连接。 客户端通过此发送控制信息。 当服务器收到此消息时，它会启动到客户端的数据连接。 只能通过一个数据连接发送一个文件。 但控制连接在整个用户会话期间保持活动状态。 我们知道HTTP是无状态的，即它不必跟踪任何用户状态。 但FTP需要在整个会话期间维护其用户的状态。

数据结构： FTP允许三种类型的数据结构：

  1. 文件结构 -在文件结构中没有内部结构，文件被认为是连续的数据字节序列。
  2. 记录结构 -在记录结构中，文件由顺序记录组成。
  3. 页面结构 -在页面结构中，文件由独立的索引页面组成。

FTP命令 -一些FTP命令是：

USER - 此命令将用户标识发送到服务器。
PASS - 此命令将用户密码发送到服务器。
CWD - 此命令允许用户使用不同的目录或数据集进行文件存储或检索，而无需更改其登录或记帐信息。
RMD - 此命令使路径名中指定的目录作为目录删除。
MKD - 此命令使路径名中指定的目录创建为目录。
PWD - 此命令导致在回复中返回当前工作目录的名称。
RETR - 此命令使远程主机启动数据连接并通过数据连接发送请求的文件。
STOR - 此命令导致将文件存储到远程主机的当前目录中。
LIST - 发送请求以显示目录中存在的所有文件的列表。
ABOR - 此命令告诉服务器中止先前的FTP服务命令以及任何相关的数据传输。
QUIT - 此命令终止USER，如果文件传输不在进行中，服务器将关闭控制连接。

FTP回复 -一些FTP回复是：

200命令没关系。
530尚未登录。
331用户名好，需要密码。
225数据连接打开; 没有转移进行中。
221服务关闭控制连接。
551请求的操作已中止：页面类型未知。
502命令未实现。
503错误的命令序列。
504未针对该参数实施命令。

匿名FTP：
在某些其文件可供公共访问的站点上启用匿名FTP。 用户无需任何用户名或密码即可访问这些文件。 相反，默认情况下，用户名设置为匿名，密码设置为来宾。 这里，用户访问非常有限。 例如，可以允许用户复制文件但不允许导航目录。
