计算机网络基础知识

开放系统：
连接到网络并准备好进行通信的系统。

封闭系统：
未连接到网络且无法与之通信的系统。

计算机网络：
它是多个设备的互连，通常称为使用多个路径连接的主机，用于发送/接收数据或媒体。
还有多个设备或介质有助于两个不同设备之间的通信，这两个设备称为网络设备 。 例如：路由器，交换机，集线器，网桥。

使用哪些设备互连的布局模式称为网络拓扑。 如Bus，Star，Mesh，Ring，Daisy chain。

OSI：
OSI代表开放系统互连 。 它是一个参考模型，它指定了通信协议的标准以及每个层的功能。



协议：
协议是一组规则或算法，它们定义了两个实体如何通过网络进行通信的方式，并且在OSI模型的每一层都存在不同的协议。 很少有这样的协议是TCP，IP，UDP，ARP，DHCP，FTP等。

独特的网络识别器
主机名：
网络中的每个设备都与称为Hostname的唯一设备名称相关联。
在命令提示符下键入“hostname”并按“Enter”，这将显示计算机的主机名。

IP地址（Internet协议地址）：
也称为逻辑地址，是整个网络中系统的网络地址。
为了识别万维网中的每个设备，Internet Assigned Numbers Authority（IANA）将IPV4（版本4）地址分配为Internet上每个设备的唯一标识符。
IP地址的长度为：32位。 （因此我们有2 32个 ip地址可用。）
在命令提示符下键入“ipconfig”并按“Enter”，这将为我们提供设备的IP地址。

MAC地址（媒体访问控制地址）：
也称为物理地址，是每个主机的唯一标识符，并与NIC（网络接口卡）相关联。
在制造时将MAC地址分配给NIC。
MAC地址的长度为：12位/ 6字节/ 48位
在命令提示符下键入“ipconfig / all”并按“Enter”，这将为我们提供MAC地址。

港口：
端口可以​​称为逻辑信道，通过该信道可以将数据发送/接收到应用程序。 任何主机都可能运行多个应用程序，并且每个应用程序都使用运行它们的端口号进行标识。
端口号是一个16位整数，因此我们有2 16个端口可用，分类如下：

端口类型 范围
众所周知的港口 0 - 1023
注册港口 1024 - 49151
短暂的港口 49152 - 65535

端口数量：65,536
范围：0 - 65535
在命令提示符下键入“ netstat -a ”并按“Enter”，这将列出正在使用的所有端口。

插座：
IP地址和端口号的唯一组合称为Socket。

几乎没有概念
DNS服务器：
DNS代表域名系统 。
DNS基本上是一个将Web地址或URL（例如：www.google.com）转换为相应IP地址的服务器。 我们不必记住每个网站的所有IP地址。
命令' nslookup '为您提供所需域名的IP地址。 这也提供了我们的DNS服务器的信息。

ARP：
ARP代表地址解析协议 。
它用于将IP地址转换为其对应的物理地址（即MAC地址）。
数据链路层使用ARP来识别接收方计算机的MAC地址。
RARP：
RARP代表反向地址解析协议 。
顾名思义，它提供给定物理地址作为输入的设备的IP地址。 但是自从DHCP出现以来，RARP已经过时了。
