虚拟电路和数据报网络之间的差异

提供面向连接的服务的计算机网络称为虚拟电路，而提供无连接服务的计算机网络称为数据报网络。 对于先前的知识，我们使用的因特网实际上是基于网络级的数据报网络（无连接），因为从源到目的地的所有分组都不遵循相同的路径。
让我们看看这两个激烈争论的主题之间的突出差异是什么：

虚拟电路 -

  1. 它是面向连接的，这意味着存在缓冲区，CPU，带宽等资源的预留。 对于新设置的VC将由数据传输会话使用的时间。
  2. 第一个数据包进入并为后续数据包保留资源，因此整个连接时间遵循相同的路径。
  3. 由于所有数据包都将遵循相同的路径，因此仅对连接的第一个数据包需要全局标头，而其他数据包通常不需要全局标头。
  4. 由于数据遵循特定的专用路径，因此数据包会到达目的地。
  5. 从以上几点可以得出结论，虚拟电路是高度可靠的传输手段。
  6. 由于每次必须在路由器上预留资源和额外信息处理来设置新连接时，实现虚拟电路的成本非常高。


数据报网络：

  1. 这是无连接服务。 由于没有用于连接会话的专用路径，因此不需要预留资源。
  2. 所有数据包都可以自由地转到任何中间路由器上的任何路径，该路由器通过动态更改路由器上的路由表来决定。
  3. 由于每个数据包都可以自由选择任何路径，因此所有数据包必须与具有关于源和上层数据的适当信息的报头相关联。
  4. 无连接属性使数据包以任何顺序到达目的地，这意味着它们无需按照发送顺序到达目的地。
  5. 数据报网络作为虚拟电路不可靠。
  6. 但是，实现数据报网络总是容易且成本有效的，因为每次应用程序必须进行通信时都不需要预留资源和专用。
