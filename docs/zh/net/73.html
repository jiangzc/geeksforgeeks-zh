开放最短路径优先（OSPF）协议基础

开放最短路径优先（OSPF）是链路状态路由协议 ，用于使用其自己的最短路径优先（SPF）算法在源路由器和目的地路由器之间找到最佳路径。 链路状态路由协议是使用触发更新的概念的协议，即，如果在学习的路由表中观察到变化，则仅触发更新，而不像路由表所在的距离矢量路由协议。在一段时间内交换。

开放最短路径优先（OSPF）由因特网工程任务组（IETF）开发，作为内部网关协议（IGP）之一，即旨在在大型自治系统或路由域内移动分组的协议。 它是一个网络层协议 ，在协议号89上工作，使用AD值110. OSPF使用组播地址224.0.0.5进行正常通信，224.0.0.6用于更新到指定路由器（DR）/备份指定路由器（BDR）。

标准 -
要在OSPF中形成邻居，两个路由器都有一个标准：

  1. 它应该存在于同一区域
  2. 路由器我必须是唯一的
  3. 子网掩码应该相同
  4. 你好和死时计应该是一样的
  5. 存根标志必须匹配
  6. 身份验证必须匹配

OSPF支持NULL，纯文本，MD5认证。

注 -两个路由器（邻居）都应该启用相同类型的身份验证。 例如，如果一个邻居启用了MD5身份验证，那么其他邻居也应启用MD5身份验证。



OSPF消息 -
OSPF使用某些消息进行运行OSPF的路由器之间的通信。

  • Hello消息 -这些是用于邻居发现/恢复的保持活动消息。 这些每10秒更换一次。 这包括以下信息：路由器I'，Hello / dead interval，Area I'，路由器优先级，DR和BDR IP地址，认证数据。
  • 数据库描述（DBD） -路由器的OSPF路由。 这包含AS或区域（路由域）的拓扑。
  • 链路状态请求（LSR） -当路由器接收DBD时，它将其与自己的DBD进行比较。 如果收到的DBD比其自己的DBD有更多的更新，则LSR将被发送到其邻居。
  • 链路状态更新（LSU） -当路由器收到LSR时，它会以包含所请求详细信息的LSU消息进行响应。
  • 链路状态确认 -这为链路状态交换过程提供了可靠性。 它作为LSU的确认发送。
  • 链路状态通告（LSA） -它是一个OSPF数据包，包含链路状态路由信息，仅与已形成邻接的路由器共享。

注 -链接状态公告和链接状态确认都是不同的消息。

计时器 -

  • Hello timer - OSPF路由器在接口上发送hello消息的时间间隔。 默认为10秒。
  • 死定时器 -如果无法发送hello数据包，则邻居将被宣告死亡的时间间隔。 默认为40秒。通常是问候间隔的4倍，但可以根据需要手动配置。

OSPF支持/提供/优势 -

  • IPv4和IPv6路由协议
  • 使用相同目的地的等成本路由进行负载均衡
  • VLSM和路由汇总
  • 无限跳数
  • 触发更新以实现快速收敛
  • 使用SPF算法的无环路拓扑
  • 在大多数路由器上运行
  • 无类协议

OSPF有一些缺点，它需要额外的CPU进程来运行SPF算法，需要更多的RAM来存储邻接拓扑，设置更复杂，难以排除故障。
