访问和中继端口

交换机端口是第2层接口，用于承载第2层流量。 无论是接入端口还是中继端口，单个交换机端口都可以承载单个VLAN流量。 帧根据它们遍历的链接类型进行不同的处理。

注意：默认情况下，所有交换机端口都分配了VLAN 1（无法修改或删除VLAN 1）。

交换环境中有两种不同类型的端口：

访问端口 -
此交换机端口属于仅承载一个vlan的流量。 默认情况下，它将承载本地VLAN（VLAN 1）的流量。如果交换机端口被指定为访问端口，则可以将其视为交换机端口属于单个广播域。 到达这些交换机端口的任何流量都被视为属于分配给该端口的VLAN。

这是一个简单的拓扑，其中连接了2个交换机，并且在两个交换机上只配置了默认VLAN（VLAN 1），即两个交换机的所有交换机端口都属于单个广播域。

现在，请注意，必须将交换机之间的链路配置为访问端口，因为只需交换单个VLAN（VLAN 1）数据。 现在，在为PC1-12.168.1.1 / 24，PC2-192.168.1.2 / 24，PC3-192.168.1.3 / 24，PC3-192.168.1.4 / 24分配IP地址后，用户应将2个交换机之间的链路配置为接入端口。 

 Switch1(config)#interface fa0/0
Switch1(config-if)#switchport mode access

这里不需要为端口分配VLAN，因为默认情况下，两台交换机上的所有交换机端口都配置为VLAN 1。

中继端口 -
这些交换机端口属于并承载多个VLAN的流量。 这对于承载VLAN组的流量来说是一个很大的优势，可以使用单个交换机端口。 如果用户想要在多个配置了多个vlan的交换机之间交换流量，则这些用途很有用。要识别属于哪个vlan的流量，使用VLAN识别方法（802.1q或ISL）。 此外，要在多个vlan之间传输流量，则需要进行VLAN间路由，其中​​路由器和交换机之间的链路配置为中继，因为链路必须承载多个VLAN的流量（如果路由器位于棒配置不在第3层交换机的inter vlan路由中）。

注意： Trunk链路可以承载不同VLAN的流量，但默认情况下，如果交换机之间的链路不是中继，则只交换来自配置的接入VLAN的信息。

现在，请注意，交换机之间的链路必须配置为中继端口，因为这里必须在交换机之间交换多个VLAN（VLAN 1,2,3）帧。 现在为PC1-12.168.1.1 / 24，PC2-192.168.2.1 / 24，PC3-192.168.1.2 / 24，PC3-192.168.2.2 / 24分配IP地址。

现在，第一个用户必须在两个交换机上建立VLAN。 

 Switch1(config)#vlan 2
Switch1(config)#vlan 3

Switch2(config)#vlan 2
Switch2(config)#vlan 3

现在，由于用户在两台交换机上配置了多个vlan。 因此，用户必须将VLAN分配到Switch1上各自的端口。 

 Switch1(config)#interface fa0/1
Switch1(config-if)#switchport access vlan 2
Switch1(config)#interface fa0/2
Switch1(config-if)#switchport access vlan 3

现在，在Switch2上各自的端口上配置VLAN。 

 Switch2(config-if)#interface fa0/1
Switch2(config-if)#switchport access vlan 2
Switch2(config)#interface fa0/2
Switch2(config-if)#switchport access vlan 3

现在，将2个交换机之间的链路配置为中继端口。 

 Switch1#interface fa0/0
Switch1#switchport trunk encapsulation dot1q
Switch1#switchport mode trunk

因此，现在用户可以将一个以上的VLAN流量从一个交换机传输到另一个交换机（这里只显示交换机端口的配置，而不显示路由器的配置。要执行inter vlan路由，路由器的配置也是需要）。
