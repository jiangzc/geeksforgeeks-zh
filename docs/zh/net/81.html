计算机网络| 热备用路由器协议（HSRP）

热备用路由器协议（HSRP）是CISCO专有协议，为本地子网提供冗余。 在HSRP中，两个或多个路由器给出了虚拟路由器的错觉。

HSRP允许您将两个或多个路由器配置为备用路由器，并且一次只配置一个路由器作为活动路由器。 单个HSRP组中的所有路由器共享单个MAC地址和IP地址，该IP地址和IP地址充当本地网络的默认网关。 Active路由器负责转发流量。 如果失败， 备用路由器将承担活动路由器的所有职责并转发流量。

与HSRP相关的一些重要术语：

  1. 虚拟IP：本地子网的IP地址被指定为网络中所有本地主机的默认网关。
  2. 虚拟MAC地址 ：MAC地址由HSRP自动生成。 前24位将是默认的CISCO地址（即0000.0c）。 接下来的16位是HSRP ID （即07.ac）。 接下来的8位将是十六进制的组号。 例如 - 如果组号是10，那么最后8位将是0a。

    虚拟MAC地址示例 - 

     0000.0c07.ac0a
  3. Hello消息：主备路由器交换的定时消息。 告诉路由器的状态，每隔3秒就会交换这些消息。
  4. 按住定时器：默认值为10秒，即大约是hello消息值的3倍。 此计时器告诉我们路由器如果没有按时收到备用路由器等待hello消息的时间。

    注意：如果活动路由器出现故障，则备用路由器将成为活动路由器。

  5. 优先级：默认情况下，优先级值为100.当活动路由器在跌落后返回时，我们可以将备用路由器（在原始活动路由器关闭后成为活动路由器）的优先级更改为小于因此，它再次成为备用路由器。

    注意：具有较高优先级的路由器将成为活动路由器。



  6. 抢占：备用路由器自动成为活动路由器的状态。

工作：

具有相同组ID的成员是同一组的成员。 该组的一个成员将被选为活动路由器，而其他成员将保留为备用路由器。 虚拟IP配置为本地子网中所有主机的默认网关，活动路由器负责转发本地主机的流量。 如果活动路由器出现故障，则主用路由器和备用路由器之间不会交换hello消息，因此备用路由器会等待直到按下定时器时间。 一旦停机时间结束，备用路由器将成为活动路由器并承担活动路由器的所有职责。 这被称为先发制人。
如果原来的活动路由器返回，那么我们可以降低备用路由器的优先级，使其再次成为备用路由器。

热备用路由器协议（HSRP）有2个版本 ：

版本1：消息在224.0.0.2多播，并使用UDP端口1985.此版本允许组号范围从0到255。
版本2：消息在224.0.0.102多播并使用UDP端口1985.此版本允许组号范围从0到4095。

配置：


考虑上面给出的拓扑。 有2个名为R1和R2的路由器。 R1（f 0/0）的IP地址为10.1.1.1/24，R2（f 0/0）为10.1.1.2/24。

为路由器R1分配IP地址。 

 r1# int fa0/0
r1# ip add 10.1.1.1 255.255.255.0

为路由器R2分配IP地址。 

 r2# int fa0/0
r2# ip address 10.1.1.2 255.255.255.0

现在，让我们提供虚拟IP地址（10.1.1.100），组名HSRP_TEST ，组号1和优先级110.此外，已启用抢占，即如果活动路由器出现故障，则备用路由器将自动成为活动路由器。 

 r1# standby 1 ip 10.1.1.100
r1# standby 1 name HSRP_TEST
r1# standby 1 priority 110
r1# standby 1 preempt

现在，我们将提供虚拟IP地址（10.1.1.100），组名HSRP_TEST和优先级100.此外，已启用组号1和抢占。 

 r2# int fa0/0
r2# standby 1 ip 10.1.1.100
r2# standby 1 name HSRP_TEST
r2# standby 1 priority 100
r2# standby 1 preempt

注意：由于我们已经为r1提供了优先级110，因此它将成为活动路由器。
