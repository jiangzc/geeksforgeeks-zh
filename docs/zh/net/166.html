扩展访问列表

先决条件 - 访问列表（ACL） ， 标准访问列表
访问列表（ACL）是为控制网络流量和减少网络攻击而定义的一组规则。 ACL用于根据为传入或传出网络定义的规则集来过滤流量。

扩展访问列表 -
它是主要用于访问列表的类型之一，因为它可以区分IP流量，因此不会像标准访问列表那样允许或拒绝整个流量。 这些是使用源和目标IP地址以及端口号来区分IP流量的ACL。 在这些类型的ACL中，我们还可以提及应允许或拒绝哪些IP流量。 这些使用范围为100-199和2000-2699。

特征 -

  1. 扩展访问列表通常应用于源附近，但并非总是如此。
  2. 在扩展访问列表中，数据包过滤基于源IP地址，目标IP地址，端口号进行。
  3. 在扩展访问列表中，将允许或拒绝特定服务。
  4. 扩展ACL的创建范围为100 - 199和扩展范围2000 - 2699。
  5. 如果使用扩展的Access-list编号，则记住规则不能删除。 如果删除其中一个规则，则将删除整个访问列表。
  6. 如果使用扩展的Access-list命名，那么我们可以灵活地从access-list中删除规则。

配置 -



这是一个小型拓扑，其中有3个部门，即销售，财务和营销。 网络172.16.10.40/24的销售部门，网络172.16.50.0/24的财务部门以及网络172.16.60.0/24的营销部门。 现在，我们要拒绝从销售部门到财务部门的FTP连接，并拒绝从销售和营销部门到财务部门的telnet。

现在，首先配置编号扩展访问 - 拒绝从销售到财务部门的FTP连接列表。 

 R1# config terminal
R1(config)# access-list 110 
            deny tcp 172.16.40.0 0.0.0.255 172.16.50.0 0.0.0.255 eq 21

在这里，我们首先创建一个编号的访问列表，其中我们使用110（从扩展访问列表范围使用）并拒绝销售网络（172.16.40.0）以建立到财务网络（172.16.50.0）的FTP连接。

注 -这里，因为FTP使用TCP和端口号21.因此，我们必须根据需要指定permit或deny条件。 此外，在eq之后，我们使用指定应用层协议的端口号。

现在，我们不得不拒绝销售和营销部门与财务部门的telnet连接，这意味着没有人应该远程到财务部门。 配置相同。 

 R1(config)# access-list 110 
            deny tcp any 172.16.50.0 0.0.0.255 eq 23

在这里，我们使用了关键字any，意思是0.0.0.0 0.0.0.0，即来自任何子网掩码的任何ip地址。 因为telnet使用端口号23，所以我们必须在eq之后指定端口号23。 

 R1(config)# access-list 110 permit ip any any

现在，这是最重要的部分。 正如我们已经知道的那样，在每个访问列表的末尾都有一个隐式拒绝，这意味着如果流量与Access-list的任何规则都不匹配，那么流量将被丢弃。

通过指定任何具有任何IP地址流量的源将到达财务部门的方式，除了它与我们已经制定的上述规则匹配的流量。 现在，我们必须在路由器的接口上应用访问列表： 

 R1(config)# int fa0/1
R1(config-if)# ip access-group 110 out

我们记得，我们必须尽可能地将扩展访问列表应用于源，但是我们已经将它应用于靠近目的地，因为我们必须阻止来自销售和营销部门的流量，因此，我们必须申请它接近目的地，否则我们必须为fa0 / 0和fa1 / 0入站制作单独的访问列表。

命名访问列表示例 -

现在，考虑到相同的拓扑，我们将创建一个命名的扩展访问列表。 

 R1(config)# ip access-list extended blockacl

通过使用此命令，我们创建了一个名为blockacl的访问列表。 

 R1(config-ext-nacl)# deny tcp 172.16.40.0 0.0.0.255 172.16.50.0 0.0.0.255 eq 21 
R1(config-ext-nacl)# deny tcp any 172.16.50.0 0.0.0.255 eq 23
R1(config-ext-nacl)# permit ip any any

然后我们在编号的access-list中完成了相同的配置。 

 R1(config)# int fa0/1
R1(config-if)# ip access-group blockacl out
