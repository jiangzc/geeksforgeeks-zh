交换机间链路（ISL）和IEEE 802.1Q

VLAN用于在第2层划分广播域。默认情况下，所有交换机端口都在VLAN 1中。配置VLAN 1以外的VLAN然后承载这些VLAN的流量，用户必须建立连接到的交换机端口中继另一个开关。 如果帧被转发到接入链路，则认为该帧属于在该交换机端口上配置的VLAN。 但是，如果帧被转发到中继链路，那么其他设备如何知道流量属于哪个VLAN？ 有了VLAN识别方法的概念。

VLAN识别方法 -
如果帧被转发到中继链路，则将标头或标签添加到帧头，其指定帧所属的VLAN。 帧被封装在发送方的交换机上，并在接收方的交换机上被移除，然后被转发到属于该VLAN的端口（根据交换机的处理）。

有2种VLAN识别方法：

  1. 交换机间链路（ISL） -
    这是一种VLAN识别方法，其中VLAN信息被明确标记到以太网帧上。 ISL是CISCO交换机的专有产品。 ISL通过使用新报头封装数据帧并执行新的循环冗余校验（CRC）来在第2层起作用。

    在ISL中，原始帧被封装，并且在通过中继链路传送帧之前添加附加报头。 在接收端，删除标头，并将帧转发到指定的VLAN。



    ISL支持多达1000个vlan。 本地VLAN的概念对于ISL并不重要，因为包括本地VLAN的所有帧都被标记。

    ISL帧封装是30字节，26字节头和4字节FCS（帧校验序列）被插入。 因此总共有30字节的开销。 因此，它不太优选。 甚至思科建议使用802.1Q。

    配置（ISL）： 

     Switch(config)#interface Fa0/1
Switch(config-if)#switchport trunk encapsulation isl
Switch(config-if)#switchport mode trunk
  2. 802.1Q -
    它是帧标记的IEEE标准方法。 如果用户想要在Cisco交换机链路和不同的品牌交换机链路之间中继，则必须使用802.1Q封装来使中继工作。

    在802.1Q中，集群设备在原始帧中插入4字节标记，并在设备通过中继链路发送帧之前重新计算帧校验序列（FCS）。 在接收端，标签被删除，帧被转发到指定的VLAN。

    在4个字节中，有一个最重要的12位VLAN。 该字段说明帧所属的VLAN编号。 VLAN可以在1到4094之间变化，即支持4094个VLAN，因为VLAN 0 4095是保留VLAN。 802.1Q支持本机Vlan的概念，即流量将为此Vlan取消标记。

    注 -交换机间链路（ISL）和802.1Q用于提供交换机间VLAN通信。 如果帧被转发到访问链路，则删除ISL或802.1Q帧标记。

    配置（802.1Q）： 

     Switch(config)#interface Fa 0/1
Switch(config-if)#switchport trunk encapsulation 802.1q
Switch(config-if)#switchport mode trunk
