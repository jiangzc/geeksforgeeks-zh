CSMA / CD的退避算法

先决条件 - CSMA / CD的基础知识，CSMA / CD中的碰撞检测
退避算法是一种冲突解决机制，用于随机接入MAC协议（CSMA / CD）。 该算法通常用于以太网以调度冲突后的重传。

如果两个站之间发生碰撞，它们可能会在碰撞后尽快重新开始传输。 这将总是导致另一次碰撞并形成无限的碰撞循环，从而导致死锁。 为了防止这种情况，使用退避算法。

让我们考虑两个站A和B传输一些数据的情景：

在冲突之后，时间被分成离散时隙（ T 时隙 ），其长度等于2t，其中t是网络中的最大传播延迟。



碰撞中涉及的站随机从集合K中选择一个整数，即{0,1}。 该集合称为争用窗口。 如果源因为选择了相同的整数而再次发生冲突，则争用窗口大小加倍并变为{0,1,2,3}。 现在，第二次碰撞中涉及的源随机从集合{0,1,2,3}中选择一个整数，并在再次尝试之前等待该数量的时隙。 在他们尝试传输之前，他们只有在信道空闲时才收听信道并进行传输。 这使得在竞争窗口中选择最小整数的源成功发送其帧。

因此，退避算法定义了冲突所涉及的站的等待时间 ，即站等待重新发送的时间。 

 Waiting time = back–off time
Let n = collision number or re-transmission serial number. 
Then, 
Waiting time = K * T slot
where K = [0, 2 n – 1 ]

示例 -

情况1 ：
假设2个站A和B同时开始发送数据（包1），则发生冲突。 因此，它们的数据（分组1）的冲突数n = 1.现在，两个站都从组K中随机选择一个整数，即{0,1}。

  • 当A和B都选择K = 0时
    - >等待时间A = 0 * T slot = 0
    B = 0 * T slot = 0的等待时间

    因此，两个站将同时发送，因此发生冲突。

  • 当A选择K = 0而B选择K = 1时
    - >等待时间A = 0 * T slot = 0
    等待时间为B = 1 * T slot = T slot

    因此，A发送分组，B等待时间T 时隙用于发送，因此A获胜。

  • 当A选择K = 1而B选择K = 0时
    - > A = 1 * T slot = T slot的等待时间
    B = 0 * T slot = 0的等待时间

    因此，B发送分组，A等待时间T 时隙用于发送，因此B获胜。

  • 当A和B都选择K = 1时
    - > A = 1 * T slot = T slot的等待时间
    等待时间为B = 1 * T slot = T slot

    因此，两者都会等待T 槽然后再发送。 因此，发生碰撞。 

 Probability that A wins = 1/4
Probability that B wins = 1/4
Probability of collision = 2/4

案例2：
假设A在案例1中胜出并传输其数据（数据包1）。 现在，只要B发送其分组1，A就发送其分组2.因此，发生冲突。 现在碰撞没有。 对于A的分组2，n变为1，对于B的分组1变为2。
对于A的数据包2，K = {0,1}
对于B的包1，K = {0,1,2,3} 

 Probability that A wins = 5/8
Probability that B wins = 1/8
Probability of collision = 2/8

因此，与情况1相比，碰撞概率降低。

优势 -

  • 碰撞概率呈指数下降。

缺点 -

  • 捕获效果：赢得胜利的站一直在赢。
  • 仅适用于2个站点或主机。
