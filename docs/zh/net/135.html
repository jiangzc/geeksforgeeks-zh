算术编码的数据压缩

在继续阅读之前，建议阅读类似的文章： LZW（Lempel-Ziv-Welch）压缩技术

数据压缩是减少数据大小以节省空间或传输时间的过程。

没有压缩：

  • 分辨率为800 * 600的32位图像将采用（（800 X 600）X 32）/（8 X 1024 X 1024）= 1.83 MB。
  • 2000个字符的文本文件将占用1.95 KB。
  • 1分钟的音频立体声文件需要10 MB。
  • 1分钟的视频剪辑将占用2GB空间。

导致磁盘空间大，传输时间长的要求。 压缩使用一些数学算法减小文件大小。 由于在压缩和减压中浪费了处理时间而产生了轻微的缺点，并且涉及各种因素之间的折衷，包括压缩程度，引入的失真量（有损压缩）。

压缩是如何可能的 -
信息通常包含冗余，即信息经常重复。 示例包括重复出现的字母，数字或像素。 压缩程序消除了这种冗余。

什么是算术编码 -
算术编码是无损和有损数据压缩算法中常用的算法。 这是一种熵编码技术，其中频繁看到的符号使用比很少看到的符号更少的比特来编码。 它比霍夫曼编码等众所周知的技术具有一些优势。



算术编码如何工作 -

  • 它将整个输入数据转换为单个浮点数n，其中（0.0 <= n <1.0）。
  • 该间隔按发生频率的比率分成子间隔。
  • 对于整个范围的起点和终点，字符范围的下限是起始点+ cumulativefrequency X（endpoint -startpoint）给出的前一个字符的上限
  • 因此，每个间隔对应一个符号。
  • 第一个符号将标签位置限制在其中一个间隔中。
  • 随着更多符号的处理，减少的间隔被递归地划分。
  • 观察：一旦标签落入一个区间，它就永远不会脱离它。

算术编码的一个小例子 - 

 Consider the string "ARBER".
               Symbol ABER
               Frequency 1 1 1 2
               Probability 20% 20% 20% 40%

所以初始范围分别为= A [0 - 0.2），B [0.2 - 0.4]，E [0.4 - 0.6），R [0.6 - 1]

编码和解码程序的伪代码 - 

/* ARITHMETIC ENCODING ALGORITHM. */
/* Call encode-symbol repeatedly for each symbol in the message. */
/* Ensure that a distinguished "terminator" symbol is encoded last, then */
/* transmit any value in the range [low, high). */

encode-symbol(symbo1, cum-freq)
    range = high - low
    high = low + range * cum-freq[symbol-1]
    low = low + range*cum-freq[symbol]

/* ARITHMETIC DECODING ALGORITHM. */

/* "Value" is the number that has been received. */
/* Continue calling decode-symbol until the terminator symbol is returned. */

decode-symbol(cum-freq)
    find symbol such that
    cum-freq[symbol] <= (value-low)/(high-low) < cum-freq[symbol-1]
/* This ensures that value lies within the new  */
/* (low, high) range that will be calculated by */
/* the following lines of code. */

range = high - low
high = low + range*cum-freq[symbol-1]
low = low + range*cum-freq[symbol]
return symbol
