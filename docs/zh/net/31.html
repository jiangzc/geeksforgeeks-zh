虚拟局域网（VLAN）

虚拟LAN（VLAN）是一种概念，我们可以在第2层（数据链路层）上逻辑划分设备。 通常，第3层设备划分广播域，但广播域可以使用VLAN的概念由交换机划分。

广播域是一种网段，其中如果设备广播分组，则同一广播域中的所有设备将接收该分组。 同一广播域中的设备将接收所有广播数据包，但仅限于交换机，因为路由器不转发广播数据包。要将数据包转发到不同的VLAN（从一个VLAN到另一个VLAN）或广播域，需要VLAN间路由。 通过VLAN，可以创建不同的小尺寸子网络，相对容易处理。

VLAN范围 -

  • VLAN 0,4095：这些是保留的VLAN，无法查看或使用。
  • VLAN 1：交换机的缺省VLAN。 默认情况下，所有交换机端口都在VLAN中。 无法删除或编辑此VLAN，但可以使用该VLAN。
  • VLAN 2-1001：这是普通的VLAN范围。 我们可以创建，编辑和删除这些VLAN。
  • VLAN 1001-1005：这些是fddi和令牌环的CISCO默认值。 这些VLAN无法删除。
  • Vlan 1006-4094：这是Vlan的扩展范围。

配置 -
我们只需分配vlan-id和Vlan名称即可创建VLAN。 

 #switch1(config)#vlan 2
#switch1(config-vlan)#vlan accounts

这里，2是Vlan我和帐户是Vlan名称。 现在，我们将Vlan分配给交换机端口.eg- 



 Switch(config)#int fa0/0
Switch(config-if)#switchport mode access
Switch(config-if)#switchport access Vlan 2

此外，可以将switchport范围分配给所需的vlan。 

 Switch(config)#int range fa0/0-2
Switch(config-if)#switchport mode access
Switch(config-if) #switchport access Vlan 2

这样，switchport fa0 / 0，fa0 / 1，fa0-2将被分配Vlan 2。
示例 -

将IP地址192.168.1.1/24,192.168.1.2/24和192.168.2.1/24分配给PC。 现在，我们将在交换机上创建Vlan 2和3。 

 Switch(config)#vlan 2
Switch(config)#vlan 3

我们制作了VLAN，但最重要的部分是将交换机端口分配给VLAN。 

 Switch(config)#int fa0/0
Switch(config-if)#switchport mode access
Switch(config-if) #switchport access Vlan 2

Switch(config)#int fa0/1
Switch(config-if)#switchport mode access
Switch(config-if) #switchport access Vlan 3

Switch(config)#int fa0/2
Switch(config-if)#switchport mode access
Switch(config-if) #switchport access Vlan 2

如图所示，我们已将Vlan 2指定为fa0 / 0，fa0 / 2和Vlan 3指定为fa0 / 1。

优点 -

  • performance -网络流量充满广播和多播。 VLAN减少了将此类流量发送到不必要的目的地的需要。如果流量是针对2个用户但由于10个设备存在于同一广播域中，因此所有流量都会接收，即带宽浪费但如果我们创建VLAN，则广播或多播数据包将仅发送给目标用户。
  • 虚拟组的形成 -由于每个组织中都有不同的部门，即销售，财务等，因此VLAN可以非常有用，以便根据其部门对设备进行逻辑分组。
  • 安全性 -在同一网络中，可以广播敏感数据，可由外部人员访问，但通过创建VLAN，我们可以控制广播域，设置防火墙，限制访问。 此外，VLAN可用于通知网络管理员入侵。 因此，VLAN极大地增强了网络安全性。
  • 灵活性 - VLAN可以灵活添加，删除我们想要的主机数量。
  • 降低成本 - VLAN可用于创建广播​​域，从而无需昂贵的路由器。
  • 通过使用Vlan，可以增加小尺寸广播域的数量，与更大的广播域相比，这些域易于处理。
