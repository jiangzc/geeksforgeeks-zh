网络地址转换类型（NAT）

先决条件 - 网络地址转换（NAT）
网络地址转换（NAT）是将一个或多个本地IP地址转换为一个或多个全局IP地址的过程，反之亦然，以便为本地主机提供Internet访问。 NAT通常在路由器或防火墙上运行。

网络地址转换（NAT）工作 -
通常，边界路由器被配置用于NAT，即在本地（内部）网络中具有一个接口并且在全局（外部）网络中具有一个接口的路由器。 当数据包遍历本地（内部）网络之外时，NAT会将该本地（私有）IP地址转换为全局（公共）IP地址。 当数据包进入本地网络时，全局（公共）IP地址将转换为本地（专用）IP地址。

如果NAT用完了地址，即配置的池中没有剩余地址，则将丢弃数据包，并发送到目的地的Internet控制消息协议（ICMP）主机不可达数据包。

NAT类型 -
NAT有3种类型：

1.静态NAT -
在此，单个私有IP地址用单个公共IP地址映射，即私有IP地址被转换为公共IP地址。 它用于Web托管。



配置 -

这是一个小型拓扑，其中有PC的IP地址为192.168.1.1/24,Router R1在接口fa0 / 0上的IP地址为192.168.1.2/24,12.1.1.1/24在fa0 / 1上，服务器的IP地址为73.1 1.2 / 24。

现在，图中显示了局部和内部全局。 通过命令ip nat配置静态NAT内部源静态INSIDE_LOCAL_IP_ADDRESS INSIDE_GLOBAL_IP_ADDRESS。 

 R1(config)# ip nat inside source static 192.168.1.1 12.1.1.1

现在，我们将路由器的内部接口配置为IP NAT内部和外部接口作为IP NAT外部。 

 R1(config)# int fa0/0
R1(config-if)# ip nat inside 
R1(config)# int fa0/1
R1(config-if)# ip nat outside

2.动态NAT -
在这种类型的NAT中，多个私有IP地址被映射到公共IP地址池。 当我们知道在给定时间点固定用户想要访问互联网的数量时使用它。

配置 -

存在IP地址为192.168.1.1/24的PC，路由器R1在接口fa0 / 0上具有IP地址192.168.1.2/24，在fa0 / 1上具有12.1.1.1/24和具有IP地址73.1.1.2/24的服务器。
现在，首先配置访问列表： 

 R1(config)# access-list 1 permit 192.168.1.0 0.0.0.255

配置将从中选择公共IP的nat池。 

 R1(config)# ip nat pool pool1 12.1.1.1 12.1.1.3 netmask 255.255.255.0

现在，启用动态NAT： 

 R1(config)# ip nat inside source list 1 pool pool1

最后，我们必须将路由器接口配置为内部或外部。 

 R1(config)# int fa0/0
R1(config-if)# ip nat inside
R1(config)# int fa0/1
R1(config-if)# ip nat outside

3.端口地址转换（PAT） -
这也称为NAT过载。 在此，许多本地（私有）IP地址可以转换为单个公共IP地址。 端口号用于区分流量，即哪些流量属于哪个IP地址。 这是最经常使用的，因为它具有成本效益，因为可以通过仅使用一个真实的全局（公共）IP地址将数千个用户连接到因特网。

配置 -

采用相同的拓扑结构，PC1的IP地址为192.168.1.1/24，路由器R1的接口fa0 / 0为IP地址192.168.1.2/24,fa0 / 1为12.1.1.1/24，IP地址为73.1.1.2 / 24。
现在，首先配置访问列表： 

 R1(config)# access-list 1 permit 192.168.1.0 0.0.0.255

配置将从中选择公共IP的nat池。 

 R1(config)# ip nat pool pool1 12.1.1.1 12.1.1.1 netmask 255.255.255.0

在这里，请注意nat池仅缩小到一个IP地址，使用的IP地址是路由器的外部接口IP地址。 如果您有其他IP，那么您也可以使用它。
现在，启用动态NAT重载（PAT）： 

 R1(config)# ip nat inside source list 1 pool pool1 overload

或者我们也可以使用 

 R1(config)# ip nat inside source list 1 interface fastEthernet 0/1 overload

最后，我们必须将路由器接口配置为内部或外部。 

 R1(config)# int fa0/0
R1(config-if)# ip nat inside
R1(config)# int fa0/1
R1(config-if)# ip nat outside
