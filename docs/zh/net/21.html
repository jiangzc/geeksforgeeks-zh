分组交换和延迟

分组交换是一种以分组的形式将数据传送到网络的方法。 为了通过网络快速有效地传输文件并最小化传输延迟，数据被分成可变长度的小块，称为Packet 。 在目的地，必须重新组装所有这些小部件（包），属于同一文件。 数据包由有效载荷和各种控制信息组成。 无需预先设置或预留资源。

分组交换在切换数据包时使用存储转发技术; 在转发数据包时，每一跳首先存储该数据包然后转发。 这种技术非常有用，因为由于某种原因，数据包可能在任何一跳被丢弃。 一对源和目标之间可能有多条路径。 每个数据包都包含源和目标地址，使用它们可以独立地通过网络。 换句话说，属于同一文件的分组可以或可以不通过相同的路径。 如果在某个路径上存在拥塞，则允许分组在现有网络上选择不同的路径。

分组交换网络旨在克服电路交换网络的弱点 ，因为电路交换网络对于小消息不是非常有效。

分组交换优先于电路交换：

  • 在带宽方面更有效，因为保留电路的概念不存在。
  • 最小的传输延迟。
  • 目的地更可靠，可以检测丢失的数据包。
  • 更容错，因为在任何链路断开的情况下，数据包可能遵循不同的路径，与电路交换不同。
  • 实施具有成本效益且相对便宜。

分组交换电路交换的缺点：



  • 分组交换不按顺序提供数据包，而电路交换提供有序的数据包传输，因为所有数据包都遵循相同的路径。
  • 由于数据包是无序的，我们需要为每个数据包提供序列号。
  • 由于可以遵循多条路径，每个节点的复杂性更高。
  • 传输延迟更多是因为重新路由。
  • 分组交换仅对小消息有用，但对于突发数据（大消息），电路交换更好。

分组交换模式：

  1. 面向连接的分组交换（虚电路）： -在开始传输之前，它在发送器和接收器之间使用信令协议建立逻辑路径或虚拟连接，并且属于该流的所有分组将遵循该预定义路由。 交换机/路由器提供虚拟电路ID以唯一地标识该虚拟连接。 数据被分成小单元，所有这些小单元都附有序列号的帮助。 总的来说，这里有三个阶段 - 设置，数据传输和拆除阶段。

    所有地址信息仅在设置阶段传输。 一旦发现到目的地的路由，就将条目添加到每个中间节点的交换表中。 在数据传输期间，包头（本地头）可能包含诸如长度，时间戳，序列号等信息。
    面向连接的交换在交换WAN中非常有用。 一些使用虚电路交换方法的流行协议是X.25，帧中继，ATM和MPLS（多协议标签交换）。


  2. 无连接分组交换（数据报）： -与面向连接的分组交换不同，在无连接分组交换中，每个分组包含所有必要的寻址信息，例如源地址，目的地地址和端口号等。在数据报分组交换中，每个分组都是独立处理的。 数据包属于一个流可能采用不同的路由，因为路由决策是动态做出的，因此到达目的地的数据包可能会出现故障。 它没有连接设置和拆卸阶段，如虚拟电路。
    在无连接分组交换中不保证分组传送，因此必须由使用附加协议的终端系统提供可靠的传送。 

     A---R1---R2---B

A is the sender (start)
R1,R2 are two router that store and forward data
B is receiver(destination)

    要将数据包从A发送到B，存在延迟，因为这是存储转发网络。

    分组交换延迟：

    1. 传输延迟
    2. 传播延迟
    3. 排队延迟
    4. 处理延迟



    传输延迟：
    将数据包放到链接上所花费的时间。 换句话说，只需要将数据位置于有线/通信介质上的时间。 它取决于数据包的长度和网络的带宽。 

     Transmission Delay = Data size / bandwidth = (L/B) second

    传播延迟：
    第一位从链路的发送方传输到接收方的时间。 换句话说，它只是位从起点到达目的地所需的时间。 传播延迟所依赖的因素是距离和传播速度。 

     Propagation delay = distance/transmission speed = d/s

    排队延迟：
    排队延迟是作业在队列中等待直到可以执行的时间。 这取决于拥堵。 它是数据包到达目的地与处理或执行数据包数据之间的时间差。 它可能主要由三个原因引起，即始发开关，中间开关或呼叫接收器维修开关。 

     Average Queuing delay = (N-1)L/(2*R)
where N = no. of packets
      L=size of packet
      R=bandwidth

    处理延迟：
    处理延迟是路由器处理数据包报头所需的时间。 分组处理有助于检测在分组到目的地的传输期间发生的比特级错误。 高速路由器中的处理延迟通常在几微秒或更短的数量级。
    简单来说，它只是处理数据包所需的时间。 

     Total time or End-to-End time 
= Transmission delay + Propagation delay+ Queuing delay
                                  + Processing delay

    对于M跳和N包 - 

     Total delay 
= M*(Transmission delay + propagation delay)+
                (M-1)*(Processing delay + Queuing delay) + 
                (N-1)*(Transmission delay)

    对于电路中的N个连接链路 -
    传输延迟= N * L / R.
    传播延迟= N *（d / s）
