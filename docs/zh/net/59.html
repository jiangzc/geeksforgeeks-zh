路由类型

路由是由第3层（或网络层）设备执行的过程，以便通过选择从一个网络到另一个网络的最佳路径来传递分组。

路由有3种类型：
1.静态路由 -
静态路由是我们必须在路由表中手动添加路由的过程。
优点 -

  • 没有路由器CPU的路由开销，这意味着可以使用更便宜的路由器进行路由。
  • 它增加了安全性，因为只有管理员才允许仅路由到特定网络。
  • 路由器之间没有带宽使用。

坏处 -

  • 对于大型网络，管理员在每个路由器的路由表中手动添加网络的每个路由是一项繁忙的任务。
  • 管理员应该对拓扑有很好的了解。 如果有新的管理员，那么他必须手动添加每条路由，这样他就应该非常了解拓扑的路由。

配置 -



R1在s0 / 0/1上具有IP地址172.16.10.6/30，在fa0 / 0上具有192.168.10.1/24。
R2在s0 / 0/0上具有IP地址172.16.10.2/30，在fa0 / 0上具有192.168.20.1/24。
R3在s0 / 1上具有IP地址172.16.10.5/30，在s0 / 0上为172.16.10.1/30，在fa0 / 0上为10.10.10.1/24。

现在为路由器R3配置静态路由： 

 R3(config)#ip route 192.168.10.0 255.255.255.0 172.16.10.2
R3(config)#ip route 192.168.20.0 255.255.255.0 172.16.10.6

这里，为192.168.10.0网络提供了路由，其中​​192.168.10.0是其网络，而172.16.10.2和172.16.10.6是下一跳地址。
现在，配置R2： 

 R2(config)#ip route 192.168.20.0 255.255.255.0 172.16.10.1
R2(config)#ip route 10.10.10.0 255.255.255.0 172.16.10.1
R2(config)#ip route 172.16.10.4 255.255.255.0 172.16.10.1

类似于R1： 

 R1(config)#ip route 192.168.10.0 255.255.255.0 172.16.10.5
R1(config)#ip route 10.10.10.0 255.255.255.0 172.16.10.5
R1(config)#ip route 172.16.10.0 255.255.255.0 172.16.10.5

2.默认路由 -
这是路由器配置为将所有数据包发送到单个路由器（下一跳）的方法。 数据包属于哪个网络无关紧要，它被转发到配置为默认路由的路由器。 它通常与末节路由器一起使用。 存根路由器是一种路由器，只有一条路由可以到达所有其他网络。
配置 -
使用我们之前用于静态路由的相同拓扑。

在此拓扑中，R1和R2是末节路由器，因此我们可以为这两个路由器配置默认路由。
配置R1的默认路由： 

 R1(config)#ip route 0.0.0.0 0.0.0.0 172.16.10.5

现在为R2配置默认路由： 

 R2(config)#ip route 0.0.0.0 0.0.0.0 172.16.10.1

3.动态路由 -
动态路由根据路由表中路由的当前状态自动调整路由。 动态路由使用协议来发现网络目标以及到达它的路由。 RIP和OSPF是动态路由协议的最佳示例。 如果一条路线发生故障，将自动调整到达网络目的地。

动态协议具有以下功能：

  1. 路由器应该运行相同的动态协议以交换路由。
  2. 当路由器发现拓扑发生变化时，路由器会将其通告给所有其他路由器。

优点 -

  • 易于配置。
  • 更有效地选择到目的地远程网络的最佳路由以及发现远程网络。

坏处 -

  • 消耗更多带宽用于与其他邻居通信。
  • 安静性低于静态路由。
