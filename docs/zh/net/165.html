标准访问列表

先决条件 - 访问列表（ACL）
访问列表（ACL）是为控制网络流量和减少网络攻击而定义的一组规则。 ACL用于根据为传入或传出网络定义的规则集来过滤流量。

标准访问列表 -
这些是仅使用源IP地址创建的访问列表。 这些ACL允许或拒绝整个协议套件。 它们不区分IP流量，例如TCP，UDP，Https等。通过使用数字1-99或1300-1999，路由器将其理解为标准ACL并将指定地址理解为源IP地址。

特征 -

  1. 标准访问列表通常应用于目的地附近（但并非总是如此）。
  2. 在标准访问列表中，拒绝整个网络或子网。
  3. 标准访问列表使用范围1-99和扩展范围1300-1999。
  4. 标准访问列表仅使用源IP地址实现。
  5. 如果使用标准Access-list编号，则记住规则无法删除。 如果删除其中一个规则，则将删除整个访问列表。
  6. 如果使用标准的Access-list命名，则可以灵活地从access-list中删除规则。

注 -与扩展访问列表相比，标准访问列表使用较少，因为流量允许或拒绝整个IP协议套件，因为它无法区分不同的IP协议流量。

配置 -



这是一个小型拓扑，其中有3个部门，即销售，财务和营销。 销售部门网络172.16.40.0/24，财务部门网络172.16.50.0/24和营销部门网络172.16.60.0/24。 现在，想要拒绝从销售部门到财务部门的连接，并允许其他人访问该网络。

现在，首先配置编号标准访问 - 拒绝从销售到财务部门的任何IP连接的列表。 

 R1# config terminal
R1(config)# access-list 10 deny 172.16.40.0 0.0.0.255

这里，与扩展访问列表一样，您无法指定允许或拒绝的特定IP流量。 另请注意，已使用通配符掩码（0.0.0.255，表示子网掩码255.255.255.0）。 从数字标准访问列表范围使用10。 

 R1(config)# access-list 110 permit ip any any

现在，正如您已经知道的那样，每个访问列表末尾都有一个隐式拒绝，这意味着如果流量与访问列表的任何规则都不匹配，那么流量将被丢弃。
通过指定具有任何IP地址流量的源将到达财务部门的任何方式，除了它与您所做的上述规则匹配的流量。

现在，您必须在路由器的接口上应用访问列表： 

 R1(config)# int fa0/1
R1(config-if)# ip access-group 10 out

您还记得标准访问列表通常应用于目标，此处如果您将访问列表应用于目的地附近，它也将满足我们的需求，因此，已应用出站到接口fa0 / 1。

命名标准访问列表示例 -

现在，考虑到相同的拓扑，您将创建一个命名的标准访问列表。 

 R1(config)# ip access-list standard blockacl

通过使用此命令，您创建了一个名为blockacl的访问列表。 

 R1(config-std-nacl)# deny 172.16.40.0 0.0.0.255 
R1(config-std-nacl)# permit any

然后在编号的访问列表中完成相同的配置。 

 R1(config)# int fa0/1
R1(config-if)# ip access-group blockacl out

Telnet示例的标准访问列表 -
如您所知，您无法在标准访问列表中指定要拒绝的特定IP流量，但可以通过在行vty行上应用访问列表使用标准访问列表来允许或拒绝telnet连接。

在这里，在给定的图中，您想要从任何网络拒绝telnet到财务部门。 配置相同： 

 R1(config)# access-list 10 deny any
R1(config)# line vty 0 4
R1(config-line)# access-class 10 out
