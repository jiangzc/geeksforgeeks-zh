多种访问协议

数据链路层负责两个节点之间的数据传输。 它的主要功能是 -

  • 数据链路控制
  • 多重访问控制

数据链接控制 -
数据链路控制负责通过使用成帧，错误控制和流量控制等技术在传输信道上可靠地传输消息。 对于数据链接控制，请参阅 - 停止和等待ARQ

多重访问控制 -
如果在发送器和接收器之间存在专用链路，则数据链路控制层就足够了，但是如果不存在专用链路，则多个站可以同时访问该信道。 因此，需要多种访问协议来减少冲突并避免串扰。 例如，在一个满是学生的教室里，当一位老师提出问题并且所有学生（或站）同时开始回答（同时发送数据）时，会产生很多混乱（数据重叠或数据丢失），然后教师（多种访问协议）的工作是管理学生并让他们一次回答一个。

因此，需要协议来在非专用信道上共享数据。 多种访问协议可以进一步细分为 -



1.随机接入协议：在此，所有电台都具有相同的优势，即没有电台比其他电台具有更高的优先级。 任何站点都可以根据介质的状态（空闲或忙碌）发送数据。 它有两个功能：

  1. 发送数据没有固定的时间
  2. 没有固定的站发送数据序列

随机访问协议进一步细分为：

（a）ALOHA -它是为无线局域网设计的，但也适用于共享媒体。 在这种情况下，多个站可以同时传输数据，因此可能导致冲突和数据混乱。

  • 纯阿罗哈：
    当站发送数据时，它等待确认。 如果确认未在规定的时间内到来，则该站等待称为退避时间（Tb）的随机时间量并重新发送数据。 由于不同的站等待不同的时间量，因此进一步碰撞的可能性降低。 

     Vulnerable Time = 2* Frame transmission time
Throughput = G exp{-2*G}
Maximum throughput = 0.184 for G=0.5
  • 开槽阿罗哈：
    它类似于纯粹的aloha，除了我们将时间划分为时隙并且仅在这些时隙的开头允许发送数据。 如果一个电台错过了允许的时间，它必须等待下一个时隙。 这降低了碰撞的可能性。 

     Vulnerable Time = Frame transmission time
Throughput = G exp{-*G}
Maximum throughput = 0.368 for G=1

有关ALOHA的更多信息，请参阅 - LAN Technologies

（b）CSMA -载波侦听多路访问确保较少的冲突，因为在发送数据之前，要求工作站首先检测介质（空闲或忙碌）。 如果它是空闲的，则它发送数据，否则它等待直到信道变为空闲。 然而，由于传播延迟，CSMA中仍然存在冲突的可能性。 例如，如果站A想要发送数据，它将首先感知介质。如果它发现信道空闲，它将开始发送数据。 然而，当第一位数据从站A发送（由于传播延迟而延迟）时，如果站B请求发送数据并感测介质，它也将发现它空闲并且还将发送数据。 这将导致来自站A和B的数据冲突。

CSMA访问模式 -

  • 1-persistent：节点感知信道，如果空闲，它发送数据，否则它一直继续检查媒体是否空闲，并在信道空闲后无条件地发送（1概率）。
  • 非持久性：节点感知信道，如果空闲则发送数据，否则它在随机时间（不连续）之后检查媒体，并在发现空闲时发送。
  • P-persistent：节点感知媒体，如果空闲则以p概率发送数据。 如果数据未被传输（（1-p）概率）则等待一段时间并再次检查介质，现在如果发现它是空闲的则它以p概率发送。 此重复将继续，直到发送帧为止。 它用于Wifi和分组无线电系统。
  • O-持久性：预先确定节点的优越性，并按顺序进行传输。 如果介质空闲，节点将等待其时隙发送数据。

（c）CSMA / CD -具有冲突检测的载波侦听多路访问。 如果检测到冲突，站可以终止数据传输。 有关详细信息，请参阅 - CSMA / CD的效率

（d）CSMA / CA -具有冲突避免的载波侦听多路访问。 冲突检测过程涉及发送者接收确认信号。 如果只有一个信号（它自己的），那么数据就会被成功发送，但如果有两个信号（它自己的信号和它已经碰撞过的信号）那么就意味着发生了碰撞。 为了区分这两种情况，碰撞必须对接收信号产生很大影响。 然而，在有线网络中并非如此，因此在这种情况下使用CSMA / CA.

CSMA / CA通过以下方式避免碰撞：

  1. 帧间空间 -站等待介质变为空闲，如果发现空闲，它不会立即发送数据（以避免由于传播延迟而发生冲突），而是等待一段称为帧间空间或IFS的时间。 在此之后，它再次检查介质是否空闲。 IFS持续时间取决于站的优先级。
  2. 争用窗口 -分为插槽的时间量。 如果发送方准备好发送数据，它会选择一个随机数的时隙作为等待时间，每当未找到介质空闲时，它就会加倍。 如果发现介质忙，则不会重新启动整个过程，而是在再次发现通道空闲时重新启动定时器。
  3. 确认 -如果在超时之前未收到确认，则发送方重新发送数据。

2.受控访问：
在此，数据由该站发送，该站被所有其他站批准。 有关详细信息，请参阅 - 受控访问协议

3.渠道化：
在此，链路的可用带宽在时间，频率和代码上被共享到多个站以同时访问信道。

  • 频分多址（FDMA） -可用带宽被划分为相等的频带，以便每个站可以分配其自己的频段。 还添加了保护带，以便不重叠频带以避免串扰和噪声。
  • 时分多址（TDMA） -在此，带宽在多个站之间共享。 为了避免冲突时间被划分为时隙和站点，这些时隙被分配用于传输数据。 然而，由于每个站需要知道其时隙，因此存在同步开销。 通过向每个插槽添加同步位来解决此问题。 TDMA的另一个问题是传播延迟，它通过增加保护频带来解决。
    有关详细信息，请参阅 - 电路交换
  • 码分多址（CDMA） -一个信道同时承载所有传输。 既没有带宽划分也没有时间划分。 例如，如果一个房间里有很多人都在同一时间说话，那么如果只有两个人说同一种语言，那么也可以完美地接收数据。 类似地，来自不同站的数据可以以不同的代码语言同时传输。
