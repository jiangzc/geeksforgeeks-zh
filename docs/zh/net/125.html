统一

Unicode是一种通用编码系统，可提供全面的字符集，由Unicode联盟（一组多语言软件制造商）创建。 Unicode简化了软件本地化并改进了多语言文本处理。 它克服了ASCII和扩展ASCII固有的困难。

Unicode标准化了脚本行为，允许从任何脚本和语言组合中提取的任何字符组合在单个文档中共存。 Unicode定义了其单个字符集的多种编码：UTF-7，UTF-8，UTF-16和UTF-32。 这些编码之间的数据转换是无损的 。

Unicode最初是一个2字节的字符集。 但是，Unicode版本3是一个4字节代码，与ASCII和扩展ASCII完全兼容。

这些都支持编码同一组字符。

  • UTF-8使用每个字符1到4个字节的任何位置，具体取决于字符，但ASCII只占用1个字节，4个字节用于不常见的字符。
  • UTF-16对大多数字符使用2个字节，而非常不寻常的字符使用4个字节。
  • UTF-32每个字符使用4个字节。 我们可以通过仅计算字节来计算UTF-32字符串中的字符数。

符号使用格式的十六进制数字，如下所示。



U-XXXXXXXX -
编号从U-00000000变为U-FFFFFFFF 。 Unicode将可用空间代码划分为平面。 一个平面是一个由65,536个代码点组成的连续组。 最重要的16位定义平面（即平面数= 65,535），每个平面最多可定义65,536个字符或符号。

平面类型 -

  1. 基本多语言平面（BMP） -平面0000，基本多语言平面设计为与以前的16位Unicode兼容。 该平面中最重要的16位全为零。 它主要定义不同语言的字符集，但某些控件和特殊字符除外。 它表示为U + XXXX，其中XXXX是最低有效16位，例如：U + 0900到U + 09FF保留给Devanagari，孟加拉语U + 2200到U + 22FF保留用于数学运算等。
  2. 补充多语言平面（SMP） -平面0001，补充多语言平面，旨在为BMP中排除的那些多语言字符提供更多代码。 示例：10140-1018F保留给古希腊数字。
  3. 补充表意平面（SIP） -平面0002，补充表意平面，旨在提供表意符号的代码，提供与声音形成对比的想法的符号，例如，20000-2A6DF保留用于CJK统一扩展B
  4. 辅助专用平面（SSP） - 000E，辅助专用平面，用于特殊字符，例如，E0000-E007F保留用于标签。
  5. 私人飞机（PUP） -飞机000F和0010，私人飞机是私人使用的。 它们在内部由字体用于引用辅助字形。
