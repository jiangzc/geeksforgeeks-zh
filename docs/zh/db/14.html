ER图的最小化

实体关系（ER）图是数据库中数据的图解表示，它显示了数据的相关性。

注意：本文适合那些已经知道ER图是什么以及如何绘制ER图的人。

1）当ER图中有一对多基数时。
例如，学生只能注册一门课程，但许多学生可以注册课程


对于Student（SID，Name），SID是主键。 对于课程（CID，C_name），CID是主键 

 Student Course 
        (SID Name) ( CID C_name )
        -------------- -----------------
          1 A c1 Z
          2 B c2 Y
          3 C c3 X
          4 D

              Enroll 
            (SID CID)
             ----------
             1 C1
             2 C1
             3 c3
             4 C2

现在的问题是，什么应该是注册SID或CID或组合的主键。 我们不能将CID作为主键，因为您在注册时可以看到我们有多个SID的相同CID。 （SID，CID）可以唯一地区分表，但它不是最小的。 所以SID是关系注册的主要关键。



对于上面的ER图，我们考虑了数据库中的三个表 

 Student 
Enroll
Course

但是我们可以将Student和Enroll表结合起来重命名为Student_enroll。 

 Student_Enroll 
                ( SID Name CID )
                ---------------------
                  1 A c1
                  2 B c1
                  3 C c3
                  4 D c2

学生和注册表现在合并。

因此，Student_enroll和Course需要至少两个DBMS表。

注意：在One to Many关系中，我们可以至少有两个表。

2.当ER图中有多对多基数时。
让我们考虑上面的例子，现在学生也可以注册超过1门课程。 

 Student Course
( SID Name) ( CID C_name )
-------------- -----------------
   1 A c1 Z
   2 B c2 Y
   3 C c3 X
   4 D

              Enroll 
           ( SID CID )
             ----------
             1 C1
             1 C2
             2 C1
             2 C2
             3 c3
             4 C2

现在，同样的问题是，如果我们仔细分析Enroll的Enroll主键，那么Enroll关系的主键是什么
表是（SID，CID）。

但在这种情况下，我们无法将Enroll表与Student和Course中的任何一个合并。 如果我们尝试将Enroll与学生和课程中的任何一个合并，它将创建冗余数据。

注意：多对多关系中至少需要三个表。
