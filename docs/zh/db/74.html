数据库文件索引 - B +树（简介）

为了实现动态多级索引，通常采用B树和B +树。 然而，用于索引的B树的缺点是它存储数据指针（指向包含键值的磁盘文件块的指针），对应于特定键值，以及B节点中的键值。 -树。 该技术极大地减少了可以打包到B树的节点中的条目的数量，从而有助于B树中的级别数量的增加，因此增加了记录的搜索时间。

B +树通过仅在树的叶节点处存储数据指针来消除上述缺点。 因此，B +树的叶节点的结构与B +树的内部节点的结构完全不同。 这里可以注意到，由于数据指针仅存在于叶节点处，因此叶节点必须将所有键值以及它们对应的数据指针存储到磁盘文件块，以便访问它们。 此外，叶节点被链接以提供对记录的有序访问。 因此，叶节点形成第一级索引，内部节点形成多级索引的其他级别。 叶节点的一些关键值也出现在内部节点中，以简单地充当控制记录搜索的媒介。

从上面的讨论可以看出，与B树不同，B +树有两个命令，'a'和'b'，一个用于内部节点，另一个用于外部（或叶子）节点。

订单'a'的B +树的内部节点的结构如下：

  1. 每个内部节点的格式如下：
    <P 1 ，K 1 ，P 2 ，K 2 ，......，P c-1 ，K c-1 ，P c >
    其中c <= a且每个P i是树指针（即指向树的另一个节点），并且每个K i是键值 （参见图-I以供参考）。



  2. 每个内部节点都有：K 1 <K 2 <.... <K c-1
  3. 对于P i指向的子树中的每个搜索字段值“X”，以下条件成立：
    K i-1 <X <= K i ，1 <i <c，
    K i-1 <X，i = c
    （见图I供参考）
  4. 每个内部节点最多有一个'a'树指针。
  5. 根节点具有至少两个树指针，而其他内部节点各自具有至少\ ceil（a / 2）树指针。
  6. 如果任何内部节点有'c'指针，c <= a，那么它有'c-1'个键值。


图-I

“b”阶B +树的叶节点结构如下：

  1. 每个叶节点的形式如下：
    << K 1 ，D 1 >，<K 2 ，D 2 >，......，<K c-1 ，D c-1 >，P next >
    其中c <= b并且每个D i是数据指针（即指向磁盘中的实际记录，其键值为K i或指向包含该记录的磁盘文件块），并且每个K i是键值，并且P 接下来指向B +树中的下一个叶节点 （参见图II以供参考）。
  2. 每个叶节点都有：K 1 <K 2 <.... <K c-1 ，c <= b
  3. 每个叶节点至少具有\ ceil（b / 2）值。
  4. 所有叶节点都处于同一级别。


图-II

使用P next指针可以遍历所有叶节点，就像链表一样，从而实现对存储在磁盘中的记录的有序访问。

优势 -
与具有相同“1”级别的B树相比，具有“1”级别的B +树可以在其内部节点中存储更多条目。 这突出了对任何给定密钥的搜索时间的显着改进。 具有较低级别和存在P的下一个指针意味着B +树在从磁盘访问记录时非常快速和有效。
