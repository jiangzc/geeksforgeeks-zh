功能依赖和属性关闭

功能依赖

如果具有相同属性A值的两个元组也具有相同的属性B值，则关系中的函数依赖A-> B成立。例如，在表1中所示的STUDENT关系中，函数依赖关系 

 STUD_NO->STUD_NAME, STUD_NO->STUD_ADDR hold

但 

 STUD_NAME->STUD_ADDR do not hold



如何找到关系的功能依赖？

关系中的功能依赖关系取决于关系的域。 考虑表1中给出的学生关系。

  • 我们知道STUD_NO对每个学生都是独一无二的。 所以STUD_NO-> STUD_NAME，STUD_NO-> STUD_PHONE，STUD_NO-> STUD_STATE，STUD_NO-> STUD_COUNTRY和STUD_NO - > STUD_AGE都将成立。
  • 同样，STUD_STATE-> STUD_COUNTRY将为真，就好像两个记录具有相同的STUD_STATE，它们也具有相同的STUD_COUNTRY。
  • 对于关系STUDENT_COURSE，COURSE_NO-> COURSE_NAME将为true，因为具有相同COURSE_NO的两条记录将具有相同的COURSE_NAME。

功能依赖集：功能依赖集或关系的FD集是关系中存在的所有FD的集合。 例如，表1中显示的关系STUDENT的FD设置为： 

 { STUD_NO->STUD_NAME, STUD_NO->STUD_PHONE, STUD_NO->STUD_STATE, STUD_NO->STUD_COUNTRY, 
  STUD_NO -> STUD_AGE, STUD_STATE->STUD_COUNTRY }


属性闭包：属性集的属性闭包可以定义为可以从中进行功能确定的属性集。

如何查找属性集的属性闭包？
要查找属性集的属性闭包：

  • 将属性集的元素添加到结果集。
  • 递归地向结果集添加元素，这些元素可以从结果集的元素中进行功能确定。

使用表1的FD集，属性闭包可以确定为： 

 (STUD_NO)+ = {STUD_NO, STUD_NAME, STUD_PHONE, STUD_STATE, STUD_COUNTRY, STUD_AGE}
(STUD_STATE)+ = {STUD_STATE, STUD_COUNTRY}

如何使用属性关闭查找候选键和超级键？

  • 如果属性集的属性闭包包含关系的所有属性，则属性集将是关系的超级键。
  • 如果此属性集的任何子集都不能在功能上确定关系的所有属性，则该集合也将是候选键。 例如，使用表1的FD集，

（STUD_NO，STUD_NAME）+ = {STUD_NO，STUD_NAME，STUD_PHONE，STUD_STATE，STUD_COUNTRY，STUD_AGE}

（STUD_NO）+ = {STUD_NO，STUD_NAME，STUD_PHONE，STUD_STATE，STUD_COUNTRY，STUD_AGE}

（STUD_NO，STUD_NAME）将是超级密钥，但不是候选密钥，因为其子集（STUD_NO）+等于关系的所有属性。 因此，STUD_NO将成为候选键。

GATE问题：考虑关系方案R = {E，F，G，H，I，J，K，L，M，M}和函数依赖集{{E，F} - > {G}，{F R. - > {I，J}，{E，H} - > {K，L}，K - > {M}，L - > {N} on R. R的关键是什么？ （GATE-CS-2014）
A. {E，F}
B. {E，F，H}
C. {E，F，H，K，L}
D. {E}

答案：找到所有给定选项的属性闭包，我们得到：
{E，F} + = {EFGIJ}
{E，F，H} + = {EFHGIJKLMN}
{E，F，H，K，L} + = {{EFHGIJKLMN}
{E} + = {E}
{EFH} +和{EFHKL} +导致所有属性的集合，但EFH是最小的。 所以这将是候选人的关键。 所以正确的选项是（B）。

如何检查FD是否可以从给定的FD集导出？

要检查FD A-> B是否可以从FD集F中导出，

  1. 使用FD set F查找（A）+
  2. 如果B是（A）+的子集，则A-​​> B为真，否则不为真。

GATE问题：在具有属性A，B，C，D和E的模式中，给出了一组函数依赖性
{A - > B，A - > C，CD - > E，B - > D，E - > A}
上述设置中没有暗示以下哪些功能依赖？ （GATE IT 2005）
A. CD - > AC
B. BD - > CD
C. BC - > CD
D. AC - > BC

答案：使用有问题的FD集，
（CD）+ = {CDEAB}表示CD - > AC也适用。
（BD）+ = {BD}表示BD - > CD不能成立。 因此FD在FD集中没有暗示。 所以（B）是必需的选项。
其他人可以用同样的方式检查。

素数和非素数属性

作为关系的任何候选键的一部分的属性被称为主要属性，其他属性是非主要属性。 例如，STUDENT关系中的STUD_NO是主要属性，其他是非主要属性。

GATE问题：考虑以下函数依赖关系所依据的关系方案R =（A，B，C，D，E，H）：{A-> B，BC-> D，E-> C，D-> A }。 R的候选键是什么？ [GATE 2005]
（a）AE，BE
（b）AE，BE，DE
（c）AEH，BEH，BCH
（d）AEH，BEH，DEH

答案： （AE）+ = {ABECD}，它不是所有属性的集合。 所以AE不是候选键。 因此选项A和B是错误的。
（AEH）+ = {ABCDEH}
（BEH）+ = {BEHCDA}
（BCH）+ = {BCHDA}，它不是所有属性的集合。 所以BCH不是候选人。 因此选项C是错误的。
所以正确答案是D.
