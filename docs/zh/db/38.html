数据库规范化| 介绍

数据库规范化是组织数据库属性以减少或消除数据冗余（具有相同数据但在不同位置）的过程 。

由于数据冗余导致的问题
数据冗余不必要地增加了数据库的大小，因为在许多地方重复相同的数据。 插入，删除和更新操作期间也会出现不一致问题。

功能依赖
Functional Dependency是数据库关系中两组属性之间的约束。 功能依赖性由箭头（→）表示。 如果属性A在功能上确定B，则将其写为A→B。
例如，employee_id→name表示employee_id在功能上确定员工的姓名。 作为时间表数据库中的另一个示例，{student_id，time}→{lecture_room}，学生ID和时间决定学生应该在哪个教室。

功能依赖意味着什么？
函数依赖A→B表示A的特定值的所有实例，B的值相同。

例如，在下表中，A→B为真，但B→A不为真，因为对于B = 3，存在不同的A值。 



 AB
------
1 3
2 3
4 0
1 3
4 0

琐碎的功能依赖
只有当Y是X的子集时，X - > Y才是微不足道的。
例子 

 ABC --> AB
ABC --> A
ABC --> ABC

非平凡的功能依赖
当Y不是X的子集时，X - > Y是非平凡的函数依赖。

当X相交Y为NULL时，X - > Y被称为完全非平凡。
例子： 

 Id --> Name, 
Name --> DOB
