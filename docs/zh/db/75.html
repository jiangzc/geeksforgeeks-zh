位图索引

位图索引是一种使用位图的特殊类型的数据库索引。 当列具有低基数并且这些列在查询中最常使用时，此技术用于大型数据库。

例如，假设公司持有一个员工表，其中包含EmpNo，EmpName，Job，New_Emp和salary等条目。 让我们假设员工在一年中被雇用一次，因此表格将会更新，并且大部分时间都会保持不变。 但是这些列将经常用于查询以检索数据，例如：公司中的女性员工数量等。在这种情况下，我们需要一种文件组织方法，其速度应足够快，以便快速获得结果。 但是任何传统的文件组织方法都不是那么快，因此我们切换到一种更好的存储和检索数据的方法，称为位图索引。

如何完成位图索引 -
在上面的表employee的示例中，我们可以看到New_Emp列只有两个值Yes和No，这取决于员工是否是公司新手的事实。 我们假设员工的工作分为4类，即经理，分析员，文员和推销员。 这些列称为具有低基数的列。 即使这些列具有较少的唯一值，也可以经常查询它们。

位：位是计算中使用的信息的基本单位，只能包含0或1中的两个值中的一个。 二进制数字的两个值也可以解释为逻辑值true / false或yes / no。

在位图索引中，这些位用于表示那些低基数列中的唯一值。 这种以比特形式存储低基数行的技术称为位图索引。

SQL中的位图索引 - 在sql中创建位图索引的语法如下： 

 CREATE BITMAP INDEX Index_Name
         ON Table_Name (Column_Name);

对于employee表的上述示例，将创建New_Emp列的位图索引，如下所示： 

 CREATE BITMAP INDEX index_New_Emp
        ON Employee (New_Emp);

优点 -

  • 插入删除和更新方面的效率
  • 更快地检索记录

缺点 -

  • 仅适用于大型桌子
  • 位图索引非常耗时
