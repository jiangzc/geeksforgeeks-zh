文件组织 - 设置1

数据库由大量数据组成。 数据在RDBMS中的表中分组，每个表都有相关记录。 用户可以看到数据以表的形式存储，但是在实际中，这些大量数据以文件的形式存储在物理存储器中。

文件 -文件被命名为记录在二级存储上的相关信息的集合，例如磁盘，磁性表和光盘。

什么是文件组织？
文件组织是指构成文件的各种记录之间的逻辑关系，特别是关于识别和访问任何特定记录的方法。 简单来说，以特定顺序存储文件称为文件组织。 文件结构是指标签和数据块以及任何逻辑控制记录的格式。

文件组织的类型 -

组织文件已引入各种方法。 这些特定方法在访问或选择的基础上具有优点和缺点。 因此，程序员可以根据自己的要求决定最适合的文件组织方法。
某些类型的文件组织是：

  • 顺序文件组织
  • 堆文件组织
  • 哈希文件组织
  • B +树文件组织
  • 集群文件组织

我们将在本文的更多集合中讨论每个文件组织以及每个文件组织方法的差异和优缺点。



顺序文件组织 -

文件组织最简单的方法是顺序方法。 在该方法中，文件以顺序方式一个接一个地存储。 有两种方法可以实现此方法：

  1. 堆文件方法 -这种方法非常简单，我们将记录按顺序存储，即按照它们插入表中的顺序依次存储。

    插入新记录 -
    让R1，R3等依赖R5和R4为序列中的四个记录。 在这里，记录只不过是任何表格中的一行。 假设必须在序列中插入新记录R2，然后将其简单地放在文件的末尾。
  2. 排序文件方法 -在此方法中，正如名称本身建议每当必须插入新记录时，它总是以排序（升序或降序）方式插入。 记录的排序可以基于任何主键或任何其他键。

    插入新记录 -
    让我们假设存在预先存在的四个记录R1，R3等的排序序列，直到R7和R8。 假设必须在序列中插入新记录R2，然后将其插入文件的末尾，然后它将对序列进行排序。

顺序文件组织的优缺点 -
优点 -

  • 快速有效的大量数据方法。
  • 设计简单。
  • 文件可以很容易地存储在磁带中，即更便宜的存储机制。

缺点 -

  • 浪费时间，因为我们无法跳过所需的特定记录，但我们必须以顺序的方式移动，这需要花费我们的时间。
  • 排序文件方法效率低，因为排序记录需要时间和空间。

堆文件组织 -

堆文件组织使用数据块。 在此方法中，将记录插入到文件末尾的数据块中。 此方法不需要排序或排序。 如果数据块已满，则新记录存储在其他块中，此处另一个数据块不必是下一个数据块，但它可以是存储器中的任何块。 DBMS负责存储和管理新记录。

插入新记录 -
假设我们在堆R1，R5，R6，R4和R3中有四个记录，并假设必须在堆中插入新记录R2，因为最后一个数据块即数据块3已满，它将被插入任何一个DBMS选择的数据库，比方说数据块1。

如果我们想要搜索，删除或更新堆文件组织中的数据，我们将从文件的开头遍历数据，直到我们获得所请求的记录。 因此，如果数据库非常庞大，搜索，删除或更新记录将花费大量时间。

堆文件组织的优缺点 -
优点 -

  • 获取和检索记录比顺序记录更快，但仅限于小型数据库。
  • 当需要一次将大量数据加载到数据库中时，文件组织的这种方法最适合。

缺点 -

  • 未使用的内存块问题。
  • 对较大的数据库效率低下。
