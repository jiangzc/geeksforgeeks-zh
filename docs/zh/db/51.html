并发控制协议| 基于图的协议

基于图形的协议是实现基于锁的协议的另一种方式。

我们知道基于锁定协议的主要问题是避免死锁并确保严格的时间表。 我们已经看到严格的时间表可以使用严格或严格的2-PL 。 我们甚至看到如果我们遵循保守派2-PL可以避免死锁，但是这个协议的问题是它实际上不能使用。 基于图形的协议用作2-PL的替代方案。 基于树的协议是基于图形的协议的简单实现 。

此协议的先决条件是我们知道访问数据库项的顺序。 为此，我们对一组数据库项（D） {d 1 ，d 2 ，d 3 ，...，d n }实施部分排序 。 实施部分订购后的议定书陈述如下─

  • 如果d i - > d j，则访问d i和d j的任何事务必须在访问d j之前访问d i 。

  • 意味着集合D现在可以被视为有向无环图（DAG），称为数据库图 。



基于树的协议 -

  • 数据库项的部分顺序确定树状结构。
  • 只允许使用独家锁。
  • T i的第一次锁定可以在任何数据项上。 随后，仅当Q的父亲当前被T i锁定时，才能通过T i锁定数据Q.
  • 数据项可以随时解锁。

遵循基于树的协议确保冲突序列化和无死锁计划。 我们不必像在2-PL协议中那样等待解锁数据项，从而增加了并发性。

因此，让我们再次修改基于图的协议的关键点。
优势 -

  • 确保冲突可序列化的时间表。
  • 确保死锁免费计划
  • 解锁可以随时进行

有一些优点也有一些缺点。
坏处 -

  • 有时会发生不必要的锁定开销，比如我们想要D和E，那么至少我们必须锁定B来遵循协议。
  • 级联回滚仍然是一个问题。 我们不遵循可能发生解锁操作的规则，因此此协议仍存在此问题。

    • 总的来说，这个协议大多是已知的，并用于实现死锁自由的独特方式。
