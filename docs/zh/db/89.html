语句（P）“SQL查询可以包含HAVING子句，即使它没有GROUP BY子句”也是正确的，因为在聚合阶段之后应用了caluse，如果要过滤聚合结果并且必须使用它，则必须使用它t需要Group By子句。 没有GROUP BY子句的HAVING子句在标准SQL中是有效且（可以说）有用的语法。 考虑这个有效的标准SQL示例： 

 SELECT 'T' AS result
   FROM Book
 HAVING MIN(NumberOfPages) < MAX(NumberOfPages);

2）鉴于基本的ER和关系模型，以下哪一项是不正确的？
（A）实体的属性可以具有多于一个值
（B）实体的属性可以是复合的
（C）在关系表的一行中，属性可以具有多个值
（D）在关系表的一行中，属性可以只有一个值或NULL值

答案（C）
术语“实体”属于ER模型，术语“关系表”属于关系模型。
A和B都是真的。 ER模型支持多值和复合属性有关详细信息，请参阅此处。
（C）是假的，（D）是真的。 在Relation模型中，关系表中的条目可以只有一个值或NULL。
