与传统的基于文件的系统相比，DBMS有哪些优势？
Ans：数据库管理系统的开发是为了处理传统操作系统支持的典型文件处理系统的以下困难。
1.数据冗余和不一致
2.访问数据的困难
3.数据隔离 - 多种文件和格式
4.诚信问题
5.更新的原子性
6.多个用户的并发访问
7.安全问题

什么是超级，主要，候选和外键？
Ans： 超级密钥是关系模式的一组属性，模式的所有属性都依赖于这些属性。 没有两行可以具有相同的超级键属性值。
候选键是最小超级键，即，没有适当的候选键属性子集可以是超级键。
主键是候选键之一。 其中一个候选键被选为最重要的，并成为主键。 表中不能有多个主键。
外键是一个表中的字段（或字段集合），用于唯一标识另一个表的行。 请参阅此示例。



主键和唯一约束之间有什么区别？
Ans：主键不能有NULL值，唯一约束可以有NULL值。 表中只有一个主键，但可以有多个唯一约束。



什么是数据库规范化？
Ans：这是一个基于功能依赖性和主键分析给定关系模式以实现以下所需属性的过程：
1）尽量减少冗余
2）最小化插入，删除和更新异常
不满足属性的关系模式将分解为可以满足所需属性的较小关系模式。

什么是SQL？
SQL是结构化查询语言，用于在关系数据库系统中插入和修改。





SQL中的DDL，DML和DCL有什么区别？
答：以下是三个细节。
DDL代表数据定义语言。 像CREATE，ALTER，DROP和RENAME这样的SQL查询就属于这个问题。
DML代表数据操作语言。 SELECT，INSERT和UPDATE等SQL查询都属于此类。
DCL代表数据控制语言。 像GRANT和REVOKE这样的SQL查询就属于这个问题。

什么是身份？
Ans： Identity（或AutoNumber）是一个自动生成数值的列。 可以设置开始和增量值，但大多数DBA将这些值保留为1. GUID列也会生成数字; 这个价值无法控制。 不需要为Identity / GUID列编制索引。



SQL中的视图是什么？ 如何创建一个
Ans： 视图是基于SQL语句的结果集的虚拟表。 我们可以使用create view语法创建。 

 CREATE VIEW view_name AS
SELECT column_name(s)
FROM table_name
WHERE condition



视图的用途是什么？
1.视图可以表示表中包含的数据的子集; 因此，视图可以限制基础表暴露给外部世界的程度：给定用户可能有权查询视图，而拒绝访问基表的其余部分。
2.视图可以将多个表连接并简化为单个虚拟表
3.视图可以作为聚合表，数据库引擎聚合数据（总和，平均值等），并将计算结果作为数据的一部分显示
4.视图可以隐藏数据的复杂性; 例如，一个视图可能显示为Sales2000或Sales2001，透明地分区实际的基础表
5.意见占用的空间非常小; 数据库仅包含视图的定义，而不包含它呈现的所有数据的副本。
6.根据所使用的SQL引擎，视图可以提供额外的安全性

什么是触发器？
Ans： 触发器是与插入，更新或删除操作相关联的代码。 只要在表上执行关联查询，代码就会自动执行。 触发器可用于维护数据库中的完整性。



什么是存储过程？
Ans： 存储过程就像一个包含一组编译在一起的操作的函数。 它包含一组操作，这些操作通常在应用程序中用于执行一些常见的数据库任务。



触发器和存储过程有什么区别？
Ans：与存储过程不同，不能直接调用触发器。 它们只能与查询相关联。



什么是交易？ 什么是ACID属性？
Ans： 数据库事务是一组必须被视为整体的数据库操作，意味着要么所有操作都被执行，要么都不执行。
一个例子可以是从一个账户到另一个账户的银行交易。 借记和贷记业务都必须执行，或者都不执行。
ACID （原子性，一致性，隔离性，持久性）是一组保证数据库事务可靠处理的属性。



索引是什么？
Ans： 数据库索引是一种数据结构，它以额外写入为代价提高数据库表上数据检索操作的速度，并使用更多存储空间来维护额外的数据副本。
数据只能在磁盘上以一个顺序存储。 为了根据不同的值支持更快的访问，需要更快的搜索，例如二进制搜索不同的值，为此，在表上创建索引。 这些索引需要磁盘上的额外空间，但它们允许根据不同的频繁搜索值进行更快速的搜索。



什么是聚簇索引和非聚簇索引？
Ans：聚簇索引是根据数据物理存储在磁盘上的索引。 因此，只能在给定的数据库表上创建一个聚簇索引。
非聚集索引不定义数据的物理排序，而是定义逻辑排序。 通常，会创建一个树，其叶指向磁盘记录。 B-Tree或B +树用于此目的
