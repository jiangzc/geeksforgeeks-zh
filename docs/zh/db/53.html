并发控制协议| 两相锁定（2-PL）-II

现在我们熟悉什么是两相锁定（2-PL）以及应遵循的基本规则，以确保可串行性。 此外，我们遇到了2-PL，级联中止和死锁的问题。 现在，我们转向2-PL上的增强功能，试图使协议几乎没有错误。 简而言之，我们允许对2-PL进行一些修改以改进它。 有三类：

  1. 严格的2-PL
  2. 严谨的2-PL
  3. 保守的2-PL

现在回想一下Basic 2-PL中遵循的规则，我们做了一些额外的修改。 现在让我们看看他们解决了哪些修改和缺点。

严格的2-PL -

这要求除了锁定为2阶段之外，交易持有的所有独占（X）锁定都将被释放，直到交易提交之后。
遵循Strict 2-PL确保我们的时间表是：

  • 可采
  • Cascadeless

因此，它让我们免于级联中止，它仍然存在于Basic 2-PL中，而且保证严格的时间表，但仍然可以实现死锁 ！

严谨的2-PL -

这要求除了锁定为2阶段之外，交易所持有的所有独占（X）和共享（S）锁都将被释放，直到交易提交之后。
遵循严格的2-PL确保我们的时间表是：

  • 可采
  • Cascadeless

因此，它让我们免于级联中止，它仍然存在于Basic 2-PL中，而且保证严格的时间表，但仍然可以实现死锁 ！

注意Strict 2-PL和Rigorous 2-PL之间的区别在于Rigorous限制性更强，它需要在Transaction提交之前保持Exclusive和Shared锁，这使得Rigorous 2-PL的实现变得更加容易。



保守的2-PL -

AKA Static 2-PL ，此协议要求事务在事先开始执行之前锁定它访问的所有项目，方法是预先设置其读取集和写入集。 如果无法锁定任何所需的预先声明的项目，则事务不会锁定任何项目，而是等待所有项目都可用于锁定。

保守的2-PL是无死锁的 ，是的，它确保了无级别的时间表，而且它保证了严格的时间表。 然而，由于需要预先读取在许多情况下不可能的读取集和写入集，因此在实践中难以使用。 在实践中，最流行的2-PL变体是Strict 2-PL。
