1）考虑以下事务，数据项P和Q初始化为零： 

 T1: read (P) ;
    read (Q) ;
    if P = 0 then Q : = Q + 1 ;
    write (Q) ;
T2: read (Q) ;
    read (P) ;
    if Q = 0 then P : = P + 1 ;
    write (P) ;

任何用于并发执行的T1和T2的非串行交错都会导致
（A）可序列化的时间表
（B）不可冲突序列化的计划
（C）冲突可序列化的时间表
（D）无法绘制优先图的时间表

答案（B）
如果出现以下情况，则说两项或多项行动有冲突：
1）动作属于不同的交易。
2）至少一个动作是写操作。
3）动作访问同一对象（读或写）。

如果满足以下条件，则称时间表S1和S2是冲突等效的：
1）调度S1和S2都涉及同一组事务（包括每个事务中的动作排序）。
2）S1和S2中每对冲突动作的顺序是相同的。



当调度与一个或多个串行调度冲突等效时，调度可以说是冲突可序列化的。

来源： 附表的维基页面

在给定的场景中，有两种可能的串行调度：
1）T1后跟T2
2）T2后跟T1。
在两个串行调度中，其中一个事务作为第一步读取由其他事务写入的值。 因此，T1和T2的任何非串行交错都不会冲突串行化。

3）考虑上面的表A，B和C.以下SQL查询的结果包含多少个元组？ 

 SELECT A.id 
FROM A 
WHERE A.age > ALL (SELECT B.age 
                    FROM B 
                    WHERE B. name = "arun")

（A）4
（B）3
（C）0
（D）1

答案（B）
“ALL”的含义是A.Age应该大于子查询返回的所有值。 表B中没有名称为“arun”的条目。因此子查询将返回NULL。 如果子查询返回NULL，则对于A的所有行，条件变为true（有关详细信息，请参阅此处）。 因此，选择了表A的所有行。
