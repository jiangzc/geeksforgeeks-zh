1）考虑两个事务T1和T2，以及T1和T2的四个调度S1，S2，S3，S4，如下所示：
T1 = R1 [X] W1 [X] W1 [Y]
T2 = R2 [X] R2 [Y] W2 [Y]
S1 = R1 [X] R2 [X] R2 [Y] W1 [X] W1 [Y] W2 [Y]
S2 = R1 [X] R2 [X] R2 [Y] W1 [X] W2 [Y] W1 [Y]
S3 = R1 [X] W1 [X] R2 [X] W1 [Y] R2 [Y] W2 [Y]
S1 = R1 [X] R2 [Y] R2 [X] W1 [X] W1 [Y] W2 [Y]
上述哪些时间表是冲突可序列化的？
（A）S1和S2
（B）S2和S3
（C）仅限S3
（D）仅限S4

答案（B）
可以存在两个可能的串行调度T1 T2和T2 T1。 串行调度T1 T2具有以下操作序列
R1 [X] W1 [X] W1 [Y] R2 [X] R2 [Y] W2 [Y]
并且调度T2 T1具有以下操作序列。
R2 [X] R2 [Y] W2 [Y] R1 [X] W1 [X] W1 [Y]
附表S2与T2 T1冲突等效，S3与T1 T2发生冲突等效。



2）令R和S为关系方案，使得R = {a，b，c}且S = {c}。 现在考虑
以下对数据库的查询： 

 IV) SELECT Ra, Rb
       FROM R,S
            WHERE Rc=Sc

以上哪些查询相同？
（A）I和II
（B）I和III
（C）II和IV
（D）III和IV



答案（A）
I和II分别描述了关系代数和元组关系演算中的除法运算符。 有关更多详细信息，请参见第3页，并在此处滑动数字9,10。



3）考虑以下关系模式： 

 Suppliers( sid:integer , sname:string, city:string, street:string) 
 Parts( pid:integer , pname:string, color:string) 
 Catalog( sid:integer, pid:integer , cost:real)

在上面的数据库中考虑以下关系查询： 

 SELECT S.sname
    FROM Suppliers S
        WHERE S.sid NOT IN (SELECT C.sid
                            FROM Catalog C
                            WHERE C.pid NOT IN (SELECT P.pid  
                                                FROM Parts P                                                                                                    
                                                WHERE P.color<> 'blue'))

假设对应于上述模式的关系不为空。 以下哪一项是对上述查询的正确解释？

（A）查找提供非蓝色部件的所有供应商的名称。
（B）找出所有未提供非蓝色部件的供应商的名称。
（C）查找仅提供蓝色部件的所有供应商的名称。
（D）找出所有未提供蓝色部件的供应商的名称。

答案（A）
子查询“ SELECT P.pid FROM Parts P WHERE P.color <>'blue' ”给出了非蓝色部分的pids。 更大的子查询“ SELECT C.sid FROM Catalog C WHERE C.pid NOT IN（SELECT P.pid FROM Parts P WHERE P.color <>'blue'） ”给出了所有提供蓝色部件的供应商的sids。 完整查询给出了提供非蓝色部分的所有供应商的名称



4）假设在上面的供应商关系中，每个供应商和城市中的每条街道都具有唯一的名称，并且（sname，city）形成候选密钥。 除了主键和候选键所暗示的那些之外，没有暗示其他功能依赖性。 关于上述架构，以下哪一项是正确的？
（A）架构在BCNF中
（B）模式在3NF但不在BCNF中
（C）模式是2NF但不是3NF
（D）架构不在2NF中

答案（A）
如果对于每个依赖关系X，关系在BCNF中 ？ Y，至少满足下列条件之一： 

 X ? Y is a trivial functional dependency (Y ? X)
    X is a superkey for schema R

由于（sname，city）形成候选键，因此没有非tivvial依赖X？ Y，其中X不是超级密钥
