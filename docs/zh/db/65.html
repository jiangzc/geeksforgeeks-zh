DBMS中的事务隔离级别

先决条件 - DBMS中的并发控制，DBMS中的ACID属性

我们知道，为了保持数据库的一致性，它遵循ACID属性。 在这四个属性（原子性，一致性，隔离性和持久性）中，隔离确定了其他用户和系统对事务完整性的可见性。 这意味着事务应该在系统中以这样的方式发生，即它是访问数据库系统中资源的唯一事务。
隔离级别定义了事务必须与数据库系统中任何其他事务所做的数据修改隔离的程度。 事务隔离级别由以下现象定义 -

  • 脏读 -脏读是指事务读取尚未提交的数据时的情况。例如，假设事务1更新一行并使其不通信，同时事务2读取更新的行。 如果事务1回滚更改，则事务2将读取被认为永远不存在的数据。
  • 不可重复读取 -当事务两次读取同一行时，会发生不可重复读取，并且每次都获得不同的值。 例如，假设事务T1读取数据。 由于并发性，另一个事务T2更新相同的数据并提交，现在如果事务T1重新读取相同的数据，它将检索不同的值。
  • 幻像读取 -幻像读取在执行两个相同的查询时发生，但两者检索的行不同。 例如，假设事务T1检索满足某些搜索条件的一组行。 现在，事务T2生成一些与事务T1的搜索条件匹配的新行。 如果事务T1重新执行读取行的语句，则此次会获得一组不同的行。

基于这些现象，SQL标准定义了四个隔离级别：

  1. Read Uncommitted - Read Uncommitted是最低隔离级别。 在此级别中，一个事务可能读取其他事务尚未提交的更改，从而允许脏读。 在此级别中，事务不会彼此隔离。
  2. 读取已提交 -此隔离级别可确保在读取任何数据时将其提交。 因此它不允许脏读。 事务在当前行上保持读或写锁定，从而防止其他行读取，更新或删除它。
  3. 可重复读取 -这是最严格的隔离级别。 该事务在其引用的所有行上保持读锁定，并在其插入，更新或删除的所有行上写入锁定。 由于其他事务无法读取，更新或删除这些行，因此它避免了不可重复的读取。
  4. Serializable -这是最高隔离级别。 可序列化的执行保证可序列化。 可序列化执行被定义为执行同时执行事务似乎是串行执行的操作。
