多媒体数据库

多媒体数据库是相互关联的多媒体数据的集合，其包括文本，图形（草图，绘图），图像，动画，视频，音频等，并且具有大量的多源多媒体数据。 管理可以以不同方式存储，传递和利用的不同类型的多媒体数据的框架被称为多媒体数据库管理系统。 多媒体数据库有三类，包括静态媒体，动态媒体和维度媒体。

多媒体数据库管理系统的内容：

  1. 媒体数据 -表示对象的实际数据。
  2. 媒体格式数据 -关于媒体数据经过采集，处理和编码阶段后的格式的信息，如采样率，分辨率，编码方案等。
  3. 媒体关键字数据 -与数据生成相关的关键字描述。 它也被称为内容描述数据。 示例：日期，时间和录音地点。
  4. 媒体特征数据 -依赖于内容的数据，例如颜色的分布，纹理的种类和数据中存在的不同形状。

基于数据管理特征的多媒体应用类型包括：

  1. 存储库应用程序 -存储用于检索目的的大量多媒体数据以及元数据（媒体格式日期，媒体关键字数据，媒体特征数据），例如，卫星图像的存储库，工程图，放射学扫描图像。
  2. 演示应用程序 -它们涉及受时间限制的多媒体数据的传送。 最佳观看或收听要求DBMS以特定速率提供数据，从而提供高于特定阈值的服务质量。 这里数据在交付时进行处理。 示例：注释视频和音频数据，实时编辑分析。
  3. 使用多媒体信息的协作工作 -它涉及通过合并图纸，更改通知来执行复杂任务。 示例：智能医疗网络。

多媒体数据库仍面临许多挑战，其中一些是：

  1. 建模 -在这一领域工作可以改进数据库与信息检索技术，因此，文件构成了一个专门领域，值得特别考虑。
  2. 设计 -多媒体数据库的概念，逻辑和物理设计尚未得到充分解决，因为每个级别的性能和调优问题都要复杂得多，因为它们包含JPEG，GIF，PNG，MPEG等各种格式并不容易从一种形式转换为另一种形式。
  3. 存储 -在任何标准磁盘上存储多媒体数据库都存在表示，压缩，映射到设备层次结构，在输入 - 输出操作期间存档和缓冲的问题。 在DBMS中，“BLOB”（二进制大对象）工具允许存储和检索无类型位图。
  4. 性能 -对于涉及视频播放或音频 - 视频同步的应用，物理限制占主导地位。 并行处理的使用可以缓解一些问题，但是这些技术尚未完全开发。 除此之外，多媒体数据库还消耗大量处理时间和带宽。
  5. 查询和检索 -对于图像，视频等多媒体数据，通过查询访问数据的音频打开了许多问题，如高效的查询制定，查询执行和优化等需要处理的问题。

应用多媒体数据库的领域是：



  • 文件和记录管理：保存详细记录和各种文件的行业和企业。 示例：保险索赔记录。
  • 知识传播：多媒体数据库在提供多种资源方面是知识传播的一种非常有效的工具。 示例：电子书。
  • 教育和培训：计算机辅助学习材料可以使用多媒体资源进行设计，这些资源是当今非常受欢迎的学习资源。 示例：数字图书馆。
  • 营销，广告，零售，娱乐和旅行。 示例：虚拟城市之旅。
  • 实时控制和监控：结合主动数据库技术，多媒体信息显示可以非常有效地监控复杂任务示例：制造运行控制。
