Canonical Cover


每当用户更新数据库时，系统必须检查在此过程中是否违反了任何功能依赖性。 如果在新数据库状态中存在违反依赖项的情况，则系统必须回滚。 使用大量功能依赖项可能会导致不必要的额外计算时间。 这是规范封面发挥作用的地方。
一组功能依赖关系F的规范封面是一组简化的函数依赖关系，它们与原始集F具有相同的闭包。

重要定义：

无关属性：如果我们可以在不改变函数依赖集的闭包的情况下删除它，那么函数依赖的属性被认为是无关的。
规范封面：规范封面 一组函数依赖关系F，以满足以下所有属性：

  • F逻辑上暗示了所有依赖关系 。
  • 逻辑上暗示了F中的所有依赖关系。
  • 没有功能依赖 包含一个无关的属性。
  • 功能依赖的每个左侧 是独特的。 也就是说，没有两个依赖项 和 就是这样 。

寻找规范封面

计算集合F的规范覆盖的算法：

    2. Find a functional dependency  with an

    3. If an extraneous attribute is found, delete it from  .

例1：

考虑以下功能依赖性集F ：

F = {
一个 公元前
乙 C
一个 乙
AB C
}

寻找规范封面的步骤：

  1. 左侧有两个具有相同属性集的函数依赖项：
    一个 公元前
    一个 乙

    这两个可以结合得到
    一个 公元前。

    现在，修订后的集F成为：

    F = {
    一个 公元前
    乙 C
    AB C
    }

  2. AB中有一个无关的属性 C因为即使在删除AB之后 从集合F中得到C，我们得到相同的闭包。 这是因为B. C已经是F的一部分。

    现在，修订后的集F成为：

    F = {
    一个 公元前
    乙 C
    }

  3. C是A中的无关属性 BC，也是A. A在逻辑上暗示B. B和B. C（通过传递性）。

    F = {
    一个 乙
    乙 C
    }

  4. 在此步骤之后，F不再更改。 所以，
    因此，所需的规范封面是，
    = {
    一个 乙
    乙 C
    }

例2：

考虑另一组函数依赖关系：

F = {
一个 公元前
光盘 Ë
乙 d
Ë 一个
}

  1. F中每个功能依赖的左侧是唯一的。
  2. 任何函数依赖项的左侧或右侧的属性都不是无关的（通过在每个函数依赖项上应用无关属性的定义来检查）。
  3. 因此，规范的封面 等于F.

注意：可以有多个规范封面 功能依赖的集合F.

如何检查一组fd的F是否能够正式覆盖另一组fd的G？

考虑以下两组函数依赖项：

F = {
一个 乙
AB C
d AC
d Ë
}

G = {
d 公元前
d AB
}

现在，我们需要找出这些fd中的一个是否规范地涵盖了另一组fd。 这意味着，我们需要找出F是否规范地覆盖了G，G是否规范地覆盖了F，或者两者都没有规范地覆盖另一个。

为了找到答案，我们遵循以下步骤：

  • 创建单身右手边。 这意味着，fd箭头右侧的属性应该都是单例。
    功能依赖性D. AC被分解为两个功能依赖，D A和D. C。

    F = {
    一个 乙
    AB C
    d 一个
    d C
    d Ë
    }

  • 删除所有无关的属性。

    考虑任何功能依赖性XY Z.如果X本身可以确定Z，则属性Y是无关的并且可以被移除。 正如我们所看到的，只有在LHS中存在多个属性的那些功能依赖项中才可能出现无关属性。

    因此，考虑功能依赖性AB C。
    现在，我们必须找到A和B的闭包，以确定这些是否是无关的。

    = AB
    = B

    我们可以看到，B可以从A确定。这意味着我们可以从功能依赖AB中删除B. C。

    F = {
    一个 乙
    一个 C
    d 一个
    d C
    d Ë
    }

  • 删除所有冗余功能依赖项。

    逐个检查所有fd，看看是否删除了fd X. Y，我们仍然可以通过其他一些fd从X中找到Y.更正式的方式来说明这是找到的 没有使用我们正在测试的fd并检查Y是否是闭包的一部分。 如果是，则fd是多余的。

    在这里，检查fd D时 C，我们观察到即使在隐藏它之后，D的闭包含有C.这是因为我们可以通过两个其他fd的D的组合从D获得C. A和A. C.所以， C是多余的。

    F = {
    一个 乙
    一个 C
    d 一个
    d Ë
    }

现在，对G做同样的事情。

  • 创建单身右手边。 这意味着，fd箭头右侧的属性应该都是单例。

    G = {
    一个 乙
    一个 C
    d 一个
    d 乙
    }

  • 删除所有无关的属性。
    由于所有fd的RHS只包含1个属性，因此不存在无关属性。

  • 删除所有冗余功能依赖项。

    通过循环遍历所有fd并在所有情况下检查LHS的闭包，我们观察到fd D. B是多余的，因为它可以通过2个其他fd，D的组合获得 A和A. B.

    G = {
    一个 乙
    一个 C
    d 一个
    }

现在，由于G的所有fd已经包含在F中，我们得出结论，F涵盖了G.
