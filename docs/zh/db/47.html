并发控制 - 简介

并发控制处理多个事务的交叉执行 。 在下一篇文章中，我们将看到什么是可序列化以及如何查找计划是否可序列化。

什么是交易？

一组逻辑相关的操作称为事务。 交易的主要操作是：

读（A）：读操作读（A）或R（A）从数据库中读取A的值并将其存储在主存储器的缓冲区中。

写（A）：写操作写（A）或W（A）将值从缓冲区写回数据库。



让我们从一个账户进行借记交易，该账户包括以下操作：

  1. R（A）;
  2. A = A-1000;
  3. W（A）;

假设交易开始前A的值为5000。

  • 第一个操作从数据库中读取A的值并将其存储在缓冲区中。
  • 第二次操作将其值减少1000.因此缓冲区将包含4000。
  • 第三个操作会将值从缓冲区写入数据库。 所以A的最终值将是4000。

但是，执行某些操作后，事务也可能会失败。 失败可能是由于硬件，软件或电源等。例如，如果上面讨论的借记交易在执行操作2之后失败，则A的值将在数据库中保持5000，这是银行不能接受的。 为避免这种情况，Database有两个重要的操作：

提交：成功执行事务的所有指令后，事务所做的更改将在数据库中永久执行。

回滚：如果事务无法成功执行所有操作，则撤消事务所做的所有更改。

交易的属性

原子性：由于事务是逻辑上相关的操作集， 所以它们都应该执行或者不执行 。 上面讨论的借记交易应该执行所有三个操作或不执行。 如果在执行操作1和2之后借记交易失败，则其新值4000将不会在数据库中更新，这导致不一致。

一致性：如果同一帐户上的借方和贷方交易的操作同时执行，则可能使数据库处于不一致状态。

  • 例如，T1（从A的Rs.1000借记）和T2（500到A的信用）同时执行，数据库达到不一致状态。
  • 让我们假设A的账户余额是卢比。 5000.T1读取A（5000）并将值存储在其本地缓冲区空间中。 然后T2读取A（5000）并将值存储在其本地缓冲区空间中。
  • T1执行A = A-1000（5000-1000 = 4000）并且4000存储在T1缓冲空间中。 然后T2执行A = A + 500（5000 + 500 = 5500）并且5500存储在T2缓冲空间中。 T1将其缓冲区中的值写回数据库。
  • A的值在数据库中更新为4000，然后T2将其缓冲区中的值写回数据库。 A的值更新为5500，表明借记交易的影响已丢失且数据库变得不一致。

隔离：在提交事务之前，其他人不应该看到事务的结果。 例如，我们假设A的余额是Rs。 5000和T1借记卢比。 来自A的1000.A的新余额将是4000.如果T2信用卢比。 500到A的新余额，A将变为4500，此后T1失败。 然后我们也必须回滚T2，因为它使用T1产生的值。 因此，在提交之前，事务结果不会对其他事务可见。

持久：一旦数据库提交了事务，事务所做的更改应该是永久性的。 例如; 如果某人已将50万美元记入其帐户，则银行不能说更新已丢失。 为避免此问题，数据库的多个副本存储在不同的位置。

什么是时间表？

时间表是来自一个或多个事务的一系列操作。 时间表可以有两种类型：

  • 串行调度：当一个事务在启动另一个事务之前完全执行时，该调度称为串行调度。 连续计划始终保持一致。 例如; 如果时间表S具有借记交易T1和信用交易T2，则可能的串行调度是T1，然后是T2（T1-> T2）或T2后跟T1（（T1-> T2）。串行调度具有低吞吐量和较少的资源利用率。
  • 并发计划：当事务的操作与计划的其他事务的操作交错时，该计划称为并发计划。 例如; 表1中所示的借方和贷方交易表是并行的。 但并发可能导致数据库不一致。 上面的并发调度示例也不一致。
