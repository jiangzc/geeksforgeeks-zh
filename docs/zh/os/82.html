内存管理 - 将虚拟地址映射到物理地址

内存由大量的单词或数组组成，每个单词或数组都有与之关联的地址。 现在CPU的工作是从基于存储器的程序计数器获取指令。 现在进一步这些指令可能导致加载或存储到特定的存储器地址。

地址绑定是从一个地址空间映射到另一个地址空间的过程。 逻辑地址是CPU在执行期间生成的地址，而物理地址是指内存单元中的位置（加载到内存中的位置）。注意用户只处理逻辑地址（虚拟地址）。 逻辑地址尤其由MMU或地址转换单元进行转换。 此过程的输出是适当的物理地址或RAM中的代码/数据的位置。

地址绑定可以通过三种不同的方式完成：

编译时间 -它的工作原理是生成逻辑地址（也称为虚拟地址）。 如果您知道在编译期间进程将驻留在内存中，则会生成绝对地址。

加载时间 -它将生成物理地址。如果在编译时不知道进程将驻留在何处，则将生成可重定位地址。 在此if地址更改，然后我们需要重新加载用户代码。



执行时间 -它有助于区分物理和逻辑地址。如果在执行期间可以将进程从一个内存移动到另一个内存（动态链接 - 在加载或运行时完成的链接），则使用此选项。

MMU（内存管理单元） -
虚拟地址和物理地址之间的运行时映射由称为MMU的硬件设备完成。

在内存管理中，操作系统将处理进程并在磁盘和内存之间移动进程以便执行。 它保留了可用和已用内存的轨迹。

指令执行周期遵循以下步骤：

  1. 从存储器中取出第一条指令，例如ADD A，B
  2. 然后解码这些指令，即A和B的加法
  3. 并且进一步加载或存储在某个特定的存储位置。

基本硬件

由于主存储器和寄存器内置于处理器中，CPU只能访问这些存储器。因此每条指令都应写入直接存取存储器
设备。

  1. 如果来自寄存器的CPU访问指令则可以在一个CPU时钟周期内完成，因为寄存器内置在CPU中。
  2. 如果指令驻留在主存储器中，那么它将通过存储器总线访问，这将花费大量时间。 因此，可以在CPU和主存储器之间添加快速内存，即为事务添加缓存。
  3. 现在我们应该确保该流程属于合法地址。
  4. 合法地址包括基址寄存器（保存最小物理地址）和限制寄存器（范围大小）。

例如： 

 Base register = 300040
limit register = 120900 
then legal address = (300040+120900)= 420940(inclusive).
legal address = base register+ limit register

进程如何从磁盘映射到内存

  1. 通常进程以二进制可执行文件的形式驻留在磁盘中。
  2. 因此，要执行进程，它应该驻留在主内存中。
  3. 根据正在使用的内存管理将进程从磁盘移动到内存。
  4. 进程以就绪队列的形式在磁盘中等待以获取内存。

磁盘和内存映射的过程

正常过程是从输入队列中选择进程并加载到内存中。 当进程执行时，它从内存访问数据和指令，一旦完成，它将释放内存，现在内存可用于其他进程。

MMU计划 - 

 CPU------- MMU------Memory

  1. CPU将生成例如：346的逻辑地址
  2. MMU将生成重定位寄存器（基址寄存器），例如：14000
  3. 在内存物理地址位于例如：（346 + 14000 = 14346）
