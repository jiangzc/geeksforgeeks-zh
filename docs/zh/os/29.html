具有预测的突发时间的最短作业优先调度

先决条件 - CPU调度 ， SJF - 设置1（非抢先） ， 设置2（抢占式）

最短作业优先（SJF）是一种最优调度算法，因为它提供最大吞吐量和最小平均等待时间（WT）和周转时间（TAT），但它实际上不可实现，因为进程的突发时间无法预测提前。

我们可能不知道下一个CPU突发的长度，但我们可能能够预测其值。 我们预计下一次CPU突发的长度与之前的相似。 通过计算下一个CPU突发长度的近似值，我们可以选择具有最短预测CPU突发的进程。

有两种方法可以预测过程的突发时间：

1.静态方法 -我们可以通过两个因素来预测Burst-Time：



  • 工艺规模 -
    假设我们已经执行了具有200 KB大小的进程P old ，并且其突发时间是20个单位时间，现在假设我们有一个新进程P new，其大小为201 KB，尚未执行。
    我们采用已经执行的过程P old的 Burst-Time，其几乎与New Process的大小相同，作为New Process P new的 Burst-Time。
  • 流程类型 -
    我们可以根据流程类型预测Burst-Time。 操作系统进程（如调度程序，调度程序，分段，碎片）比用户进程（游戏，应用程序软件）更快。 任何新操作系统进程的突发时间都可以从类似类型的任何旧操作系统进程预测，并且对于用户进程也是如此。

    • 注 -突发时间预测的静态方法不可靠，因为它总是无法正确预测。

    2.动态方法 -令t i是第i 个过程的实际突发时间，并且Τn + 1是第n + 1 个过程的预测突发时间。

    • 简单平均 -给定n个过程（P 1 ，P 2 ... P n ） 
       Τ n+1 = 1/n(Σ i=1 to n t i )
    • 指数平均值（老化） - 
       Τ n+1 = αt n + (1 - α)Τ n

      其中α=是平滑因子，0 <=α<= 1，

      t n = 第 n 个过程的实际突发时间，
      Τn= 第 n 个过程的预测突发时间。

      一般术语， 

       αt n + (1 - α)αt n-1 + (1 - α) 2 αt n-2 ...+ (1 - α) j αt nj ...+ (1 - α) n+1 Τ 0

      Τ0是恒定或整体系统平均值。

    平滑因子（α） -它在我们的预测中控制近期和过去历史的相对权重。

    • 如果α= 0，则Τn + 1 =Τn，即初始预测突发时间的值没有变化。
    • 如果α= 1，则Τn + 1 = t n即，预测的新过程的突发时间将始终根据第 n 个过程的实际突发时间而改变。
    • 如果α= 1/2，则近期和过去的历史具有相同的权重。

    示例 -
    计算指数平均值，T1 = 10，α= 0.5，算法为SJF，先前运行为8,7,4,16。
    （a）9
    （b）8
    （c）7.5
    （d）没有

    说明：
    最初T1 = 10且α= 0.5，给定的运行时间为8,7,4,16，因为它是最短的工作，
    因此，由于SJF是一种非先发制人的技术，因此这些过程可能的顺序将是4,7,8,16。
    因此，使用公式：T2 =α* t1 +（1-α）T1
    所以我们有，
    T2 = 0.5 * 4 + 0.5 * 10 = 7，这里t1 = 4且T1 = 10
    T3 = 0.5 * 7 + 0.5 * 7 = 7，这里t1 = 7且T1 = 7
    T4 = 0.5 * 8 + 0.5 * 7 = 7.5，这里t1 = 8且T1 = 7
    因此，第四个过程的未来预测将是T4 = 7.5，这是选项（c）。
