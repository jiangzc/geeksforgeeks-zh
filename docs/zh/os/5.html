多任务，多线程和多处理之间的区别

  1. 多道程序设计 -一次运行多个程序的计算机（如同时运行Excel和Firefox）。
  2. 多处理 -一次使用多个CPU的计算机。
  3. 多任务处理 -共享公共资源的任务（如1个CPU）。
  4. 多线程是多任务处理的扩展。

1.多编程 -

在现代计算系统中，通常存在几个想要执行的并发应用程序进程。 现在，操作系统有责任有效地管理所有流程。
操作系统最重要的一个方面是多程序。
在计算机系统中，有多个进程等待执行，即它们在CPU被分配给它们并且它们开始执行时正在等待。 这些过程也称为作业。 现在主内存太小，无法容纳所有这些进程或作业。 因此，这些过程最初保存在称为作业池的区域中。 此作业池包含等待主内存和CPU分配的所有进程。
CPU从所有这些等待作业中选择一个作业，将其从作业池带到主存储器并开始执行。 处理器执行一个作业，直到它被某个外部因素中断或者进入I / O任务。

非多程序系统的工作 -

  • 在非多编程系统中，只要一个作业离开CPU并执行其他任务（例如I / O），CPU就会变为空闲状态。 CPU保持等待并等待，直到此作业（之前执行）返回并继续执行CPU。 所以这一切CPU仍然是免费的。
  • 现在它的缺点是CPU在很长一段时间内都处于空闲状态。 此外，等待执行的其他作业可能无法执行，因为CPU仍然分配给较早的作业。
    这带来了一个非常严重的问题，即使其他作业已准备好执行，CPU也不会分配给它们，因为CPU被分配给甚至没有使用它的作业（因为它在I / O任务中很忙）。
  • 一个作业在1小时内使用CPU而其他作业在队列中等待5个小时是不可能的。 为了避免这种情况并提出有效利用CPU，出现了多编程的概念。

多编程的主要思想是最大化CPU时间。
多程序系统的工作 -

  • 在多程序系统中，只要一个作业执行I / O任务，操作系统就会中断该作业，从作业池中选择另一个作业（等待队列），为此新作业提供CPU并开始执行。 上一个作业继续执行其I / O操作，而这个新作业执行CPU绑定任务。 现在说第二个工作也用于I / O任务，CPU选择第三个工作并开始执行它。 一旦作业完成其I / O操作并返回CPU任务，就会为其分配CPU。
  • 这样，等待I / O任务完成的系统不会浪费CPU时间。
    因此，多编程的最终目标是只要有准备执行的进程就使CPU保持忙碌状态。 这样，通过一次执行程序的一部分，之后执行另一程序的一部分，然后执行另一程序的一部分等，可以在单个处理器上执行多个程序，从而执行多个程序。 因此，CPU永远不会闲置。

2.多处理 -

在单处理器系统中，一次只执行一个进程。
多处理是在单个计算机系统中使用两个或多个CPU（处理器）。 该术语还指系统在单个计算机系统内支持多个处理器的能力。 现在，由于有多个处理器可用，因此可以一次执行多个进程。 这些多处理器共享计算机总线，有时也包括时钟，存储器和外围设备。

多处理系统的工作 -

  • 在多处理的帮助下，可以同时执行许多过程。 假设进程P1，P2，P3和P4正在等待执行。 现在在单处理器系统中，首先执行一个进程，然后执行另一个进程，然后执行另一个进程，依此类推。
  • 但是通过多处理，可以将每个进程分配给不同的处理器以执行它。 如果它是一个双核处理器（2个处理器），两个进程可以同时执行，因此速度将快两倍，同样四核处理器的速度是单个处理器的四倍。

为何使用多处理 -

  • 多处理器系统的主要优点是可以在更短的时间内完成更多的工作。 当需要非常高的速度来处理大量数据时，使用这些类型的系统。 与单处理器系统相比，多处理系统可以节省资金，因为处理器可以共享外围设备和电源。
  • 它还提供了更高的可靠性，即如果一个处理器发生故障，工作不会停止，它只会减慢速度。 例如，如果我们有10个处理器而1个失败，则工作不会停止，而其余9个处理器可以共享第10个处理器的工作。 因此整个系统的运行速度只有10％，而不是完全失败。

多处理是指硬件（即CPU单元）而不是软件（即运行进程）。 如果底层硬件提供多个处理器，那么这就是多处理。

多编程和多处理之间的区别 -

  • 通过使多个程序同时运行并通过具有多个物理处理器进行多处理，可以对系统进行多程序编程。 多处理和多编程之间的区别在于多处理基本上是在多个处理器上同时执行多个进程，而多编程是将多个程序保存在主存储器中并仅使用单个CPU同时执行它们。
  • 通过并行处理进行多处理，而通过从一个处理切换到另一个处理（称为上下文切换的现象）来进行多编程。

3.多任务处理 -

正如名称本身所暗示的，多任务是指一次执行多个任务（例如进程，程序，线程等）。 在现代操作系统中，我们能够播放MP3音乐，在Microsoft Word中编辑文档，同时浏览Google Chrome，这是通过多任务处理完成的。

多任务处理是多编程的逻辑扩展。 多任务处理与多编程不同的主要方式是多编程工作在上下文切换的概念上，而多任务处理仅基于时间共享系统。

多任务系统的工作 -

  • 在时间共享系统中，为每个进程分配一些特定的时间量，一个进程要执行该时间量。 假设有4个进程P1，P2，P3，P4准备执行。 因此，为它们中的每一个分配一些时间量，它们将被执行，例如5纳秒（5ns）的时间量子。 当一个进程开始执行（比如P2）时，它会执行该时间量程（5 ns）。 在5 ns之后，CPU开始执行另一个进程（比如P3）达到指定的时间量。
  • 因此，CPU使进程在它们之间共享时间片并相应地执行。 一旦一个进程的时间量到期，另一个进程就会开始执行。
  • 这里也基本上发生了上下文切换，但它发生得如此之快，以至于用户能够在每个程序运行时分别与每个程序进行交互。 这样，给予用户同时执行多个进程/任务的错觉。 但实际上只有一个进程/任务在特定时刻执行。 在多任务处理中，时间共享最好地体现出来，因为每个运行的进程只占用了相当长的CPU时间。

在更一般的意义上，多任务是指具有多个程序，进程，任务，线程同时运行。 当多个任务共享公共处理资源（例如，CPU和内存）时，该术语用于现代操作系统。

  • 如上图所示，在任何时候CPU只执行一项任务而其他任务正在等待轮到他们。 当CPU被重新分配给另一个任务时，实现了并行性的错觉。 即由于时间共享，所有三个任务A，B和C似乎同时出现。
  • 因此，对于多任务处理，首先应该进行多道程序设计，即存在准备执行的多个程序。 其次是分时的概念。

4.多线程 -

线程是CPU利用率的基本单位。 多线程是一种执行模型，它允许单个进程在该进程的“上下文”中同时运行多个代码段（即线程）。
例如，VLC媒体播放器，其中一个线程用于打开VLC媒体播放器，一个线程用于播放特定歌曲，另一个线程用于向播放列表添加新歌曲。

多线程是一个进程能够一次管理多个用户的使用，并由同一个用户管理多个请求而无需拥有该程序的多个副本。

多线程系统的工作 -

例1 -

  • 假设有一个处理客户端请求的Web服务器。 现在，如果它作为单线程进程执行，那么它将无法一次处理多个请求。 首先，一个客户端将发出请求并完成其执行，然后，服务器将能够处理另一个客户端请求。 这是非常昂贵，耗时且累人的任务。 为避免这种情况，可以使用多线程。
  • 现在，每当有新的客户端请求进入时，Web服务器只需创建一个新线程来处理此请求并恢复其执行以听取更多客户端请求。 因此，Web服务器的任务是侦听新的客户端请求并为每个单独的请求创建线程。 每个新创建的线程处理一个客户端请求，从而减轻Web服务器的负担。

例2 -

  • 我们可以将线程视为共享父进程资源但独立执行的子进程。 现在以GUI为例。 假设我们正在GUI上执行计算（这需要很长时间才能完成）。 现在，在此命令完成执行之前，我们无法与GUI的其余部分进行交互。 为了能够与GUI的其余部分进行交互，应将此计算命令分配给单独的线程。 所以在这个时间点，将执行2个线程，即一个用于计算，一个用于GUI的其余部分。 因此，在单个过程中，我们使用多个线程来实现多种功能。

多线程的优点 -

  • 多线程的好处包括提高响应能力。 由于程序中有多个线程，因此如果一个线程执行时间过长或者被阻塞，则其余线程会继续执行而没有任何问题。 因此，整个程序通过剩余的线程保持对用户的响应。
  • 多线程的另一个优点是成本更低。 创建全新的流程和分配资源是一项耗时的任务，但由于线程共享父流程的资源，因此创建线程并在它们之间切换相对容易。 因此，多线程是现代操作系统的需要。
