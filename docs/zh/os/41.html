进程同步| 介绍

在同步的基础上，进程被分类为以下两种类型之一：

  • 独立流程 ：执行一个流程不会影响其他流程的执行。
  • 协作过程 ：执行一个过程会影响其他过程的执行。

在协作过程的情况下出现过程同步问题也是因为资源在协作过程中共享。

关键部分问题

关键部分是一个代码段，一次只能由一个进程访问。 关键部分包含需要同步的共享变量，以保持数据变量的一致性。

在条目部分，进程请求在关键部分中输入。

临界区问题的任何解决方案都必须满足三个要求：

  • 相互排斥 ：如果进程在其关键部分中执行，则不允许在临界区中执行其他进程。
  • 进度 ：如果临界区中没有进程，则外部没有其他进程可以阻止进入临界区。
  • 有界等待 ：在进程请求进入其临界区并且在该请求被授予之前，允许其他进程进入其关键部分的次数必须存在一个限制。


彼得森的解决方案
Peterson的解决方案是针对关键部分问题的经典软件解决方案。



在Peterson的解决方案中，我们有两个共享变量：

  • boolean flag [i]：初始化为FALSE，最初没有人有兴趣进入临界区
  • int turn：进入临界区的进程。



彼得森的解决方案保留了所有三个条件：

  • 确保相互排除，因为任何时候只有一个进程可以访问关键部分。
  • 由于关键部分之外的过程不会阻止其他过程进入关键部分，因此也确保了进展。
  • 有限的等待被保留，因为每个过程都有一个公平的机会。


    • 彼得森解决方案的缺点

    • 它涉及忙碌的等待
    • 它仅限于2个流程。

    检查并设置
    TestAndSet是同步问题的硬件解决方案。 在TestAndSet中，我们有一个共享锁变量，它可以取两个值0或1中的任何一个。 

     0 Unlock
1 Lock

    在进入关键部分之前，进程会查询锁定。 如果它被锁定，它会一直等待直到它被释放，如果它没有被锁定，它就会锁定并执行关键部分。

    在TestAndSet中，保留了相互排除和进度，但无法保留有限等待。

    信号灯

    信号量是一个整数变量，只能通过wait（）和signal（）两个操作来访问。
    信号量有两种类型：二进制信号量和计数信号量

    • 二进制信号量：它们只能是0或1.它们也称为互斥锁，因为锁可以提供互斥。 所有进程可以共享初始化为1的相同互斥信号量。然后，进程必须等到锁变为0.然后，进程可以使互斥信号量1并启动其临界区。 当它完成其关键部分时，它可以将互斥量信号量的值重置为0，其他一些进程可以进入其临界区。
    • 计数信号量：它们可以具有任何值，并且不受某个域的限制。 它们可用于控制对同时访问次数有限制的资源的访问。 信号量可以初始化为资源的实例数。 每当进程想要使用该资源时，它会检查剩余实例的数量是否大于零，即进程是否有可用的实例。 然后，进程可以进入其关键部分，从而将计数信号量的值减少1.在使用资源实例的过程结束后，它可以离开临界区，从而将可用实例的数量加1资源。
