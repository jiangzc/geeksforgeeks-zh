操作系统中的车队效应

先决条件： FCFS调度的基础知识（FCFS调度程序|第1组 ， FCFS调度程序|第2组 ）

Convoy Effect是与先到先服务（FCFS）算法相关的现象，其中整个操作系统由于很少的慢进程而减慢。

FCFS算法本质上是非抢占式的，也就是说，一旦CPU时间被分配给进程，其他进程只有在当前进程完成后才能获得CPU时间。 FCFS调度的这种属性导致称为Convoy Effect的情况。

假设就绪队列中存在一个CPU密集型（大突发时间）进程，并且其他几个进程具有相对较少的突发时间但是输入/输出（I / O）绑定（经常需要I / O操作）。



然后发生以下情况 -

  • I / O绑定进程首先分配CPU时间。 由于它们的CPU密集度较低，因此可以快速执行，然后转到I / O队列。
  • 现在，CPU密集型进程被分配了CPU时间。 由于其突发时间很长，因此需要一段时间才能完成。
  • 在执行CPU密集型进程时，I / O绑定进程完成其I / O操作并移回就绪队列。
  • 但是，由于CPU密集型进程仍未完成，因此I / O绑定进程会等待。 这导致I / O设备空闲。
  • 当CPU密集型进程结束时，它会被发送到I / O队列，以便它可以访问I / O设备。
  • 同时，I / O绑定进程获得所需的CPU时间并返回I / O队列。
  • 但是，它们会等待，因为CPU密集型进程仍在访问I / O设备。 结果， CPU现在处于空闲状态 。

因此，在Convoy效果中，一个缓慢的过程会降低整个过程集的性能，并导致CPU时间和其他设备的浪费。

为了避免Convoy效应，可以使用诸如循环调度之类的抢占式调度算法 - 因为较小的进程不必等待CPU时间 - 使得执行速度更快并且导致更少的资源闲置。
