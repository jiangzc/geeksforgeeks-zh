文件目录：
文件集是一个文件目录。 该目录包含有关文件的信息，包括属性，位置和所有权。 大部分此类信息（尤其是与存储有关的信息）由操作系统管理。 该目录本身就是一个文件，可以通过各种文件管理例程访问。

设备目录中包含的信息是：

  • 名称
  • 类型
  • 地址
  • 当前长度
  • 最大长度
  • 上次访问日期
  • 上次更新日期
  • 所有者ID
  • 保护信息

在目录上执行的操作是：

  • 搜索文件
  • 创建一个文件
  • 删除文件
  • 列出目录
  • 重命名文件
  • 遍历文件系统

维护目录的优点是：

  • 效率：文件可以更快地定位。
  • 命名：对于用户来说很方便，因为两个用户对于不同的文件可以具有相同的名称，或者对于同一文件可以具有不同的名称。
  • 分组：文件的逻辑分组可以通过属性完成，例如所有java程序，所有游戏等。

单级目录
在此，为所有用户维护单个目录。



  • 命名问题：用户的两个文件名称不能相同。
  • 分组问题：用户无法根据需要对文件进行分组 。



两级目录
在这个单独的目录中维护每个用户。

  • 路径名：由于两个级别，每个文件都有一个路径名来查找该文件。
  • 现在，我们可以为不同的用户提供相同的文件名。
  • 搜索在这种方法中是有效的。



树状结构目录：
目录以树的形式维护。 搜索效率很高，并且还有分组功能。 我们有文件的绝对路径名或相对路径名。


文件分配方法
1.连续分配：在创建文件时，将一个连续的块集分配给文件。 因此，这是使用可变大小部分的预分配策略。 文件分配表只需要每个文件的一个条目，显示起始块和文件的长度。 从单个顺序文件的角度来看，这种方法是最好的。 可以一次读入多个块以提高顺序处理的I / O性能。 检索单个块也很容易。 例如，如果文件在块b处开始，并且需要文件的第i个块，则其在辅助存储器上的位置仅为b + i-1。


坏处

  • 将发生外部碎片，使得难以找到足够长度的连续空间块。 需要压缩算法来释放磁盘上的额外空间。
  • 此外，通过预分配，有必要在创建时声明文件的大小。

2.链接分配（非连续分配）：分配基于单个块。 每个块包含指向链中下一个块的指针。 同样，文件表只需要每个文件的一个条目，显示起始块和文件的长度。 尽管可以进行预分配，但更简单的是根据需要分配块。 任何空闲块都可以添加到链中。 块不需要是连续的。 如果可用空闲磁盘块，则始终可以增加文件大小。 没有外部碎片，因为一次只需要一个块，但可能存在内部碎片，但它只存在于文件的最后一个磁盘块中。

坏处：

  • 内部碎片存在于文件的最后一个磁盘块中。
  • 在每个磁盘块中维护指针都有开销。
  • 如果任何磁盘块的指针丢失，该文件将被截断。
  • 它仅支持文件的顺序访问。

3.索引分配：
它解决了连续和链式分配的许多问题。 在这种情况下，文件分配表包含每个文件的单独的一级索引：索引为分配给该文件的每个块都有一个条目。 分配可以基于固定大小的块或可变大小的块。 按块分配可消除外部碎片，而可变大小块的分配可改善局部性。 这种分配技术支持对文件的顺序和直接访问，因此是最流行的文件分配形式。

磁盘可用空间管理

正如必须管理分配给文件的空间一样，因此必须管理当前未分配给任何文件的空间。 要执行任何文件分配技术，必须知道磁盘上有哪些块可用。 因此，除文件分配表外，我们还需要磁盘分配表。以下是用于可用空间管理的方法。

  1. 位表 ：此方法使用包含磁盘上每个块的一位的向量。 0的每个条目对应于空闲块，并且每个条目对应于使用中的块。
    例如：00011010111100110001

    在该向量中，每个位对应于特定向量，0表示该特定块是空闲的，1意味着该块已被占用。 位表的优点在于，找到一个或一组连续的空闲块相对容易。 因此，位表适用于任何文件分配方法。 另一个优点是它尽可能小。

  2. 空闲阻止列表 ：在此方法中，每个块按顺序分配一个数字，并且所有空闲块的编号列表保留在磁盘的保留块中。
