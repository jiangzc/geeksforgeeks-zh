磁盘调度算法

磁盘调度由操作系统完成，以调度到达磁盘的I / O请求。 磁盘调度也称为I / O调度。

磁盘调度很重要，因为：

  • 多个I / O请求可能由不同的进程到达，并且磁盘控制器一次只能提供一个I / O请求。 因此，其他I / O请求需要在等待队列中等待并需要进行调度。
  • 两个或多个请求可能彼此相距很远，因此可能导致更大的磁盘臂移动。
  • 硬盘驱动器是计算机系统中最慢的部分之一，因此需要以有效的方式访问。

有许多磁盘调度算法，但在讨论它们之前，让我们快速浏览一些重要的术语：

  • 寻道时间 ：寻道时间是将磁盘臂定位到要读取或写入数据的指定磁道所需的时间。 因此，提供最小平均寻道时间的磁盘调度算法更好。
  • 旋转延迟：旋转延迟是所需的磁盘扇区旋转到一个位置所需的时间，以便它可以访问读/写磁头。 因此，提供最小旋转延迟的磁盘调度算法更好。
  • 传输时间：传输时间是传输数据的时间。 它取决于磁盘的转速和要传输的字节数。
  • 磁盘访问时间：磁盘访问时间为： 
             
      Disk Access Time = Seek Time + 
                         Rotational Latency + 
                         Transfer Time

  • 磁盘响应时间：响应时间是等待执行其I / O操作的请求所花费的时间的平均值。 平均响应时间是所有请求的响应时间。 方差响应时间是衡量个人请求在平均响应时间方面的服务方式。 因此，给出最小方差响应时间的磁盘调度算法更好。

磁盘调度算法



  1. FCFS： FCFS是所有磁盘调度算法中最简单的。 在FCFS中，请求按照它们到达磁盘队列的顺序进行寻址。

优点：

  • 每个请求都有一个公平的机会
  • 没有无限期的延期

缺点：

  • 不试图优化寻道时间
  • 可能无法提供最好的服务
  1. SSTF：在SSTF（最短寻找时间优先）中，首先执行具有最短寻道时间的请求。 因此，每个请求的搜索时间在队列中预先计算，然后根据它们计算的搜索时间进行调度。 结果，磁盘臂附近的请求将首先执行。 SSTF肯定是对FCFS的改进，因为它减少了平均响应时间并增加了系统的吞吐量。

优点：

  • 平均响应时间减少
  • 吞吐量增加

缺点：

  • 开销以提前计算寻道时间
  • 如果请求与传入请求相比具有更高的搜索时间，则可能导致请求饥饿
  • 由于SSTF仅支持某些请求，因此响应时间差异很大
  1. 扫描：在SCAN算法中，磁盘臂移动到特定方向并为其路径中的请求提供服务，在到达磁盘末尾后，它会反转其方向并再次为到达其路径的请求提供服务。 因此，该算法就像电梯一样工作，因此也称为电梯算法。 结果，中端的请求得到了更多的服务，而那些到达磁盘臂后面的请求将不得不等待。

优点：

  • 高吞吐量
  • 响应时间差异很小
  • 平均响应时间

缺点：

  • 请求刚刚访问磁盘臂的位置的漫长等待时间
  1. CSCAN ：在SCAN算法中，磁盘臂在反转其方向后再次扫描已扫描的路径。 因此，可能有太多请求在另一端等待，或者在扫描区域可能有零或几个未决请求。

在CSAN算法中避免了这些情况，其中磁盘臂而不是反转其方向到达磁盘的另一端并开始为那里的请求提供服务。 因此，磁盘臂以圆形方式移动，并且该算法也类似于SCAN算法，因此它被称为C-SCAN（循环扫描）。

优点：

  • 与SCAN相比，提供更均匀的等待时间
  1. 看起来：它类似于SCAN磁盘调度算法，不同之处在于磁盘臂尽管到达磁盘末端只能到达头部前面的最后一个请求，然后才从那里反转它的方向。 因此，它可以防止由于不必要的遍历到磁盘末端而发生的额外延迟。
  1. CLOOK：由于LOOK类似于SCAN算法，类似地，CLOOK类似于CSCAN磁盘调度算法。 在CLOOK中，尽管走到最后，磁盘臂只能到达头部前面的最后一个请求，然后从那里进入另一端的最后一个请求。 因此，它还防止了由于不必要的遍历到磁盘末端而发生的额外延迟。

每种算法都以其自己的方式是唯一的。整体性能取决于请求的数量和类型。
