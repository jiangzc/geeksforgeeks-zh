操作系统的类型

操作系统执行所有基本任务，如管理文件，进程和内存。 因此，操作系统充当所有资源的管理者 ，即资源管理器 。 因此，操作系统成为用户和机器之间的接口。

操作系统的类型：一些广泛使用的操作系统如下 -

1.批量操作系统 -
此类操作系统不直接与计算机交互。 有一个操作员，它接受具有相同要求的类似工作并将它们分组。 运营商有责任对具有类似需求的作业进行分类。

批处理操作系统的优点：



  • 很难猜测或知道任何工作完成所需的时间。 批处理系统的处理器知道作业在队列中的时间
  • 多个用户可以共享批处理系统
  • 空闲时间批处理系统非常少
  • 在批处理系统中反复管理大型工作很容易

批处理操作系统的缺点：

  • 计算机操作员应该熟知批处理系统
  • 批处理系统很难调试
  • 这有点贵
  • 如果任何工作失败，其他工作将不得不等待未知时间

基于批处理的操作系统示例：薪资系统，银行对帐单等。

2.分时操作系统 -
每个任务都有一些时间来执行，以便所有任务顺利进行。 每个用户在使用单个系统时都会获得CPU时间。 这些系统也称为多任务系统。 任务可以来自单个用户，也可以来自不同的用户。 每个任务执行的时间称为量子。 在此时间间隔结束后，OS切换到下一个任务。

分时操作系统的优点：

  • 每项任务都有平等的机会
  • 减少重复软件的机会
  • CPU空闲时间可以减少

分时操作系统的缺点：

  • 可靠性问题
  • 必须要注意用户程序和数据的安全性和完整性
  • 数据通信问题

时间共享操作系统的示例包括： Multics，Unix等。

3.分布式操作系统 -
这些类型的操作系统是计算机技术领域的最新进展，并且在全世界范围内被广泛接受，并且也以很快的速度被广泛接受。 各种自主互连计算机使用共享通信网络彼此通信。 独立系统拥有自己的内存单元和CPU。 这些被称为松散耦合系统或分布式系统。 这些系统处理器的尺寸和功能不同。 使用这些类型的操作系统的主要好处是，一个用户总是可以访问其系统中实际不存在的文件或软件，但是在该网络内连接的其他系统上，即在其中启用远程访问在该网络中连接的设备。

分布式操作系统的优点：

  • 一个故障不会影响其他网络通信，因为所有系统彼此独立
  • 电子邮件提高了数据交换速度
  • 由于资源是共享的，因此计算速度非常快且持久
  • 主机上的负载减少
  • 这些系统易于扩展，因为许多系统可以轻松添加到网络中
  • 数据处理延迟减少

分布式操作系统的缺点：

  • 主网络故障将停止整个通信
  • 为了建立分布式系统，使用的语言尚未明确定义
  • 这些类型的系统不是很容易获得，因为它们非常昂贵。 不仅底层软件非常复杂而且还不能很好地理解

分布式操作系统的示例是 - LOCUS等。

4.网络操作系统 -
这些系统在服务器上运行，并提供管理数据，用户，组，安全性，应用程序和其他网络功能的功能。 这些类型的操作系统允许通过小型专用网络共享文件，打印机，安全性，应用程序和其他网络功能。 网络操作系统的另一个重要方面是所有用户都非常了解底层配置，网络中所有其他用户，他们各自的连接等等，这就是为什么这些计算机通常被称为紧耦合系统的原因 。

网络操作系统的优点：

  • 高度稳定的集中式服务器
  • 安全问题通过服务器处理
  • 新技术和硬件升级很容易集成到系统中
  • 可以从不同位置和系统类型远程访问服务器

网络操作系统的缺点：

  • 服务器成本很高
  • 用户必须依赖于大多数操作的中心位置
  • 需要定期维护和更新

网络操作系统的示例有： Microsoft Windows Server 2003，Microsoft Windows Server 2008，UNIX，Linux，Mac OS X，Novell NetWare和BSD等。

5.实时操作系统 -
这些类型的OS服务于实时系统。 处理和响应输入所需的时间间隔非常小。 此时间间隔称为响应时间 。

当导弹系统，空中交通管制系统，机器人等时间要求非常严格时，使用实时系统 。

两种类型的实时操作系统如下：

  • 硬实时系统：
    这些操作系统适用于时间限制非常严格的应用，即使是最短的延迟也是不可接受的。 这些系统专为挽救生命而设计，如自动降落伞或气囊，如果发生任何意外，需要随时提供。 在这些系统中几乎找不到虚拟内存。
  • 软实时系统：
    这些操作系统适用于时间约束不太严格的应用程序。

RTOS的优点：

  • 最大消耗：设备和系统的最大利用率，因此来自所有资源的更多输出
  • 任务转换：为这些系统中的转移任务分配的时间非常少。 例如，在旧系统中，将一个任务转移到另一个任务需要大约10微秒，而在最新系统中需要3微秒。
  • 专注于应用程序：专注于运行应用程序，而不是重视队列中的应用程序。
  • 嵌入式系统中的实时操作系统：由于程序的规模很小，RTOS也可以用于运输等嵌入式系统。
  • 无错误：这些类型的系统没有错误。
  • 内存分配：在这些类型的系统中最好管理内存分配。

RTOS的缺点：

  • 有限的任务：很少有任务同时运行，并且在少数应用程序中它们的集中度非常低，以避免错误。
  • 使用繁重的系统资源：有时系统资源不是很好，而且它们也很昂贵。
  • 复杂算法：算法非常复杂，设计师难以编写。
  • 设备驱动程序和中断信号：它需要特定的设备驱动程序和中断信号才能最早响应中断。
  • 线程优先级：设置线程优先级并不好，因为这些系统对切换任务的要求非常低。

    • 实时操作系统的示例有：科学实验，医学成像系统，工业控制系统，武器系统，机器人，空中交通管制系统等。
