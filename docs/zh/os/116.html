最后一分钟 - 操作系统

请在此处查看所有主题的最后一分钟注释。

操作系统：它是用户和计算机硬件之间的接口。

OS的类型：

  • 批处理OS：一组类似的作业存储在主存储器中以供执行。 只有在上一个作业的执行完成时，才会将作业分配给CPU。
  • 多道程序操作系统：主存储器由等待CPU时间的作业组成。 操作系统选择其中一个进程并为其分配CPU时间。 每当执行进程需要等待任何其他操作（如I / O）时，操作系统从作业队列中选择另一个进程并将其分配给CPU。 这样，CPU永远不会保持空闲状态，并且用户可以获得一次完成多个任务的风格。
  • 多任务操作系统：多任务操作系统结合了多道程序操作系统和CPU调度的优点，可在作业之间快速切换。 交换机非常快，用户可以在运行时与每个程序进行交互
  • 时间共享操作系统：时间共享系统需要与用户交互以指示操作系统执行各种任务。 操作系统以输出响应。 指令通常通过键盘等输入设备提供。
  • 实时操作系统：实时操作系统通常是为专用系统构建的，以在截止日期内完成特定的任务集。

主题
线程是一个轻量级进程，是CPU利用率的基本单位。 一个过程可以执行更多
通过包含多个线程，同时完成一项任务。

    • 线程有自己的程序计数器，寄存器组和堆栈
    • 线程与相同进程的其他线程共享代码段，数据段，文件和信号。

可以使用fork（）系统调用引入新线程或给定进程的子进程。 使用n个fork（）系统调用的进程生成2 n - 1个子进程。
有两种类型的线程：

      • 用户线程
      • 内核线程

示例：Java线程，POSIX线程。示例：Window Solaris。



用户级线程 内核级线程
用户线程由用户实现。 内核线程由OS实现。
操作系统无法识别用户级线程。 内核线程被OS识别。
用户线程的实现很容易。 内核线程的实现很复杂。
上下文切换时间较短。 上下文切换时间更长。
上下文切换不需要硬件支持。 需要硬件支持。
如果一个用户级线程执行阻塞操作，则将阻止整个进程。 如果一个内核线程执行阻塞操作，则另一个线程可以继续执行。


处理：
流程是正在执行的程序。 程序计数器（PC）的值指示正在执行的进程的当前指令的地址。 每个过程由过程控制块（PCB）表示。


流程调度：
以下是关于流程的不同时间。 

 Arrival Time: Time at which the process arrives in the ready queue.
Completion Time: Time at which process completes its execution. 
Burst Time: Time required by a process for CPU execution.
Turn Around Time: Time Difference between completion time and arrival time.          
     Turn Around Time = Completion Time - Arrival Time

Waiting Time(WT): Time Difference between turn around time and burst time.
     Waiting Time = Turn Around Time - Burst Time

为什么我们需要安排？
典型的过程涉及I / O时间和CPU时间。 在像MS-DOS这样的单编程系统中，等待I / O所花费的时间被浪费，并且在此期间CPU是空闲的。 在多道程序设计系统中，一个进程可以使用CPU而另一个进程正在等待I / O. 这仅适用于进程调度。

过程调度算法的目标 

 Max CPU utilization [Keep CPU as busy as possible]
Fair allocation of CPU.
Max throughput [Number of processes that complete their execution per time unit]
Min turnaround time [Time taken by a process to finish execution]
Min waiting time [Time a process waits in ready queue]
Min response time [Time when a process produces first response]

不同的调度算法
先到先服务（FCFS）：最简单的调度算法，根据进程的到达时间进行调度。

最短作业优先（SJF）：首先安排具有最短突发时间的进程。

最短剩余时间优先（SRTF）：它是SJF算法的抢占模式，其中作业根据最短的剩余时间进行调度。

循环调度：以循环方式为每个过程分配固定时间。

基于优先级的调度（非抢先）：在此调度中，根据其优先级调度进程，即，优先级最高的进程是先调度的。 如果两个进程的优先级匹配，则根据到达时间进行计划。

最高响应率下一个（HRRN）在此调度中，调度具有最高响应比的进程。 该算法避免了饥饿。 

 Response Ratio = (Waiting Time + Burst time) / Burst time

多级队列调度：根据进程的优先级，将进程放在不同的队列中。 通常，高优先级进程放在顶级队列中。 只有在从顶级队列完成进程后，才会调度较低级别的排队进程。

多级反馈队列调度：它允许进程在队列之间移动。 这个想法是根据CPU突发的特征来分离进程。 如果进程使用太多CPU时间，则会将其移至优先级较低的队列。

关于调度算法的一些有用的事实：
1） FCFS可能导致等待时间过长，尤其是当第一个作业占用过多CPU时间时。

2） SJF和最短剩余时间第一算法都可能导致饥饿。 考虑一种情况，即长队在准备队列中并且较短的流程继续存在。

3）如果循环调度的时间量非常大，那么它的行为与FCFS调度相同。

4）对于给定的一组过程，SJF在平均等待时间方面是最佳的。 SJF给出了最小的平均等待时间，但SJF的问题是如何知道/预测下一个工作的时间。


关键部分问题
关键部分：程序中代码中访问和/或更新共享变量的部分。

剩余部分：除关键部分外的其余部分。

围绕条件竞争：代码的最终输出取决于访问变量的顺序。 这被称为围绕条件的种族。

临界区问题的解决方案必须满足以下三个条件：

  1. 相互排斥：如果进程Pi在其关键部分中执行，则不允许其他进程进入临界区。
  2. 进度：如果临界区中没有正在执行的进程，则进程临界区的决定不能由其余部分中执行的任何其他进程决定。 选择过程不能无限期推迟。
  3. 有界等待：在进程请求访问临界区之后和授予请求之前，其他进程可以进入临界区的次数存在约束。


同步工具
信号量：信号量是一个整数变量，只能通过两个原子操作wait（）和signal（）访问。 原子操作在单个CPU时间片中执行，没有任何抢占。

信号量有两种类型：

  1. 计数信号量：计数信号量是一个整数变量，其值可以超出不受限制的域。
  2. 互斥锁：二进制信号量称为互斥锁。 它们只能有两个值，0或1.操作wait（）和signal（）以类似的方式对它们进行操作。


      3. 僵局
      由于每个进程持有资源并等待某个其他进程获取的另一个资源而阻塞一组进程的情况。

      如果同时遵守四个条件（必要条件），可能会出现死锁
      相互排斥：一个或多个资源是不可共享的（一次只能使用一个进程）
      保持和等待：进程持有至少一个资源并等待资源。
      No Preemption：除非进程释放资源，否则无法从进程中获取资源。
      循环等待：一组进程以循环形式相互等待。


      处理死锁的方法
      有三种方法可以解决死锁问题
      1）防止死锁或避免：想法是不让系统进入死锁状态。

      2）死锁检测和恢复：发生死锁，然后抢先处理它一旦发生。

      3）一起忽略问题：如果死锁非常罕见，那就让它发生并重启系统。 这是Windows和UNIX采用的方法。

      内存管理：

      这些技术允许在多个进程之间共享内存。

      覆盖：内存应仅包含给定时间所需的那些指令和数据。

      交换：在多道程序设计程序中，使用时间片的指令从内存中换出。

      内存管理技术：

      1：单分区分配方案：内存分为两部分。 一部分保留供操作系统使用，另一部分供用户使用。

      2：多个分区方案：
      固定分区：内存分为固定大小的分区。
      可变分区：内存分为可变大小的分区。

      可变分区分配方案：
      First Fit：到达的过程分配给它完全适合的第一个记忆孔。
      最佳匹配：通过将最小内存留空，将到达的进程分配给最适合的内存空洞。
      最适合：到达的过程被分配给它留下最大间隙的记忆洞。 注意：最佳拟合必然会为内存分配提供最佳结果。

      1.寻呼：物理内存被分成相同大小的帧。 主存储器分为固定大小的页面。 物理内存帧的大小等于虚拟内存帧的大小。

      2.分段：实现分段以向用户提供内存视图。 逻辑地址空间是段的集合。 可以在使用或不使用分页的情况下实现分段。


      页面错误
      页面错误是一种中断，当正在运行的程序访问映射到虚拟地址空间但未加载到物理内存中的内存页面时由硬件引发。

      页面替换算法

      先入先出
      这是最简单的页面替换算法。 在该算法中，操作系统跟踪队列中存储器中的所有页面，最旧的页面位于队列的前面。 当需要替换页面时，选择队列前面的页面进行删除。

      例如，考虑页面引用字符串1,3,0,3,5,6和3页面槽。

      最初所有插槽都是空的，所以当1,3,0出现时，它们被分配给空插槽 - > 3页面故障。
      当3来时，它已经在内存中了 - > 0页面错误。
      然后5来了，它在内存中不可用，因此它取代了最旧的页面槽，即1. - > 1 Page Fault。
      最后6，它在内存中也不可用，因此它取代了最旧的页面槽，即3 - > 1页面错误。

      贝拉迪的异常
      Belady的异常证明，当使用先进先出（FIFO）页面替换算法增加页面帧数时，可能会出现更多页面错误。 例如，如果我们考虑参考字符串3 2 1 0 3 2 4 3 2 1 0 4和3个插槽，我们总共有9个页面错误，但如果我们将插槽增加到4个，则会出现10个页面错误。

      现在为进一步的页面引用字符串 - > 0页面错误，因为它们已在内存中可用。

      最佳页面替换是完美的，但在实践中不可能，因为操作系统无法知道将来的请求。 使用最佳页面替换是设置基准，以便可以针对它分析其他替换算法。
