文件目录| 路径名称

先决条件 - 文件系统

分层目录系统 -

目录以树的形式维护。每个用户可以拥有所需数量的目录，因此，这些文件可以以自然的方式组合在一起。

这种结构的优点：

  • 搜索效率很高
  • 文件的摸索能力增加

将文件系统组织为目录树时，需要某种方式来指定文件名。



通常使用两种不同的方法：

  1. 绝对路径名 -在此方法中，每个文件都有一个绝对路径名，该路径名包含从根目录到文件的路径。 例如，路径/ usr / ast / mailbox表示根目录包含子目录usr，而子目录又包含子目录ast，其中包含文件邮箱。

    绝对路径名始终从根目录开始，并且是唯一的。

    在UNIX中，路径的组件由/分隔。 在Windows中，分隔符是\。
    Windows \ usr \ ast \ mailbox
    UNIX / usr \ ast \ mailbox

  2. 相对路径名称 -与工作目录 （也称为当前目录 ）的概念一起使用。用户可以将一个目录指定为当前工作目录，在这种情况下，所有不在根目录下开始的路径名称都将被取相对于工作目录。

    例如 ，如果当前工作目录是/ usr / ast，则绝对路径为/ usr / ast / mailbox的文件可以简单地称为邮箱。
    换句话说，UNIX
    命令： cp / usr / ast / mailbox /usr/ast/mailbox.bak
    和命令： cp mailbox mailbox.bak
    如果工作目录是/ usr / ast，则完全相同。

何时使用哪种方法？
某些程序需要访问特定文件而不考虑工作目录。 在这种情况下，它们应始终使用绝对路径名。 例如，拼写检查器可能需要读取/ usr / lib / dictionary来完成其工作。 在这种情况下，它应该使用完整的绝对路径名，因为它在调用时不知道工作目录是什么。 无论工作目录是什么，绝对路径名都将始终有效。

当然，如果拼写检查器需要来自/ usr / lib的大量文件，则另一种方法是发出系统调用以将其工作目录更改为/ usr / lib，然后仅使用字典作为第一个参数打开。 通过显式更改工作目录，它可以确定它在目录树中的位置，因此它可以使用相对路径。
