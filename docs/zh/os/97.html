编译C程序时，编译器会生成目标代码。 生成目标代码后，编译器也会调用链接器。 链接器的主要任务之一是使程序可以使用库函数代码（例如printf（），scanf（），sqrt（），.. iec）。 链接器可以通过两种方式完成此任务：将库函数的代码复制到目标代码，或者进行一些安排，以便不复制库函数的完整代码，但在运行时可用。

静态链接和静态库是链接器将所有使用的库函数复制到可执行文件的结果。 静态链接会创建更大的二进制文件，并且需要更多磁盘和主内存空间。 静态库（静态链接的库）的示例是Linux中的.a文件和Windows中的.lib文件。

创建静态库的步骤让我们在UNIX或UNIX中创建和使用静态库，如OS。
1.创建包含库中函数的C文件。 

/* Filename: lib_mylib.c */
#include <stdio.h>
void fun(void)
{
  printf("fun() called from a static library");
}

为简单起见，我们只创建了一个文件。 我们还可以在库中创建多个文件。

2.为库创建头文件 



/* Filename: lib_mylib.h */
void fun(void);

3.编译库文件。 

 gcc -c lib_mylib.c -o lib_mylib.o

4.创建静态库。 此步骤是将多个目标文件捆绑在一个静态库中（有关详细信息，请参阅ar ）。 此步骤的输出是静态库。 

 ar rcs lib_mylib.a lib_mylib.o

5.现在我们的静态库已经可以使用了。 此时我们可以将lib_hello_static.a复制到其他地方以使用它。 出于演示目的，让我们将库保留在当前目录中。

让我们创建一个使用上面创建的静态库的驱动程序 。
1.使用main函数创建一个C文件 

/* filename: driver.c  */
#include "lib_mylib.h"
void main() 
{
  fun();
}

2.编译驱动程序。 

 gcc -c driver.c -o driver.o

3.将已编译的驱动程序链接到静态库。 请注意-L 。 用于表示静态库位于当前文件夹中（有关-L和-l选项的详细信息，请参阅此处）。 

 gcc -o driver driver.o -L. -l_mylib

4.运行驱动程序 

 ./driver 
fun() called from a static library

以下是关于静态库的一些重要观点。
1.对于静态库，链接器从库中提取实际代码，并在编译/构建应用程序时使用它来构建最终的可执行文件。

2.每个进程都有自己的代码和数据副本。 在动态库的情况下，它只是代码共享，数据特定于每个进程。 对于静态库，内存占用量更大。 例如，如果所有窗口系统工具都是静态链接的，那么典型用户将浪费几十兆字节的RAM，并且用户将因大量分页而变慢。

3.由于库代码在编译时连接，最终的可执行文件在运行时没有依赖于库，即没有额外的运行时加载成本，这意味着您不需要携带库的副本正在使用，你拥有一切在你的控制之下，并没有依赖。

4.在静态库中，一旦将所有内容捆绑到您的应用程序中，您就不必担心客户端将在其系统上提供正确的库（和版本）。

5.静态库的一个缺点是，对于静态库中的任何更改（升级），您必须每次都重新编译主程序。

6.静态库的一个主要优点即使现在也是优选的“速度”。 静态库中不会有动态查询符号。 许多生产线软件甚至在今天也使用静态库。



动态链接和动态库动态链接不需要复制代码，只需将库的名称放在二进制文件中即可完成。 当程序运行时，二进制文件和库都在内存中时，会发生实际链接。 动态库（在运行时链接的库）的示例是Linux中的.so和Windows中的.dll 。

我们将很快介绍动态库的更多要点以及创建它们的步骤。
