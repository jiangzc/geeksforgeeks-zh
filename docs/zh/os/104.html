Unix文件系统

Unix文件系统是一种以易于管理的方式组织和存储大量信息的逻辑方法。 该文件是存储信息的最小单位。 Unix文件系统有几个重要的功能。 Unix中的所有数据都被组织成文件。所有文件都被组织到目录中。 这些目录被组织成一个称为文件系统的树状结构。
Unix系统中的文件以多级层次结构组织，称为目录树。 在文件系统的最顶层是一个名为“root”的目录，用“/”表示。 所有其他文件都是root的“后代”。

目录或文件及其描述 -

  • /：斜杠/字符单独表示文件系统树的根。
  • / bin：代表“二进制文件”并包含某些基本实用程序，例如ls或cp，这些实用程序通常是所有用户都需要的。
  • / boot：包含成功启动过程所需的所有文件。
  • / dev：代表“设备”。 包含外围设备和伪设备的文件表示。
  • / etc：包含系统范围的配置文件和系统数据库。 最初还包含“危险的维护工具”，例如init，但这些通常已移至/ sbin或其他地方。
  • / home：包含用户的主目录。
  • / lib：包含系统库和一些关键文件，如内核模块或设备驱动程序。
  • / media：可移动设备的默认安装点，例如USB记忆棒，媒体播放器等。
  • / mnt：代表“mount”。 包含文件系统挂载点。 例如，如果系统使用多个硬盘或硬盘分区，则使用这些。 它还经常用于远程（网络）文件系统，CD-ROM / DVD驱动器等。
  • / proc： procfs虚拟文件系统，显示有关进程的信息。
  • / root：超级用户“root”的主目录 - 即系统管理员。 此帐户的主目录通常位于初始文件系统上，因此不在/ home（可能是另一个文件系统的安装点），以防需要执行特定维护，在此期间其他文件系统不可用。 例如，如果硬盘驱动器遭受物理故障并且无法正确安装，则可能发生这种情况。
  • / tmp：临时文件的位置。 许多系统在启动时清除此目录; 它可能在其上安装了tmpfs，在这种情况下，它的内容不会在重新启动后继续存在，或者它可能会在启动时由启动脚本显式清除。
  • / usr：最初是保存用户主目录的目录，它的使用已经改变。 它现在包含非系统关键的可执行文件，库和共享资源，如X Window系统，KDE，Perl等。但是，在某些Unix系统上，某些用户帐户可能仍然有一个主目录，它是一个直接子目录/ usr，例如Minix中的默认值。 （在现代系统上，这些用户帐户通常与服务器或系统使用有关，而不是由人直接使用）。
  • / usr / bin：此目录存储随操作系统分发的所有二进制程序，这些二进制程序不驻留在/ bin，/ sbin或（很少）/ etc中。
  • / usr / include：存储整个系统中使用的开发标头。 头文件主要由C / C ++编程语言中的#include指令使用。
  • / usr / lib：为/ usr或其他地方存储的程序存储所需的库和数据文件。
  • / var： “变量”的缩写。可能经常更改的文件的位置 - 尤其是大小，例如发送给系统上的用户的电子邮件或进程ID锁定文件。
  • / var / log：包含系统日志文件。
  • / var / mail：存储所有传入邮件的位置。 用户（root除外）只能访问自己的邮件。 通常，此目录是/ var / spool / mail的符号链接。
  • / var / spool：假脱机目录。 包含打印作业，邮件假脱机和其他排队任务。
  • / var / tmp：应在系统重新引导之间保留的临时文件的位置。

Unix文件的类型 - UNIX文件系统包含几种不同类型的文件：



1.普通文件 -普通文件是系统中包含数据，文本或程序指令的文件。

  • 用于存储您的信息，例如您撰写的某些文字或您绘制的图像。 这是您经常使用的文件类型。
  • 始终位于目录文件内/下。
  • 不包含其他文件。
  • 在ls -l的长格式输出中，此类型的文件由“ - ”符号指定。

2.目录 -目录存储特殊文件和普通文件。 对于熟悉Windows或Mac OS的用户，UNIX目录等同于文件夹。 目录文件包含它所包含的每个文件和子目录的条目。 如果目录中有10个文件，则目录中将有10个条目。 每个条目都有两个组件。
（1）文件名
（2）文件或目录的唯一标识号（称为inode编号）

  • 分层树中的分支点。
  • 用于组织文件组。
  • 可能包含普通文件，特殊文件或其他目录。
  • 永远不要包含您将使用的“真实”信息（例如文本）。 基本上，只用于组织文件。
  • 所有文件都是根目录的后代，（名为/）位于树的顶部。

    • 在ls -l的长格式输出中，这种类型的文件由“d”符号指定。

    3.特殊文件 -用于表示用于输入/输出（I / O）操作的真实物理设备，如打印机，磁带机或终端。 设备或特殊文件用于UNIX和Linux系统上的设备输入/输出（I / O）。 它们像普通文件或目录一样出现在文件系统中。
    在UNIX系统上，每个设备有两种特殊文件，即字符特殊文件和块特殊文件：

    • 当字符特殊文件用于设备输入/输出（I / O）时，数据一次传输一个字符。 这种类型的访问称为原始设备访问。
    • 当块特殊文件用于设备输入/输出（I / O）时，数据以大的固定大小块传输。 这种类型的访问称为块设备访问。

    对于终端设备，它一次只能是一个字符。 但是对于磁盘设备，原始访问意味着读取或写入整块数据 - 块，这些块是磁盘的原生块。

    • 在ls -l的长格式输出中，字符特殊文件用“c”符号标记。
    • 在ls -l的长格式输出中，块特殊文件用“b”符号标记。

    4.管道 - UNIX允许您使用管道将命令链接在一起。 管道作为一个临时文件，只存在一个命令中的数据，直到被另一个命令读取。一个Unix管道提供单向数据流。第一个命令序列的输出或结果用作输入第二个命令序列。 要创建管道，请在两个命令之间的命令行上放置一个垂直条（|）。例如： who | wc -l

    在ls -l的长格式输出中，命名管道用“p”符号标记。

    5.套接字 - Unix套接字（或进程间通信套接字）是一种特殊文件，允许进行高级进程间通信。 Unix Socket用于客户端 - 服务器应用程序框架中。 实质上，它是一个数据流，非常类似于网络流（和网络套接字），但所有事务都是文件系统的本地事务。

    在ls -l的长格式输出中，Unix套接字用“s”符号标记。

    6.符号链接 -符号链接用于引用文件系统的其他文件。符号链接也称为软链接。 它包含它引用的文件的路径的文本形式。 对于最终用户，符号链接似乎有自己的名称，但是当您尝试读取或写入此文件的数据时，它会将这些操作引用到它指向的文件。 如果我们删除软链接本身，数据文件仍然存在。如果我们删除源文件或将其移动到其他位置，符号文件将无法正常工作。

    在ls -l的长格式输出中，符号链接用“l”符号标记（这是一个小写的L）。
