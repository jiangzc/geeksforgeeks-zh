Linux find（）命令中的mindepth和maxdepth用于限制搜索到特定目录。

如何限制在linux中搜索指定目录？
在linux中有一个命令可以搜索名为“find”的目录层次结构中的文件。 它通过根据优先级规则从左到右评估给定表达式来搜索以每个给定起始点为根的目录树，直到结果已知（左侧为false，为操作，true为或），此时，find会移动到下一个文件名。 如果没有指定起始点，`。' 假设。
默认情况下，find命令以递归方式向下遍历整个目录树，这耗费时间和资源。 但是，可以指定目录遍历的深度（ mindepth和maxdepth ）。

什么是mindepth和maxdepth级别？

  • maxdepth级别 ：在起始点以下的目录的大多数级别（非负整数）级别下降。 -maxdepth 0表示仅将测试和操作应用于起点本身。
  • mindepth级别 ：不要在低于级别（非负整数）的级别应用任何测试或操作。 -mindepth 1表示处理除起点之外的所有文件。

下面给出一些示例来说明如何使用mindepth和maxdepth指定目录traveal的深度

  • 从根目录开始查找所有子目录下的passwd文件。 
     find / -name passwd
  • 在root下找到passwd文件，向下找一级。 （即根级别1和一个子目录 - 级别2） 
     find / -maxdepth 2 -name passwd
  • 在root下找到passwd文件，向下找两个级别。 （即根级别1和两个子目录 - 级别2和3） 
     find / -maxdepth 3 -name passwd
  • 找到子目录级别2和4之间的密码文件。 
     find / -mindepth 3 -maxdepth 5 -name passwd
