页面替换算法

在使用分页进行内存管理的操作系统中，需要页面替换算法来决定在新页面进入时需要替换哪个页面。每当引用新页面而不存在于内存中时，就会发生页面错误并且操作系统替换一个页面新页面的现有页面。 不同的页面替换算法提出了决定替换哪个页面的不同方法。 所有算法的目标是减少页面错误的数量。

页面错误 -页面错误是一种中断，当正在运行的程序访问映射到虚拟地址空间但未加载到物理内存中的内存页面时由硬件引发。

页面替换算法：

  • 先进先出（FIFO） -
    这是最简单的页面替换算法。 在该算法中，操作系统跟踪队列中存储器中的所有页面，最旧的页面位于队列的前面。 当需要替换页面时，选择队列前面的页面进行删除。

    例如-1，考虑页面引用字符串1,3,0,3,5,6和3页面槽。



    最初所有插槽都是空的，所以当1,3,0出现时，它们被分配给空插槽 - > 3页面故障。
    当3来时，它已经在内存中了 - > 0页面错误。
    然后5来了，它在内存中不可用，因此它取代了最旧的页面槽，即1. - > 1 Page Fault。
    最后6，它在内存中也不可用，因此它取代了最旧的页面槽，即3 - > 1页面错误。

    例2，设有一个参考字符串：a，b，c，d，c，a，d，b，e，b，a，b，c，d，帧的大小为4。

    使用FIFO算法有9页错误。

    Belady的异常 - Belady的异常证明，当使用先进先出（FIFO）页面替换算法增加页面帧数时，可能会出现更多页面错误。 例如，如果我们考虑参考字符串3,2,1,0,3,2,4,3,2,1,0,4和3个插槽，我们总共有9个页面错误，但如果我们将插槽增加到4个，我们得到10页错误。

  • 最佳页面替换 -
    在该算法中，替换页面，这些页面在将来的最长持续时间内不使用。

    让我们考虑页面参考字符串7 0 1 2 0 3 0 4 2 3 0 3 2和4页槽。

    最初所有插槽都是空的，因此当7 0 1 2分配给空插槽时 - > 4页面故障
    0已经存在 - > 0页面错误。
    当3出现时，它将代替7，因为它在将来的最长时间内不会被使用.-> 1页面错误。
    0已经存在 - > 0页面错误。 。
    4将取代1 - > 1 Page Fault。

    现在为进一步的页面引用字符串 - > 0页面错误，因为它们已在内存中可用。

    例2，设有一个参考字符串：a，b，c，d，c，a，d，b，e，b，a，b，c，d，帧的大小为4。

    使用最优算法有6个页面错误。

    最佳页面替换是完美的，但在实践中不可能，因为操作系统无法知道将来的请求。 使用最佳页面替换是设置基准，以便可以针对它分析其他替换算法。

  • 最近最少使用 -
    在此算法中，将替换最近最少使用的页面。

    假设页面引用字符串7 0 1 2 0 3 0 4 2 3 0 3 2。 最初我们有4个页面空位。
    最初所有插槽都是空的，因此当7 0 1 2分配给空插槽时 - > 4页面故障
    0已经是他们的 - > 0页面错误。
    当3出现时它将取代7，因为它是最近最少使用的 - > 1页面错误
    0已经在内存中所以 - > 0页面错误 。
    4将取代1 - > 1 Page Fault
    现在为进一步的页面引用字符串 - > 0页面错误，因为它们已在内存中可用。
