我们在上一篇文章中介绍了有关共享库的基本信息。 在本文中，我们将学习如何在Linux上创建共享库。

在此之前，我们需要了解程序如何加载到内存中，以及该过程中涉及的各种（基本）步骤。

让我们在C中看到一个典型的“Hello World”程序。简单的Hello World程序屏幕图像如下。

我们使用命令“ gcc -o sample shared.c ”编译代码。当我们编译代码时，编译器不会解析函数printf（）的实现 。 它只验证语法检查。 工具链在我们的应用程序中留下了一个存根，它将由动态链接器填充。 由于printf是标准函数，因此编译器隐式调用其共享库。 更多细节。



我们使用ldd列出程序二进制映像的依赖项。 在屏幕图像中，我们可以看到我们的示例程序依赖于三个二进制文件，即linux-vdso.so.1 ， libc.so.6和/lib64/ld-linux-x86-64.so.2 。

文件VDSO是系统调用接口和其他一些东西的快速实现，它不是我们关注的焦点（在一些较旧的系统上，您可能会看到* .vsdo。*的liue中的不同文件名）。 忽略此文件。 我们对其他两个文件感兴趣。

libc.so.6文件是各种标准函数的C实现。 这是我们看到Hello World所需的printf定义的文件。 需要将共享库加载到内存中以运行Hello World程序。

第三个文件/lib64/ld-linux-x86-64.so.2实际上是在调用应用程序时运行的可执行文件。 当我们在bash终端上调用程序时，通常bash会自行分叉并用运行程序的图像替换它的地址空间（所谓的fork-exec对）。 内核验证libc.so.6是否驻留在内存中。 如果没有，它会将文件加载到内存中并重新定位libc.so.6符号。 然后它调用动态链接器（/lib64/ld-linux-x86-64.so.2）来解析应用程序代码的未解析符号（在本例中为printf）。 然后控件转移到我们的程序main 。 （我在内容中遗漏了许多细节，我们的重点是了解基本细节）。

创建我们自己的共享库：

让我们在Linux上使用简单的共享库。 使用以下内容创建文件library.c .

文件library.c定义了一个函数signum ，它将由我们的应用程序代码使用。 使用以下命令编译文件library.c文件。

gcc -shared -fPIC -o liblibrary.so library.c

标志-shared指示编译器我们正在构建共享库。 标志-fPIC用于生成与位置无关的代码（暂时忽略）。 该命令在当前工作目录中生成共享库liblibrary.so 。 我们可以使用我们的共享对象文件（Linux中的共享库名）。

使用以下内容创建另一个文件application.c 。

在application.c文件中，我们调用在共享库中定义的函数signum。 使用以下命令编译application.c文件。

gcc application.c -L / home / geetanjali / coding / -llibrary -o sample

标志-llibrary指示编译器查找当前代码中不可用的符号定义（在我们的例子中为signum函数）。 选项-L提示编译器查看目录，然后查看任何共享库的选项（仅在链接时）。 该命令生成一个名为“ sample ”的可执行文件。

如果调用可执行文件，则动态链接器将无法找到所需的共享库。 默认情况下，它不会查看当前工作目录。 您必须明确指示工具链提供正确的路径。 动态链接器搜索LD_LIBRARY_PATH中可用的标准路径，还搜索系统缓存（有关详细信息，请参阅命令ldconfig ）。 我们必须将我们的工作目录添加到LD_LIBRARY_PATH环境变量中。 以下命令执行相同操作。

export LD_LIBRARY_PATH = / home / geetanjali / coding /：$ LD_LIBRARY_PATH
