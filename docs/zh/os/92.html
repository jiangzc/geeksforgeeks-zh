内存管理中的叠加

固定分区的主要问题是进程的大小必须受到分区最大大小的限制，这意味着进程永远不会跨越另一个进程。为了解决这个问题，早期人们已经使用了一些解决方案。称为叠加层。

叠加的概念是，无论何时一个进程运行它都不会同时使用整个程序，它只会使用它的某些部分。然后叠加概念说无论你需要什么部分，你只需要加载一个部分。完成，然后你只需卸载它，意味着只需将其拉回来并获得所需的新部件并运行它。
从形式上看，
“将程序代码块或其他数据传输到内部存储器中的过程，替换已存储的内容”。
有时它会发生与最大分区的大小相比，程序的大小会更大，那么，在这种情况下，你应该使用叠加。

因此，覆盖是一种运行比物理内存大小更大的程序的技术，只保留在任何给定时间所需的那些指令和数据。将程序划分为模块，使得并非所有模块都需要同时在记忆中。

优势 -

  • 减少内存需求
  • 减少时间要求

坏处 -



  • 必须由程序员指定重叠映射
  • 程序员必须知道内存要求
  • 覆盖模块必须完全不相交
  • 覆盖结构的编程设计是复杂的，并且在所有情况下都不可能

示例 -
覆盖的最好例子是汇编程序。考虑到汇编程序有2遍，2遍传递意味着它只做一件事，无论是第一遍还是第二遍。这意味着它将先完成第一遍然后第二遍。假设可用主内存大小为150KB，总代码大小为200KB 

 Pass 1.......................70KB
Pass 2.......................80KB
Symbol table.................30KB
Common routine...............20KB

由于总代码大小为200KB，主内存大小为150KB，因此无法一起使用2个通道。因此，在这种情况下，我们应该采用叠加技术。根据叠加概念，任何时候只需一次通过使用和两个通道总是需要符号表和常见例程。现在的问题是，如果覆盖 - 驱动程序*是10KB，那么所需的最小分区大小是多少？对于通过1，所需的总内存是=（70KB + 30KB + 20KB + 10KB）= 130KB，并且对于pass 2需要的总内存=（80KB + 30KB + 20KB + 10KB）= 140KB。如果我们有最小140KB大小的分区，那么我们可以非常容易地运行这个代码。

*覆盖驱动程序： - 用户负责处理覆盖，操作系统不会提供任何内容。这意味着用户应该写第一遍中需要的部分，一旦第一遍结束，用户应该编写代码以拉出传递1并加载传递2.这就是用户的责任，即所谓的Overlays驱动程序.Overlays驱动程序将帮助我们移出并移动到的各个部分码。

题 -
程序的覆盖树如下所示：

加载所需的分区大小（物理内存）是多少（和
运行）这个程序？
（a）12KB（b）14KB（c）10KB（d）8KB
说明 -
使用叠加概念我们实际上不需要将整个程序放在主存储器中。只需要在那个时刻需要的部分，我们需要Root-AD或Root-AE或Root-BF或Root- CG部分。 

 Root+A+D = 2KB + 4KB + 6KB = 12KB
Root+A+E = 2KB + 4KB + 8KB = 14KB
Root+B+F = 2KB + 6KB + 2KB = 10KB
Root+C+G = 2KB + 8KB + 4KB = 14KB

因此，如果我们有14KB大小的分区，那么我们可以运行它们中的任何一个。
答案 - （b）14KB
