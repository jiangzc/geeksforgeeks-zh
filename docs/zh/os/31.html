自私的循环调度

先决条件 - 循环调度程序

在传统的循环调度算法中，所有过程都被同等地处理。 Selfish Round Robin的目标是为已经执行一段时间而不是新手的流程提供更好的服务。 与普通的Round Robin算法相比，它更具逻辑性和优越性。

实施： -

  • 就绪列表中的进程被分为两个列表：NEW和ACCEPTED。
  • 新流程等待，而循环服务接受流程。
  • 新流程的优先级以“a”的速率增加，而已接受流程的优先级以“b”的速率增加。
  • 当新进程的优先级达到已接受进程的优先级时，该新进程将被接受。
  • 如果所有接受的流程完成，则接受优先级最高的新流程。

让我们来看看这个算法的一般工作原理： -

步骤1：假设当第一个A到达时，最初没有准备好的过程。 它的优先级为0。 由于没有其他已接受的流程，因此立即接受A.
第2步：过了一会儿，另一个过程B到了。 只要b / a <1，B的优先级最终会赶上A，所以它被接受; 现在A和B都具有相同的优先级。
步骤3：所有接受的流程共享一个共同的优先级（以b的速率上升）; 这使得该策略易于实现，即任何新进程的优先级必然会在某些时候被接受。 所以没有过程必须经历饥饿。
步骤4：即使b / a> 1，A最终也会完成，然后B可以被接受。 

 Adjusting the parameters a and b : 
          -> If b / a >= 1, a new process is not accepted 
                 until all the accepted processes have finished, so SRR becomes FCFS. 
          -> If b / a = 0, all processes are accepted immediately, so SRR becomes RR. 
          -> If 0 < b / a < 1, accepted processes are selfish, but not completely.
