页表条目

先决条件 - 寻呼

页表具有页表条目，其中每个页表条目存储帧号和可选状态（如保护）位。 虚拟内存系统中使用的许多状态位。 PTE中最重要的是帧号 。

页表条目具有以下信息 -

  1. 帧编号 -它给出了您要查找的当前页面所在的帧编号。 所需的位数取决于帧数。 
     Number of bits for frame = Size of physical memory/frame size
  2. 存在/不存在位 -存在或不存在位表示您要查找的特定页面是否存在。 如果它不存在，则称为Page Fault。 如果相应页面不在内存中，则设置为0。 用于通过操作系统控制页面错误以支持虚拟内存。 有时这个位也称为有效/无效位。
  3. 保护位 -保护位表示您在该页面上需要什么样的保护。 所以，这些位用于保护页面帧（读，写等）。
  4. 参考位 -参考位将说明该页是否在最后一个时钟周期中被引用。 访问页面时，硬件设置为1。
  5. 启用/禁用缓存 -有时我们需要新数据。 让我们说用户正在键盘输入一些信息，你的程序应该根据用户给出的输入运行。 在这种情况下，信息将进入主存储器。 因此，主存储器包含用户键入的最新信息。 现在，如果您尝试将该页面放在缓存中，该缓存将显示旧信息。 因此，无论何时需要新鲜度，我们都不希望进行缓存或存储器的多个级别。与CPU最接近的级别中存在的信息以及与用户最接近的级别中存在的信息可能不同。 所以我们希望信息必须是一致性的，这意味着用户提供的信息，CPU应该能够尽可能地首先看到它。 这就是我们想要禁用缓存的原因。 因此，此位启用或禁用页面缓存。
  6. 修改位 -修改位表示页面是否已被修改。 修改意味着有时您可能会尝试在页面上写入内容。 如果页面被修改，那么无论何时应该将该页面替换为其他页面，修改后的信息都应该保存在硬盘上，或者必须将其写回或者必须保存回来。 在写入访问页面时由硬件设置为1，用于避免在换出时写入。 有时这个修改过的位也称为Dirty位 。
