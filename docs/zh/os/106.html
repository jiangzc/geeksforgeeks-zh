文件分配方法

分配方法定义文件如何存储在磁盘块中。 有三种主要的磁盘空间或文件分配方法。

  • 连续分配
  • 链接分配
  • 索引分配

这些方法背后的主要思想是提供：

  • 高效的磁盘空间利用率。
  • 快速访问文件块。

所有这三种方法都有各自的优缺点，如下所述：

1.连续分配

在此方案中，每个文件占用磁盘上的一组连续块。 例如，如果文件需要n个块并且给出块b作为起始位置，则分配给该文件的块将是： b，b + 1，b + 2，...... b + n-1。 这意味着给定起始块地址和文件长度（根据所需的块），我们可以确定文件占用的块。

优点：

  • 此次支持Sequential和Direct Accesses。 对于直接访问，可以容易地获得在块b处开始的文件的第k个块的地址为（b + k）。
  • 这是非常快的，因为由于文件块的连续分配，寻道次数很少。

缺点：

  • 该方法受到内部和外部碎片的影响。 这使得它在内存利用率方面效率低下。
  • 增加文件大小很困难，因为它取决于特定实例中连续内存的可用性。

2.链接列表分配

在该方案中，每个文件是磁盘块的链接列表，其不需要是连续的。 磁盘块可以分散在磁盘上的任何位置。
目录条目包含指向开始和结束文件块的指针。 每个块包含指向文件占用的下一个块的指针。

下图中的文件“吉普”显示了块是如何随机分布的。 最后一个块（25）包含-1，表示空指针，并且不指向任何其他块。

优点：

  • 这在文件大小方面非常灵活。 文件大小可以轻松增加，因为系统不必查找连续的内存块。
  • 此方法不会受到外部碎片的影响。 这使得它在内存利用率方面相对更好。

缺点：

  • 由于文件块在磁盘上随机分布，因此需要大量搜索才能单独访问每个块。 这使得链接分配更慢。
  • 它不支持随机或直接访问。 我们无法直接访问文件的块。 可以通过块指针从文件的起始块顺序遍历k个块（顺序访问）来访问文件的块k。
  • 链接分配中所需的指针会产生一些额外的开销。

3.索引分配

在此方案中，称为索引块的特殊块包含指向文件占用的所有块的指针。 每个文件都有自己的索引块。 索引块中的第i个条目包含第i个文件块的磁盘地址。 目录条目包含索引块的地址，如图所示：


优点：

  • 这支持直接访问文件占用的块，因此可以快速访问文件块。
  • 它克服了外部碎片的问题。

缺点：

  • 索引分配的指针开​​销大于链接分配。
  • 对于非常小的文件，比如只扩展2-3个块的文件，索引分配将保留指针的整个块（索引块），这在内存利用率方面是低效的。 但是，在链接分配中，每个块只丢失了1个指针的空间。

对于非常大的文件，单个索引块可能无法保存所有指针。
以下机制可用于解决此问题：

  1. 链接方案：该方案将两个或多个索引块链接在一起以保存指针。 然后，每个索引块都包含一个指针或下一个索引块的地址。
  2. 多级索引：在此策略中，第一级索引块用于指向第二级索引块，这些索引块指向文件占用的磁盘块。 根据最大文件大小，这可以扩展到3个或更多级别。
  3. 组合方案：在此方案中，名为Inode（信息节点）的特殊块包含有关文件的所有信息，如名称，大小，权限等，Inode的剩余空间用于存储包含的磁盘块地址实际文件如下图所示。 Inode中的前几个指针指向直接块，即指针包含包含文件数据的磁盘块的地址。 接下来的几个指针指向间接块。 间接块可以是单个间接，双重间接或三个间接。 单个间接块是不包含文件数据的磁盘块，而是包含文件数据的块的磁盘地址。 类似地， 双重间接块不包含文件数据，而是包含包含文件数据的块地址的块的磁盘地址。
