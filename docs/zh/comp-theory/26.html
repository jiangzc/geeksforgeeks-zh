上下文中的歧义免费语法和上下文无关语言

先决条件 - 下推自动机和上下文自由语言 。

假设我们有一个带有生成规则的无上下文语法G：S-> aSb | bSa | SS |ɛ

最左派生（LMD）和派生树：从起始符号S最左边的字符串派生是通过用相应的生产规则的RHS替换最左边的非终端符号来完成的。 例如：上面语法G中最左边的字符串abab派生如下： 

 S => a S b => ab S ab => abab

带下划线的符号将使用生产规则替换。

派生树：它告诉我们如何使用S的生产规则派生字符串，如图1所示。



图1

最右边的推导（RMD）：通过用相应的生产规则的RHS替换最右边的非终端符号来完成从凝视符号S最右边的字符串的推导。 例如：上面语法G中最右边的字符串abab派生如下： 

 S => S S => Sa S b => S ab => a S bab => abab

带下划线的符号将使用生产规则替换。 使用最右边推导的abab的推导树如图2所示。

图2

推导可以是LMD或RMD，也可以是两者或无。 例如： 

 S => a S b => ab S ab => abab is LMD as well as RMD

but S => S S => Sa S b => S ab => a S bab => abab is RMD but not LMD.

不明确的上下文自由语法：如果对于由语法生成的字符串存在多个LMD或多个RMD，则上下文无关语法称为不明确的。 在模糊语法中，字符串也会有多个派生树。 上面描述的语法是模糊的，因为有两个派生树（图1和图2）。 字符串abab可以有多个RMD，它们是： 

 S => S S => Sa S b => S ab => a S bab => abab

S => a S b => ab S ab => abab

模糊上下文自由语言：如果没有明确的语法来定义该语言，则无上下文语言被称为模糊语言，并且它也被称为固有模糊的上下文自由语言。

注意：

  • 如果无上下文语法G是不明确的，则由语法L（G）生成的语言可能是或可能不是模糊的
  • 并不总是可以将模糊的CFG转换为明确的CFG。 只有一些模棱两可的CFG可以转换为明确的CFG。
  • 没有算法可以将模糊的CFG转换为明确的CFG。
  • 始终存在与明确的CFL相对应的明确的CFG。
  • 确定性CFL始终是明确的。

问题：考虑以下有关上下文无关语法的陈述 

 G = {S->SS, S->ab, S->ba, S->ɛ}
  • 我是暧昧的
  • II G产生具有相同数量的a和b的所有字符串
  • III G可被确定性PDA接受

下面哪个组合表达了关于G的所有真实陈述？

  1. 我只是
  2. 仅限I和III
  3. 仅限II和III
  4. I，II和III

解决方案：字符串abab有不同的LMD可以 

 S => S S => S SS => ab S S => abab S => abab

S => S S => ab S => abab

所以语法含糊不清。 所以声明我是真的。

声明II声明语法G产生具有相同数量的a和b的所有字符串但它不能生成aabb字符串。 声明II不正确。

声明III也是正确的，因为它可以被确定性PDA接受。 所以正确的选项是（B）。

问题：以下哪一项陈述是错误的？

  1. 存在无上下文的语言，使得生成它们的所有无上下文的语法都是不明确的。
  2. 明确的上下文无关语法对于由它生成的语言的每个字符串总是具有唯一的解析树。
  3. 确定性和非确定性下推自动机都始终接受相同的语言集。
  4. 一个字母表中的一组有限字符串始终是常规语言。

解决方案： （A）是正确的，因为对于模糊的CFL，与其对应的所有CFG都是不明确的。

（B）也是正确的，因为明确的CFG对于由它生成的语言的每个字符串具有唯一的解析树。

（C）是错误的，因为某些语言被非确定性PDA接受，但不被确定性PDA接受。

（D）也是正确的，因为有限的字符串集总是规则的。

因此选项（C）是正确的选项。
