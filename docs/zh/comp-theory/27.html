运算符语法和优先级解析器

为定义数学运算符而生成的语法称为运算符语法 ，对语法有一些限制。 运算符优先级语法是一种无上下文语法，其具有以下属性：没有生产在其右侧没有空的右侧（空产生）或两个相邻的非终端。

例子 -
这是运算符语法的示例： 

 E->E+E/E*E/id

但是，下面给出的语法不是运算符语法，因为两个非终端彼此相邻： 

 S->SAS/a
A->bSb/b

虽然，我们可以将其转换为运算符语法： 

 S->SbSbs/SbS/a
A->bSb/b

运算符优先级解析器 -
运算符优先级解析器是解释运算符优先级语法的自下而上解析器之一。 此解析器仅用于运算符语法。 除了运算符优先级解析器之外，任何解析器都不允许使用不明确的语法 。
有两种方法可以确定一对终端之间应该保持的优先级关系：



  1. 使用运算符的常规关联性和优先级。
  2. 选择运算符优先级关系的第二种方法是首先为语言构造一个明确的语法，这种语法反映了其解析树中正确的关联性和优先级。

该解析器依赖于以下三个优先级关系： ⋖，≐，⋗
a⋖b这意味着“收益优先于”b。
a⋗b这意味着“优先于”b。
a≐b这意味着“优先”为b。


图 -语法的运算符优先级关系表E-> E + E / E * E / id

id和id之间没有任何关系，因为id不会被比较，并且两个变量不能并排。 这个表也有一个缺点，好像我们有n个运算符而不是table的大小是n * n，复杂度将是0（n 2 ）。 为了增加表的大小，请使用运算符函数表 。

运算符优先级解析器通常不存储带有关系的优先级表; 相反，它们是以特殊方式实施的。 运算符优先级解析器使用将终端符号映射到整数的优先级函数 ，因此符号之间的优先级关系通过数值比较来实现。 解析表可以由两个优先级函数f和g编码，它们将终端符号映射到整数。 我们选择f和g，这样：

  1. f（a）<g（b）只要a优先于b
  2. f（a）= g（b）只要a和b具有优先权
  3. f（a）> g（b）只要优先于b

示例 -考虑以下语法： 

 E -> E + E/E * E/( E )/id

表示优先级函数的有向图：

由于图中没有任何循环所以我们可以创建函数表： 

 fid -> g* -> f+ ->g+ -> f$
gid -> f* -> g* ->f+ -> g+ ->f$

表的大小是2n 。

函数表的一个缺点是evev虽然我们有空白条目，但我们在函数表中得到了非空条目。 空白条目也称为错误。 因此，关系表的错误检测能力大于功能表。 

#include <stdio.h>
#include <string.h>

// function f to exit from the loop
// if given condition is not true
void f()
{
    printf("Not operator grammar");
    exit(0);
}

void main()
{
    char grm[20][20], c;

    // Here using flag variable,
    // considering grammar is not operator grammar
    int i, n, j = 2, flag = 0;

    // taking number of productions from user
    scanf("%d", &n);
    for (i = 0; i < n; i++)
        scanf("%s", grm[i]);

    for (i = 0; i < n; i++) {
        c = grm[i][2];

        while (c != '&#092;&#048;') {

            if (grm[i][3] == '+' || grm[i][3] == '-'
                || grm[i][3] == '*' || grm[i][3] == '/')

                flag = 1;

            else {

                flag = 0;
                f();
            }

            if (c == '$') {
                flag = 0;
                f();
            }

            c = grm[i][++j];
        }
    }

    if (flag == 1)
        printf("Operator grammar");
}

 Input :3
A=A*A
B=AA
A=$

Output : Not operator grammar

Input :2
A=A/A
B=A+A

Output : Operator grammar

$是一个null生成，在运算符语法中也是不允许的。

优点 -

  1. 它可以很容易地手工构建
  2. 实现这种类型的解析很简单

缺点 -

  1. 很难处理像减号（ - ）这样的令牌，它有两个不同的优先级（取决于它是一元还是二元）
  2. 它仅适用于小类语法
