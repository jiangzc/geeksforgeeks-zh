为所有长度的回文构建下推自动机

下推自动机 （PDA）就像具有无限堆栈的epsilon非确定性有限自动机（NFA）。 PDA是一种实现无上下文语言的方法。 因此，学习如何绘制PDA是很重要的。

这里以奇数长度回文为例：
Que-1：为语言L = {wcw'|构建PDA w = {0,1} *}其中w'与w相反。

此PDA使用的方法 -
继续推动0和1，无论堆栈顶部是什么，直到到达中间元素。 当扫描中间元素“c”然后处理它而不对堆栈进行任何更改。 现在，如果扫描符号为'1'且堆栈顶部也包含'1'，则从堆栈顶部弹出元素，或者如果扫描符号为'0'，堆栈顶部也包含'0'，则从堆栈顶部弹出元素。 如果字符串变空或扫描符号为'$'且堆栈变空，则达到最终状态，否则移至死状态。

  • 步骤1：在接收0或1时，继续将其推到堆栈顶部而不进入下一状态。
  • 步骤2：在接收元素'c'时，移动到下一状态而不对堆栈进行任何更改。
  • 步骤3：接收元素后，检查扫描的符号是否为“1”，堆栈顶部是否包含“1”，或者如果扫描的符号为“0”，堆栈顶部也包含“0”，则从堆栈顶部弹出元素否则会进入死亡状态。 继续重复步骤3，直到字符串变空。
  • 步骤4：检查扫描的符号是否为'$'并且堆栈不包含任何元素然后移动到最终状态否则移动到死状态。

例子： 



 Input : 1 0 1 0 1 0 1 0 1
Output :ACCEPTED

Input : 1 0 1 0 1 1 1 1 0
Output :NOT ACCEPTED

现在，以长度回文为例：
Que-2：为语言L = {ww'|构建PDA w = {0,1} *}其中w'与w相反。

此PDA使用的方法 -
对于构建均匀长度的回文，用户必须使用非确定性下推自动机（NPDA）。 NPDA基本上是添加了堆栈的NFA。
除了epsilon转换之外，该语言的NPDA与前一个语言相同。 然而，有一个显着的区别，这个PDA必须猜测何时停止推动符号，跳转到最终状态并开始匹配堆栈。 因此，这台机器绝对是不确定的。
继续按0和1，无论堆栈顶部是什么，同时检查输入字符串，是否到达输入字符串的后半部分。 如果到达输入字符串的前半部分的最后一个元素，则在处理输入字符串的前半部分的最后一个元素之后，使epsilon移动并移动到下一状态。 现在，如果扫描符号为'1'且堆栈顶部也包含'1'，则从堆栈顶部弹出元素，或者如果扫描符号为'0'，堆栈顶部也包含'0'，则从堆栈顶部弹出元素。 如果字符串变空或扫描符号为'$'且堆栈变空，则达到最终状态，否则移至死状态。

  • 步骤1：在接收0或1时，继续将其推到堆栈顶部，同时继续检查是否到达输入字符串的后半部分。
  • 步骤2：如果到达输入字符串前半部分的最后一个元素，则将该元素推到堆栈顶部，然后使epsilon移动到下一个状态。
  • 步骤3：接收元素后，检查扫描的符号是否为“1”，堆栈顶部是否包含“1”，或者如果扫描的符号为“0”，堆栈顶部也包含“0”，则从堆栈顶部弹出元素否则会进入死亡状态。 继续重复步骤3，直到字符串变空。
  • 步骤4：检查扫描的符号是否为'$'并且堆栈不包含任何元素然后移动到最终状态否则移动到死状态。

例子： 

 Input : 1 0 0 1 1 1 1 0 0 1
Output :ACCEPTED

Input : 1 0 0 1 1 1
Output :NOT ACCEPTED

现在，以全长回文为例，即可以接受奇数长度回文和均匀长度回文的PDA：
Que-3：为语言L = {ww'|构建PDA wcw'，w = {0,1} *}其中w'与w相反。

此PDA使用的方法 -
对于全长回文构造，用户必须使用NPDA。
该方法类似于上面的示例，除了现在随着epsilon移动，用户必须再显示符号'c'的一个过渡移动，即如果字符串是奇数长度并且如果到达中间元素'c'然后只是处理它并移动到下一个状态而不做任何堆栈更改。

  • 步骤1：在接收0或1时，继续将其推到堆栈顶部并同时继续检查，如果输入字符串是偶数长度则是否到达输入字符串的后半部分，但是如果输入字符串是奇怪的长度然后继续检查是否到达中间元素。
  • 步骤2：如果输入字符串是偶数长度并且到达输入字符串的前半部分的最后一个元素，则将该元素推到堆栈顶部然后使epsilon移动到下一状态或者如果输入字符串是奇数长度然后打开接收元素'c'，移动到下一状态而不对堆栈进行任何更改。
  • 步骤3：接收元素后，检查扫描的符号是否为“1”，堆栈顶部是否包含“1”，或者如果扫描的符号为“0”，堆栈顶部也包含“0”，则从堆栈顶部弹出元素否则会进入死亡状态。 继续重复步骤3，直到字符串变空。
  • 步骤4：检查扫描的符号是否为'$'并且堆栈不包含任何元素然后移动到最终状态否则移动到死状态。

例子： 

 Input : 1 1 0 0 1 1 1 1 0 0 1 1
Output :ACCEPTED

Input : 1 0 1 0 1 0 1
Output :ACCEPTED
