简化上下文无关语法

上下文无关语法（CFG）的定义允许我们开发各种语法。 大多数时候，一些CFG的制作没有用，而且是多余的。 之所以发生这种情况，是因为CFG的定义并不限制我们制作这些多余的作品。

通过简化CFG，我们从语法中删除所有这些冗余产品，同时保持转换后的语法等同于原始语法。 如果两个语法生成相同的语言，则称它们是等效的。 简化CFG是必要的，以便以后将它们转换为普通形式。

冗余产品的类型和去除它们的过程如下所述。
1.无用的制作 - 永远不会参与衍生任何字符串的制作，被称为无用的制作。 类似地，一个永远不会参与派生任何字符串的变量称为无用变量。 例如。 

 S -> abS | abA | abB
A -> cd
B -> aB
C -> dc

在上面的例子中，生产'C - > dc'是没用的，因为变量'C'永远不会出现在任何字符串的派生中。 其他产品的编写方式是变量“C”永远不能从起始变量“S”到达。

生产'B - > aB'也没用，因为它无法终止。 如果它永远不会终止，那么它永远不会产生字符串。 因此，生产永远不会参与任何推导。



为了删除无用的产品，我们首先找到所有永远不会导致终端字符串的变量，例如变量'B'。 然后我们删除变量'B'出现的所有产品。
所以修改后的语法变成了 - 

 S -> abS | abA
A -> cd
C -> dc

然后我们尝试识别从起始变量永远无法达到的所有变量，例如变量'C'。 然后我们删除变量“C”出现的所有产品。
下面的语法现在没有无用的制作 - 

 S -> abS | abA
A -> cd

2.λ产品 - 'A - >λ'类型的产品被称为λ产品（也称为lambda产品和零产品）。 这些产品只能从那些不产生λ（空字符串）的语法中删除。 语法可能包含空产生但不产生空字符串。

要删除null产品，我们首先必须找到所有可空的变量。 如果λ可以从'A'导出，则变量'A'被称为可空。 对于所有类型'A - >λ'的产生，'A'是可空的变量。 对于所有类型为'B - > A1A2 ... An'的产品，其中所有'Ai'都是可空变量，'B'也是可以为空的变量。

在找到所有可空变量之后，我们现在可以开始构造空生成自由语法。 对于原始语法中的所有制作，我们添加原始制作以及可以通过用λ替换制作中的可空变量而形成的所有制作组合。 如果生产的RHS上的所有变量都可以为空，那么我们不会在新语法中添加“A - >λ”。 一个例子可以说明问题。 考虑语法 - 

 S -> ABCd (1)
A -> BC (2)
B -> bB | λ (3)	
C -> cC | λ (4)

让我们首先找到所有可空的变量。 变量'B'和'C'显然可以为空，因为它们在生产的RHS中包含'λ'。 变量'A'也可以为空，因为在（2）中，RHS上的两个变量也可以为空。 同样，变量'S'也可以为空。 因此变量'S'，'A'，'B'和'C'是可以为空的变量。

让我们创建新的语法。 我们从第一次制作开始。 按原样添加第一个作品。 然后我们创建所有可能的组合，可以通过用λ替换可空变量来形成。 因此，行（1）现在变为'S - > ABCd | ABd | ACd | BCd | 广告| Bd | Cd | d'。我们将相同的规则应用于第（2）行，但我们不添加'A - >λ'，即使它是一种可能的组合。 我们删除了'V - >λ'类型的所有产品。 新语法现在变成了 - 

 S -> ABCd | ABd | ACd | BCd | Ad | Bd |Cd | d
A -> BC | B | C
B -> bB | b
C -> cC | c

3.单元制作 - “A - > B”类型的制作称为单元制作。
要从原始语法'G'创建单元制作免费语法'Guf'，我们按照下面提到的步骤。

首先在'Guf'中添加'G'的所有非单位制作。 然后对于语法'G'中的每个变量'A'，找到所有变量'B'，使得'A * => B'。 现在，对于所有变量，如'A'和'B'，添加'A - > x1 | x2 | ...... xn'到'Guf'其中'B - > x1 | x2 | ...... xn'在'Guf'中。 x1，x2 ... xn都不是单个变量，因为我们只在'Guf'中添加了非单位产品。 因此，得到的语法是单位生产免费的。 例如。 

 S -> Aa | B
A -> b | B
B -> A | a

让我们在'Guf'中添加'G'的所有非单位制作。 'Guf'现在变成了 - 

 S -> Aa
A -> b
B -> a

现在我们找到满足'X * => Z'的所有变量。 这些是'S * => A'，'S * => B'，'A * => B'和'B * => A'。 对于'A * => B'，我们添加'A - > a'，因为'B - > a'存在于'Guf'中。 'Guf'现在变成了 

	
S -> Aa
A -> b | a
B -> a

对于'B * => A'，我们添加'B - > b'，因为'A - > b'存在于'Guf'中。 新的语法现在变成了 

 S -> Aa
A -> b | a
B -> a | b

我们按照'S * => A'和'S * => B'的步骤进行操作，最后得到以下语法 - 

 S -> Aa | b | a
A -> b | a
B -> a | b

注意：要删除上面提到的所有类型的作品，首先删除空制作，然后删除单位制作，最后删除无用的制作。 遵循此顺序对于获得正确的结果非常重要。
