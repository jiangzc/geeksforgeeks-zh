最终状态下推自动机接受

我们已经讨论了Pushdown Automata（PDA）及其空堆文章的接受度 。 现在，在本文中，我们将讨论PDA如何基于最终状态接受CFL。 鉴于PDA P为： 

 P = (Q, Σ, Γ, δ, q0, Z, F)

P接受的语言是所有字符串的集合，PDA可以从初始状态移动到最终状态，而不管堆栈上留下的任何符号，可以描述为： 

 L(P) = {w |(q0, w, Z) =>(qf, ɛ, s)}

这里，从开始状态q0和堆栈符号Z，当输入w被消耗时，达到最终状态qfɛF。 堆栈可以包含字符串s，这与达到最终状态无关，并且将接受w。

示例：定义语言{a ^ nb ^ n |的下推自动机 n> 0}使用最终状态。
解： M =其中Q = {q0，q1，q2，q3}且Σ= {a，b}且Γ= {A，Z}且F = {q3}且δ由下式给出： 

 δ( q0, a, Z ) = { ( q1, AZ ) }
δ( q1, a, A) = { ( q1, AA ) }
δ( q1, b, A) = { ( q2, ɛ) }
δ( q2, b, A) = { ( q2, ɛ) }
δ( q2, ɛ, Z) = { ( q3, Z) }

在读取b之后（如第4行所示），它将弹出A并移动到状态q2，堆栈将是AAZ。 当读取所有b时，状态将为q2，堆栈将为Z.在第7行中，在输入符号ɛ和堆栈上的Z上，它将移至q3。 在处理输入后已达到最终状态q3，将接受该字符串。
这种接受被称为最终状态的接受 。

接下来，给定的PDA P可以写成： 

 Q = {q0, q1, q2} and ∑ = {a, b} 
And Γ = {A, Z} and F={q0,q2} and δ is given by :
δ( q0, a, Z ) = {( q0, XZ)}
δ( q0, a, X) = {( q0, XX )}
δ( q0, b, X) = {( q1, ɛ)}
δ( q1, b, X) = {( q1, ɛ)}
δ( q1, ɛ, Z) = {( q2, Z)}

我们可以看到，q0是初始状态，也是最终状态，ɛ将被接受。 对于每一个，X被推到堆栈上，PDA保持最终状态。 因此，PDA可以接受任何数量的a。

如果输入包含b，则每隔b从堆栈中弹出X. 然后，如果堆栈在处理输入后变为空，则PDA移动到最终状态（δ（q1，ɛ，Z）= {（q2，Z）}）。 因此，如果存在b，则b的数量必须等于b的数量。

由于给定状态和输入只有一个动作，PDA是确定性的。 所以，正确的选项是（D）。
