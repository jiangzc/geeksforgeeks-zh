先决条件 - 下推自动机 ， 按最终状态下推自动机接受
下推自动机类似于确定性有限自动机，除了它具有比DFA更多的属性。用于实现PDA的数据结构是堆栈。 PDA具有与每个输入相关联的输出。 所有输入都被推入堆栈或被忽略。 用户可以在用于PDA的堆栈上执行基本的推送和弹出操作。 与DFA相关的问题之一是无法计算输入到机器的字符数。 PDA避免了这个问题，因为它使用的堆栈也为我们提供了这种便利。

下推式自动机（PDA）可定义为 -
M =（Q，Σ，Γ，δ，q0，Ζ，F）其中

  • Q是一组有限的状态
  • Σ是一个有限集，称为输入字母表
  • Γ是一个有限集，称为堆栈字母表
  • δ是QX的有限子集（Σ∪{ε}XΓXQXΓ * ）的过渡关系。
  • q0∈Q是开始状态
  • Ζ∈Γ是初始堆栈符号
  • F⊆Q是接受状态的集合

Q）为语言L = {0 n 1 m 2 m 3 n |构建PDA n> = 1，m> = 1}

此PDA使用的方法 -
第一个0被推入堆栈。 然后1被推入堆栈。
然后，对于每2个输入，1将从堆栈中弹出。 如果仍然剩下一些2并且堆栈顶部为0则PDA不接受该字符串。 此后，如果2完成并且堆栈顶部为0，那么对于每3个输入，相等数量的0将从堆栈中弹出。 如果字符串已完成且堆栈为空，则PDA接受字符串，否则不接受。

  • 步骤1：在接收0时将其推入堆栈。 收到1后，将其推入堆栈并转到下一个状态
  • 步骤2：接收1时将其推入堆栈。 在接收2时，从堆栈弹出1并转到下一个状态
  • 步骤3：从堆栈接收2弹出1。 如果所有1都已从堆栈中弹出并且现在接收3，则从堆栈弹出0并转到下一状态
  • 步骤4：从堆栈接收3 pop 0。 如果输入完成且堆栈为空，则转到最后一个状态并接受字符串

例子： 

 Input : 0 0 1 1 1 2 2 2 3 3
Result : ACCEPTED

Input : 0 0 0 1 1 2 2 2 3 3 
Result : NOT ACCEPTED

Q）为语言L = {0 n 1 m |构建PDA n> = 1，m> = 1，m> n + 2}

此PDA使用的方法 -
第一个0被推入堆栈。当0完成时，两个1被忽略。 此后，对于每1输入，从堆栈中弹出0。 当堆栈为空并且仍然剩下一些1时，则忽略所有这些。

  • 步骤1：在接收0时将其推入堆栈。 收到1后，忽略它并转到下一个状态
  • 步骤2：收到1后，忽略它并转到下一个状态
  • 步骤3：收到1后，从堆栈顶部弹出0并进入下一状态
  • 步骤4：收到1后，从堆栈顶部弹出0。 如果堆栈为空，则在收到1个ingore它并转到下一个状态
  • 步骤5：收到1后忽略它。 如果输入完成，则转到最后一个状态

例子： 

 Input : 0 0 0 1 1 1 1 1 1
Result : ACCEPTED

Input : 0 0 0 0 1 1 1 1
Result : NOT ACCEPTED
