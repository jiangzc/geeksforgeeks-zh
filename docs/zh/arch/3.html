寻址模式

寻址模式 - 术语寻址模式是指指定操作数的方式。 寻址模式指定用于在实际执行操作数之前解释或修改指令的地址字段的规则。

8086指令的寻址模式分为两类：

1）寻址数据模式

2）寻址分支模式

8086内存寻址模式提供对内存的灵活访问，允许您轻松访问变量，数组，记录，指针和其他复杂数据类型。 良好的汇编语言编程的关键是正确使用内存寻址模式。



汇编语言程序指令由两部分组成


操作数的内存地址由两部分组成：  

重要条款

  • 内存段的起始地址 。
  • 有效地址或偏移 ：通过添加三个地址元素的任意组合来确定偏移： 位移，基数和索引。
    • 位移：指令中给出的8位或16位立即值。
    • 基数 ：基址寄存器，BX或BP的内容。
    • 索引 ：索引寄存器SI或DI的内容。

根据8086微处理器指定操作数的不同方式，8086使用不同的寻址模式。

下面讨论8086微处理器使用的寻址模式 ：

  • 立即模式 ：立即寻址操作数在指令本身中指定。 在此模式下，数据为8位或16位长，数据是指令的一部分。 

     Example:  MOV AL, 35H (move the data 35H into AL register)
  • 寄存器模式：在寄存器寻址中，操作数放在8位或16位通用寄存器之一中。 数据位于指令指定的寄存器中。 

     Example: MOV AX,CX (move the contents of CX register to AX register)
  • 寄存器间接模式 ：在此寻址中，操作数的偏移量被放置在指令中指定的寄存器BX，BP，SI，DI中的任何一个中。 数据的有效地址位于基址寄存器或指令指定的索引寄存器中。
    8086 CPU允许您使用寄存器间接寻址模式通过寄存器间接访问存储器。 

     MOV AX, [BX](move the contents of memory location s 
             addressed by the register BX to the register AX)
  • 自动递增模式 ：操作数的有效地址是指令中指定的寄存器的内容。 访问操作数后，该寄存器的内容自动递增，指向下一个连续的存储单元。 （R1）+ 。例如： 
     Add R1, (R2)+  // OR 
     R1 = R1 +M[R2]
R2 = R2 + d

    用于在循环中单步执行数组。 R2 - 数组 d的 开始 - 元素的大小

  • 自动递减模式 ：操作数的有效地址是指令中指定的寄存器的内容。 在访问操作数之前，该寄存器的内容会自动递减以指向前一个连续的内存位置。 - （R1）示例： 
     Add R1,-(R2)   //OR 
     R2 = R2- d
R1 = R1 + M[R2]

    自动减量模式与自动增量模式相同。 两者都可以用于实现堆栈作为推送和弹出。 自动递增和自动递减模式对于实现“后进先出”数据结构非常有用。

  • 直接模式 ：操作数的偏移在指令中给出为8位或16位位移元素。 在该寻址模式中，数据的16位有效地址是指令的一部分。 
     Example:ADD AL,[0301]   //add the contents of offset address 0301 to AL
  • 基 址寻址 ：操作数的偏移量是8位或16位位移的总和，基址寄存器BX或BP.BX的内容用作数据段的基址寄存器，BP用作堆栈段的基址寄存器。 
     Example:MOV AL,[BX+05]

    //假设寄存器BX包含0301.偏移量将为0301 + 05 = 0306。内存位置0306的内容将移至AL。

  • 索引寻址模式 ：操作数的偏移量是索引寄存器SI或DI的内容与8位或16位位移的总和。 
     Example:MOV AX, [SI +05]
  • 基于索引的寻址：操作数的偏移量是基址寄存器BX或BP的内容与索引寄存器SI或DI的总和。 
     Example: ADD AX, [BX+SI]
  • 基于索引加位移寻址模式：在这种寻址操作数偏移的模式中，由偏移= [BX或BP] + [SI或DI] +8位或16位位移给出。 
  •  Example:MOV AX, [BX+SI+05]

    分支相关寻址模式：

    • Intrasegment Direct ：有效分支地址是8或16位位移的总和以及IP（指令指针）的当前内容。它可以与条件或无条件分支一起使用。
    • 内部段间接：有效分支地址是使用除立即模式之外的任何数据相关寻址模式访问的寄存器或存储器位置的内容。 它只能用于无条件分支指令。
    • Intersegment Direct ：用指令的一部分替换IP的内容，用指令的另一部分替换CS的内容。 此模式提供了从一个代码段到另一个代码段的分支方式。
    • Intersegment Indirec t：用内存中两个连续字的内容替换IP和CS的内容，这些内容使用除立即和寄存器模式之外的任何一种数据相关寻址模式引用。

    示例GATE问题

    将左侧给出的每个高级语言语句与右侧列出的最自然的寻址模式进行匹配。 

     1. A[1] = B[J]; a. Indirect addressing
2. while [*A++]; b. Indexed addressing
3. int temp = *x; c. Autoincrement

    （A ）（1，c），（2，b），（3，a）
    （B） （1，a），（2，c），（3，b）
    （C） （1，b），（2，c），（3，a）
    （D） （1，a），（2，b），（3，c）

    答案： （C）

    说明： 

     List 1                           List 2
1) A[1] = B[J]; b) Index addressing 
Here indexing is used

2) while [*A++]; c) auto increment
The memory locations are automatically incremented

3) int temp = *x; a) Indirect addressing
Here temp is assigned the value of int type stored
at the address contained in X

    因此（C）是正确的解决方案。
