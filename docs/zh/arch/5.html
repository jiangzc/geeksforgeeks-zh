简化教学计算机（SIC）

简化教学计算机（SIC）是一种假设的计算机，具有通常在真实机器中找到的硬件功能。 这台机器有两个版本：

  1. SIC标准型号
  2. SIC / XE（额外设备或昂贵）

SIC的目标程序可以在SIX / XE上正确执行，这称为向上兼容性。

SIC机器架构/组件 -

  1. 记忆 -
    • 存储器是字节可寻址的，字是由其最低编号字节的位置寻址的。
    • 计算机内存中有2 ^ 15个字节（1个字节= 8位）
      3个连续字节= 1个字（24位= 1个字）
  2. 登记册 -
    SIC中有5个寄存器。 每个寄存器都有一个与之相关的地址，称为寄存器号。 每个寄存器的大小为4个字节。 根据寄存器大小，整数大小是相关的。

    I. A（累加器-0）：用于数学运算。
    II。 X（索引寄存器-1）：用于寻址。
    III。 L（Linkage Register-2）：在子程序的情况下存储指令的返回地址。
    IV。 PC（程序计数器-8）：它保存要执行的下一条指令的地址。
    V. SW（状态字-9）：它包含各种信息



    状态字注册：

    • 模式位指的是用户模式（值= 0）或监督模式（值= 1）。 它占据了1位。[0]
    • state bit指的是进程是处于运行状态（value = 0）还是处于空闲状态（value = 1）。 它也占据了1位。[1]
    • id位指的是进程ID（PID）。 它占用3位。[2-5]
    • CC位是指条件代码，即它告诉设备是否准备就绪。 它占用2位。[6-7]
      掩码位指的是中断掩码。 它占用4位。[8-11]
    • X指未使用的位。 它也占4位。[12-15]
    • ICode指的是中断代码，即中断服务程序。 它占据了剩下的比特。[16-23]
  3. 数据格式 -
    • 整数用24位表示。
    • 负数用2的补码表示。
    • 字符由8位ASCII值表示。
    • 没有浮点表示可用。
  4. 指令格式 -
    SIC中的所有指令都是24位格式。

    • 如果x = 0，则表示直接寻址模式。
    • 如果x = 1，则表示索引寻址模式。
  5. 指令系统 -
    • 加载和存储指令：将数据从累加器移动或存储到存储器，反之亦然。 例如LDA，STA，LDX，STX等。
    • 比较指令：用于通过累加器中的内容比较存储器中的数据。 例如COMP数据。
    • 算术指令：用于对累加器和存储器执行操作，并将结果存储在累加器中。 例如ADD，SUB，MUL，DIV等。
    • 条件跳转：比较累加器和内存的内容，并根据条件执行任务。 例如JLT，JEQ，JGT
    • 子程序链接：与子程序相关的指令。 例如JSUB，RSUB
  6. 输入和输出 -
    它通过从累加器的最右边8位一次传输1个字节来执行。 每个设备都有8位唯一代码。
    有3个I / O指令：

    • 测试设备（TD）测试设备是否准备就绪。 状态字寄存器中的条件代码用于此目的。 如果cc为<则设备准备就绪，否则设备正忙。
    • 读数据（RD）从器件读取一个字节并存储在寄存器A中。
    • 写数据（WD）将寄存器A中的一个字节写入器件。
