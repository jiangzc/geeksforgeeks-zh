优先级中断| （S / W轮询和菊花链）

在I / O接口（中断和DMA模式）中 ，我们讨论了中断启动的I / O背后的概念。

总而言之，当I / O设备准备好进行I / O传输时，它们会向计算机生成中断请求信号。 CPU接收此信号，暂停正在执行的当前指令，然后继续向该传输请求提供服务。 但是如果多个设备同时产生中断呢？ 在这种情况下，我们必须有办法决定首先处理哪个中断。 换句话说，我们必须在所有设备中设置优先级以进行系统中断服务。

在同时请求的情况下，在设备之间定义优先级以便知道哪个首先被服务的概念被称为优先中断系统。 这可以通过软件或硬件方法完成。

软件方法 - 轮询

在此方法中，通过分支到同一服务程序来服务所有中断。 然后，该程序检查每个设备是否是产生中断的设备。 检查顺序由必须设置的优先级决定。 首先检查具有最高优先级的设备，然后按优先级的降序检查设备。 如果检查设备是否产生中断，则调用另一个专门针对该特定设备的服务程序。
结构看起来像这样 -

    device[0].service();

这种方法的主要缺点是速度很慢。 为了克服这个问题，我们可以使用硬件解决方案，其中之一涉及串联设备。 这称为菊花链方法。



硬件方法 - DAISY CHAINING

菊花链方法涉及连接所有可以串行方式请求中断的设备。 此配置由设备的优先级决定。 首先放置具有最高优先级的设备，然后放置第二高优先级设备，依此类推。 给出的图描绘了这种安排。

加工：
有一个中断请求线，它对所有设备都是通用的并进入CPU。

  • 当没有待处理的中断时，该线路处于高电平状态。 但是，如果任何器件产生中断，它会将中断请求线置于低电平状态。
  • CPU从线路确认此中断请求，然后启用中断应答线以响应请求。
  • 在设备1的PI（优先输入）输入处接收该信号。
  • 如果设备未请求中断，它会通过其PO（优先输出）输出将此信号传递给下一个设备。 （PI = 1＆PO = 1）
  • 但是，如果设备已请求中断，（PI = 1＆PO = 0）

    • 器件消耗确认信号并通过将0置于其PO（优先输出）输出来阻止其进一步使用。
    • 然后，器件继续将其中断向量地址（VAD）放入CPU的数据总线中。
    • 器件将其中断请求信号置于高电平状态，以指示其中断已被处理。

    注意： VAD是为该设备提供服务的服务例程的地址。

  • 如果设备在其PI输入处获得0，则它在PO输出处生成0以告知其他设备确认信号已被阻止。 （PI = 0＆PO = 0）

因此，具有PI = 1和PO = 0的设备是请求中断的最高优先级设备。 因此，通过菊花链排列，我们确保优先级最高的中断首先得到服务并建立了层次结构。 设备离第一个设备越远，其优先级越低。
