计算机架构| 弗林的分类学

并行计算是一种计算，其中作业被分解为可以同时执行的离散部分。 每个部分进一步细分为一系列指令。 每个部分的指令在不同的CPU上同时执行。 并行系统处理多个计算机资源的同时使用，这些计算机资源可以包括具有多个处理器的单个计算机，通过网络连接以形成并行处理集群的多个计算机或两者的组合。
并行系统比具有单个处理器的计算机更难编程，因为并行计算机的体系结构相应地变化，并且必须协调和同步多个CPU的过程。

并行处理的关键是CPU。 根据可以同时处理的指令和数据流的数量，计算系统分为四大类：

Flynn的分类 -

  1. 单指令，单数据（SISD）系统 -
    SISD计算系统是单处理器机器，其能够执行在单个数据流上操作的单个指令。 在SISD中，机器指令以顺序方式处理，采用该模型的计算机通常称为顺序计算机。 大多数传统计算机都具有SISD架构。 所有要处理的指令和数据都必须存储在主存储器中。



    SISD模型中处理元件的速度受到计算机内部传输信息的速率的限制（相关）。 主要的代表性SISD系统是IBM PC，工作站。

  2. 单指令多数据（SIMD）系统 -
    SIMD系统是一种多处理器机器，能够在所有CPU上执行相同的指令，但在不同的数据流上运行。 基于SIMD模型的机器非常适合于科学计算，因为它们涉及大量的矢量和矩阵运算。 为了将信息传递给所有处理元件（PE），有组织的矢量数据元素可以被分成多个集合（N个集合用于N个PE系统），并且每个PE可以处理一个数据集。

    主要的代表性SIMD系统是Cray的矢量处理机。

  3. 多指令，单数据（MISD）系统 -
    MISD计算系统是一种多处理器机器，能够在不同的PE上执行不同的指令，但它们都在同一数据集上运行。

    例子Z = sin（x）+ cos（x）+ tan（x）
    系统对同一数据集执行不同的操作。 使用MISD模型构建的机器在大多数应用程序中都没有用，建立了一些机器，但它们都不是商业上可用的。

  4. 多指令，多数据（MIMD）系统 -
    MIMD系统是一种多处理器机器，能够在多个数据集上执行多个指令。 MIMD模型中的每个PE都有独立的指令和数据流; 因此，使用此模型构建的机器可以进行任何类型的应用。 与SIMD和MISD机器不同，MIMD机器中的PE异步工作。

    MIMD机器基于PE耦合到主存储器的方式大致分为共享存储器MIMD和分布式存储器MIMD 。

    在共享内存MIMD模型（紧密耦合的多处理器系统）中，所有PE都连接到单个全局内存，并且它们都可以访问它。 该模型中的PE之间的通信通过共享存储器进行，通过一个PE修改存储在全局存储器中的数据对于所有其他PE是可见的。 主导代表共享内存MIMD系统是Silicon Graphics机器和Sun / IBM的SMP（对称多处理）。
    在分布式存储器MIMD机器（松散耦合的多处理器系统）中，所有PE都具有本地存储器。 该模型中的PE之间的通信通过互连网络（进程间通信信道或IPC）进行。 连接PE的网络可以配置为树形，网状或根据要求。
    共享内存MIMD架构更易于编程，但对故障的容忍度较低，而且相对于分布式内存MIMD模型而言难以扩展。 共享内存MIMD中的故障会影响整个系统，而分布式模型则不是这样，其中每个PE都可以轻松隔离。 此外，共享内存MIMD架构不太可能扩展，因为添加更多PE会导致内存争用。 这种情况在分布式存储器的情况下不会发生，其中每个PE都有自己的存储器。 由于实际结果和用户的要求，分布式存储器MIMD架构优于其他现有模型。
