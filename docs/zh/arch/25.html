CPU Cache和TLB有什么区别？

CPU Cache和TLB都是微处理器中使用的硬件，但有什么区别，特别是有人说TLB也是一种Cache？

首先是第一件事。 CPU Cache是一种快速内存，用于改善从主存储器（RAM）到CPU寄存器的信息获取延迟。 因此CPU Cache位于主内存和CPU之间。 此缓存暂时存储信息，以便下次访问相同信息的速度更快。 用于存储可执行指令的CPU缓存，称为指令缓存（I-Cache）。 用于存储数据的CPU缓存，称为数据缓存（D-Cache）。 因此，I-Cache和D-Cache分别加快了指令和数据的获取时间。 现代处理器包含I-Cache和D-Cache。 为了完整起见，我们也讨论一下D-cache层次结构。 D-Cache通常以层次结构组织，即1级数据高速缓存，2级数据高速缓存等。应当注意，与L2 D-Cache相比，L1 D-Cache更快/更小/更昂贵。 但“ CPU缓存 ”的基本思想是加快从主存储器到CPU的指令/数据获取时间。

仅当处理器使用虚拟内存时才需要转换后备缓冲区（即TLB） 。 简而言之，TLB通过将页表存储在更快的内存中来加速虚拟地址到物理地址的转换。 事实上，TLB也位于CPU和主内存之间。 准确地说，当物理地址需要转换为虚拟地址时，MMU使用TLB。 通过将虚拟物理地址的映射保持在快速存储器中，对页表的访问得到改善。 应当注意，页表（其本身存储在RAM中）跟踪虚拟页面在物理存储器中的存储位置。 从这个意义上讲，TLB也可以被视为页表的缓存。

但TLB和CPU Cach e的操作范围是不同的。 TLB是关于“加速虚拟内存的地址转换”，因此不需要为每个地址访问页表。 CPU Cache是​​关于“加速主存储器访问延迟”，因此CPU不会始终访问RAM。 TLB操作在MMU进行地址转换时进行，而CPU高速缓存操作在CPU进行内存访问时进行。 事实上，任何现代处理器都部署了所有I-Cache，L1和L2 D-Cache以及TLB。
