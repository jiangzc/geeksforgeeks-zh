I / O接口（中断和DMA模式）

用于在内部存储器和外部I / O设备之间传输信息的方法称为I / O接口。 通过连接到任何计算机系统的外围设备使用特殊通信链路连接CPU。 这些通信链路用于解决CPU和外设之间的差异。 CPU和外围设备之间存在特殊的硬件组件，用于监督和同步所有称为接口单元的输入和输出传输。

转让方式：

从外部设备接收的二进制信息通常存储在存储器单元中。 从CPU传输到外部设备的信息源自存储器单元。 CPU仅处理信息，但源和目标始终是内存单元。 CPU和I / O设备之间的数据传输可以以不同的模式完成。

可以以三种可能的方式中的任何一种来进行与外围设备之间的数据传输

  1. 程序化I / O.
  2. 中断启动的I / O.
  3. 直接内存访问（DMA）。

现在让我们逐个讨论每种模式。

  1. 编程I / O：这是由于计算机程序中写入的I / O指令的结果。 每个数据项传输由程序中的指令启动。 通常传输来自CPU寄存器和存储器。 在这种情况下，它需要由外围设备的CPU进行持续监视。
    编程I / O示例：在这种情况下，I / O设备无法直接访问存储器单元。 从I / O设备到存储器的传输需要CPU执行多个指令，包括用于将数据从设备传输到CPU的输入指令以及用于将数据从CPU传输到存储器的存储指令。 在编程的I / O中，CPU保持在程序循环中，直到I / O单元指示它已准备好进行数据传输。 这是一个耗时的过程，因为它不必要地使CPU忙碌。 使用中断工具可以避免这种情况。 这将在下面讨论。
  2. 中断启动的I / O：因为在上面的例子中我们看到CPU不必要地忙着。 通过使用中断驱动的方法进行数据传输，可以很好地避免这种情况。 通过使用中断工具和特殊命令，可以在任何设备提供数据时通知接口发出中断请求信号。 与此同时，CPU可以继续执行任何其他程序。 接口同时持续监控设备。 每当确定设备准备好进行数据传输时，它就向计算机发起中断请求信号。 在检测到外部中断信号时，CPU暂时停止其已执行的任务，分支到服务程序以处理I / O传输，然后返回到它最初执行的任务。
  3. 直接内存访问 ：快速存储介质（如磁盘和内存单元）之间的数据传输受CPU速度的限制。 因此，我们可以允许外围设备使用存储器总线直接相互通信，从而消除了CPU的干预。 这种类型的数据传输技术称为DMA或直接存储器访问。 在DMA期间，CPU处于空闲状态，并且无法控制内存总线。 DMA控制器接管总线以直接管理I / O设备和存储器单元之间的传输。





    4. 总线请求： DMA控制器使用它来请求CPU放弃对总线的控制。

    总线授权：由CPU激活，通知外部DMA控制器总线处于高阻态，请求DMA可以控制总线。 一旦DMA控制了总线，它就会传输数据。 这种转移可以以多种方式进行。
