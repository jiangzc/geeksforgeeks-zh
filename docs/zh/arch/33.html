为了提高CPU的性能，我们有两个选择：
1）通过引入更快的电路来改善硬件。
2）安排硬件，以便可以同时执行多个操作。

由于硬件速度有限制，而且电路速度较快，我们必须采用第二种选择。

流水线操作：流水线操作是一种安排CPU硬件元素的过程，以提高其整体性能。 在流水线处理器中同时执行多个指令。

让我们看一个真实的例子，它适用于流水线操作的概念。 考虑一下水瓶包装厂。 让瓶子通过3个阶段，插入瓶子（ I ），在瓶子里加水（ F ），然后密封瓶子（ S ）。 让我们将这些阶段分别视为第1阶段，第2阶段和第3阶段。 让每个阶段花1分钟完成其操作。
现在，在非流水线操作中，首先将瓶子插入工厂中，1分钟后将其移至第2阶段，在那里填充水。 现在，在第一阶段没有任何事情发生。 类似地，当瓶子​​移动到阶段3时，阶段1和阶段2都是空闲的。 但是在流水线操作中，当瓶子处于阶段2时，可以在阶段1加载另一个瓶子。类似地，当瓶子​​处于阶段3时，在阶段1和阶段2中可以分别存在一个瓶子。因此，在每分钟之后，我们在第3阶段结束时得到一个新瓶子。因此，制造1瓶所需的平均时间是：

没有流水线 = 9/3分钟= 3米

基本管道的设计

  • 在流水线处理器中，管道有两端，即输入端和输出端。 在这些端之间，存在多个级/段，使得一级的输出连接到下一级的输入，并且每级执行特定操作。
  • 接口寄存器用于保持两级之间的中间输出。 这些接口寄存器也称为锁存器或缓冲器。
  • 流水线中的所有级以及接口寄存器都由公共时钟控制。

管道阶段

RISC处理器具有5级指令流水线，用于执行RISC指令集中的所有指令。 以下是RISC管道的5个阶段及其各自的操作：

  • 第1阶段（取指令）
    在此阶段，CPU从存储器中的地址读取指令，该存储器的值存在于程序计数器中。
  • 第2阶段（指令解码）
    在此阶段，解码指令并访问寄存器文件以从指令中使用的寄存器中获取值。
  • 第3阶段（指令执行）
    在此阶段，执行ALU操作。
  • 第4阶段（内存访问）
    在此阶段，存储器操作数从/向指令中存在的存储器读取和写入。
  • 第五阶段（回写）
    在此阶段，计算/取出的值被写回指令中的寄存器。

流水线处理器的性能
考虑一个'k'段流水线，时钟周期时间为'Tp'。 让流水线处理器完成'n'任务。 现在，第一条指令将从管道中取出“k”个周期，但其他“n - 1”指令每个只需“1”个周期，即总共“n - 1”个周期。 因此，在流水线处理器中执行'n'指令所花费的时间： 

 ET pipeline = k + n – 1 cycles
                              = (k + n – 1) Tp

在相同的情况下，对于非流水线处理器，'n'指令的执行时间将是： 

 ET non-pipeline = n * k * Tp

因此，当在同一处理器上执行'n'任务时，流水线处理器在非流水线处理器上的加速（S）为： 

 S = Performance of pipelined processor /
        Performance of Non-pipelined processor

由于处理器的性能与执行时间成反比，我们有， 

 S = ET non-pipeline / ET pipeline
    => S = [n * k * Tp] / [(k + n – 1) * Tp]
       S = [n * k] / [k + n – 1]

当任务'n'的数量明显大于k时，即n >> k 

 S = n * k / n
    S = k

其中'k'是管道中的阶段数。

此外， 效率 =给定加速/最大加速= S / S 最大
我们知道，Smax = k

所以， 效率 = S / k

吞吐量 =指令数/完成指令的总时间

因此， 吞吐量 = n /（k + n - 1）* Tp

注意：理想流水线处理器的每指令周期数（CPI）值为1
