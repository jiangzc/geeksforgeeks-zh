指令格式（零，一，二和三地址指令）

计算机根据提供的指令执行任务。 计算机中的指令包括称为字段的组。 这些字段包含不同的信息，因为计算机中的每个事物都在0和1中，因此每个字段具有不同的重要性，CPU在此基础上决定执行的操作。 最常见的领域是：

  • 操作字段，指定要添加的操作。
  • 包含操作数位置的地址字段，即寄存器或存储器位置。
  • 模式字段，指定如何建立操作数。

指令的长度取决于它包含的地址数。 通常，CPU组织根据地址字段的数量有三种类型：

  1. 单一累积组织
  2. 一般登记组织
  3. 堆栈组织

在第一次组织操作中，涉及一个称为累加器的特殊寄存器。 在第二个上，多个寄存器用于计算目的。 在第三个组织中，基于堆栈的操作由于它不包含任何地址字段。 没有必要只应用一个组织，各种组织的混合大多是我们通常所看到的。

根据地址指令的数量分类为：

请注意，我们将使用X =（A + B）*（C + D）表达式来展示该过程。



  1. 零地址说明 -

    基于堆栈的计算机不在指令中使用地址字段。要首先计算表达式，它将转换为尊重波兰表示法，即修复后表示法。 

     Expression: X = (A+B)*(C+D)
Postfixed : X = AB+CD+*
TOP means top of stack
M[X] is any memory location
    推 一个 TOP = A.
    推 乙 TOP = B.
    加 TOP = A + B.
    推 C TOP = C.
    推 d TOP = D.
    加 TOP = C + D.
    MUL TOP =（C + D）*（A + B）
    POP X M [X] = TOP
  2. 一个地址说明 -
    这使用隐含的ACCUMULATOR寄存器进行数据操作。一个操作数位于累加器中，另一个位于寄存器或存储器位置。隐含意味着CPU已经知道一个操作数在累加器中，因此不需要指定它。 

     Expression: X = (A+B)*(C+D)
AC is accumulator
M[] is any memory location
M[T] is temporary location
    加载 一个 AC = M [A]
    加 乙 AC = A [C] + M [B]
    商店 Ť M [T] = AC
    加载 C AC = M [C]
    加 d AC = AC + M [D]
    MUL Ť AC = AC * M [T]
    商店 X M [X] = AC
  3. 两个地址说明 -
    这在商用计算机中是常见的。在指令中可以指定两个地址。与之前的一个地址指令不同，结果存储在累加器中，结果cab存储在不同的位置而不仅仅是累加器，但需要更多的位来表示地址。


    这里目标地址也可以包含操作数。 

     Expression: X = (A+B)*(C+D)
R1, R2 are registers
M[] is any memory location
    MOV R1，A R1 = M [A]
    加 R1，B R1 = R1 + M [B]
    MOV R2，C R2 = C.
    加 R2，D R2 = R2 + D.
    MUL R1，R2 R1 = R1 * R2
    MOV X，R1 M [X] = R1
  4. 三个地址说明 -
    这有三个地址字段来指定寄存器或存储器位置。 创建的程序大小很短，但每条指令的位数增加。 这些指令使程序的创建变得更加容易，但这并不意味着程序运行得更快，因为现在指令只包含更多信息，但每个微操作（改变寄存器内容，地址总线中的加载地址等）将在一个周期内执行只要。 

     Expression: X = (A+B)*(C+D)
R1, R2 are registers
M[] is any memory location
    加 R1，A，B R1 = M [A] + M [B]
    加 R2，C，D R2 = M [C] + M [D]
    MUL X，R1，R2 M [X] = R1 * R2
